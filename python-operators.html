Python Operators: Complete University-Level Guide (2026)

Operators are the fundamental building blocks of every Python expression. Whether you're computing values, comparing objects, manipulating bits, or controlling logic flow, operators are the symbols and keywords that tell Python exactly what to do. This university-level guide covers every operator category in Python 3.12+ — from familiar arithmetic and comparison operators to the nuanced walrus operator, bitwise tricks, and the full power of operator overloading via dunder methods.

You will find 40+ runnable examples in this guide. Every example is executable directly in your browser via the built-in Python runtime powered by Pyodide — no installation needed. By the end of this guide you will understand not just the syntax of each operator, but its semantics, performance characteristics, and the idiomatic Python patterns that surround it.

1.1 What Is an Operator?

In Python, an operator is a special symbol (or keyword) that performs an operation on one or more operands. The expression 3 + 5 uses the + operator with operands 3 and 5. Operators are categorised by the number of operands they act on: unary operators take one operand (e.g., -x, not x), and binary operators take two (e.g., a + b, a == b). Python also supports one ternary operator — the conditional expression x if condition else y.

Under the hood, every Python operator maps to a corresponding dunder (double-underscore) method on the operand's type. When Python evaluates a + b, it calls type(a).__add__(a, b). This dispatch mechanism is what makes operator overloading possible and is the key insight that unifies the entire operator system.

2.0 ARITHMETIC OPERATORS

2.1 The Seven Arithmetic Operators

Python provides seven arithmetic operators. Addition (+), subtraction (-), and multiplication (*) work as expected on both integers and floats. True division (/) always returns a float in Python 3, even when dividing two integers. Floor division (//) returns the largest integer less than or equal to the true quotient — it rounds toward negative infinity, not toward zero. The modulo operator (%) returns the remainder after floor division. Exponentiation (**) raises a number to a power and supports fractional exponents for roots.

2.2 Floor Division vs. Truncation

A subtle but important distinction: // performs floor division (toward negative infinity), while C-style integer division truncates toward zero. In Python, -7 // 2 gives -4 (floor), not -3 (truncation). The relationship a == (a // b) * b + (a % b) always holds, which is why Python's modulo can return positive results even when the dividend is negative.

💡 Performance Tip: For integer arithmetic, // and % are both computed in a single CPU division instruction. Python's divmod(a, b) built-in returns (a // b, a % b) simultaneously and is slightly faster when you need both values.

2.3 The Unary Operators

Python supports three unary arithmetic operators: -x (negation), +x (unary plus, rarely needed but legal), and ~x (bitwise NOT, covered in section 5). These bind more tightly than binary operators, so -x**2 is parsed as -(x**2), not (-x)**2 — a common source of bugs.

⚠ Caution: -x**2 does NOT square a negative number. Python parses it as -(x**2) because ** has higher precedence than unary -. Write (-x)**2 explicitly if you intend to negate first.

EXAMPLE 2.1 — All Arithmetic Operators in Action 
# ── Arithmetic Operators ──────────────────────────────────────────
a, b = 17, 5

print(f"Addition:        {a} + {b}  = {a + b}")    # 22
print(f"Subtraction:     {a} - {b}  = {a - b}")    # 12
print(f"Multiplication:  {a} * {b}  = {a * b}")    # 85
print(f"True Division:   {a} / {b}  = {a / b}")    # 3.4  (always float)
print(f"Floor Division:  {a} // {b} = {a // b}")   # 3
print(f"Modulo:          {a} % {b}  = {a % b}")    # 2
print(f"Exponentiation:  {a} ** {b} = {a ** b}")   # 1419857

# ── Floor division toward negative infinity ───────────────────────
print(f"\nFloor of -7 // 2  = {-7 // 2}")     # -4 (NOT -3)
print(f"Modulo  -7 %  2   = {-7 % 2}")      # 1  (always same sign as divisor)

# ── divmod returns both quotient and remainder ────────────────────
q, r = divmod(17, 5)
print(f"\ndivmod(17, 5) = ({q}, {r})")      # (3, 2)

# ── Operator precedence: ** before unary minus ────────────────────
x = 3
print(f"\n-x**2  = {-x**2}")        # -9   (-(3**2))
print(f"(-x)**2 = {(-x)**2}")       # 9    ((-3)**2)

# ── Fractional exponents for roots ───────────────────────────────
print(f"\nSquare root of 144: {144 ** 0.5}")    # 12.0
print(f"Cube root of 27:    {27 ** (1/3):.6f}") # 3.000000

# ── String and list operators reuse + and * ───────────────────────
print("\n" + "─" * 30)
words = ["hello"] * 3
print(words)                 # ['hello', 'hello', 'hello']
print("Py" + "thon")          # Python

3.0 COMPARISON OPERATORS

3.1 The Six Comparison Operators

Python's six comparison operators are == (equal), != (not equal), < (less than), > (greater than), <= (less than or equal), and >= (greater than or equal). They always return a bool (True or False) and can be applied to any objects that implement the corresponding dunder methods.

3.2 Chained Comparisons

Python uniquely supports chained comparisons: 0 < x < 10 is legal and means exactly what it says in mathematics — it evaluates to True only when both 0 < x and x < 10 are true. Internally, each intermediate value is evaluated only once, so chaining is both readable and efficient. You can chain any number of comparisons: a < b <= c < d.

3.3 Comparing Objects

By default, custom objects compare by identity (same as is). To enable meaningful value-based equality, define __eq__ on your class. To enable ordering (<, >, etc.), define __lt__ and decorate with @functools.total_ordering to auto-generate the remaining four comparison methods. The @dataclass(order=True) decorator generates all six comparison methods automatically based on field declaration order.

💡 Idiomatic Python: Use chained comparisons whenever checking a value falls within a range: if 18 <= age < 65: is clearer and more Pythonic than if age >= 18 and age < 65:.

EXAMPLE 3.1 — Comparison Operators and Chained Comparisons 
# ── Basic comparisons ─────────────────────────────────────────────
a, b = 10, 20
print(a == b)    # False
print(a != b)    # True
print(a <  b)    # True
print(a >  b)    # False
print(a <= 10)  # True
print(b >= 20)  # True

# ── Chained comparisons ───────────────────────────────────────────
score = 75
print(f"\n{score} in [60,100)? {60 <= score < 100}")   # True

# Each intermediate value computed only once:
def mid():
    print("  mid() called")
    return 5

result = 1 < mid() < 10
print(f"  result = {result}")    # mid() printed once, result is True

# ── Comparing strings (lexicographic order) ───────────────────────
print(f"\n'apple' < 'banana': {'apple' < 'banana'}")   # True
print(f"'Zoo'   < 'ant':    {'Zoo' < 'ant'}")         # True (uppercase < lowercase)

# ── Comparing lists (element by element) ─────────────────────────
print(f"\n[1,2,3] == [1,2,3]: {[1,2,3] == [1,2,3]}")  # True
print(f"[1,2,4] > [1,2,3]:  {[1,2,4] > [1,2,3]}")   # True (4 > 3 at index 2)

# ── Custom class with comparison ─────────────────────────────────
import functools

@functools.total_ordering
class Version:
    def __init__(self, major, minor, patch):
        self.v = (major, minor, patch)
    def __eq__(self, other): return self.v == other.v
    def __lt__(self, other): return self.v < other.v
    def __repr__(self): return f"v{'.'.join(map(str, self.v))}"

v1 = Version(1, 9, 0)
v2 = Version(2, 0, 1)
print(f"\n{v1} < {v2}: {v1 < v2}")    # True
print(f"{v2} > {v1}: {v2 > v1}")    # True (auto-generated by total_ordering)
versions = [Version(3,0,0), v1, v2]
print(f"Sorted: {sorted(versions)}")

4.0 LOGICAL OPERATORS

4.1 and, or, not

Python has three logical operators: and, or, and not. Unlike many languages where logical operators always return a boolean, Python's and and or return one of their operands — specifically the operand whose value determines the result. This makes them powerful tools for concise, idiomatic code.

4.2 Short-Circuit Evaluation

and evaluates left to right and returns the first falsy operand, or the last operand if all are truthy. or returns the first truthy operand, or the last if all are falsy. Neither operator evaluates the right side if the result is already determined from the left — this is called short-circuit evaluation. This matters when the right operand has side effects (function calls, I/O) or when it would raise an error if the left side is falsy.

4.3 Truthy and Falsy Values

Python defines a clear set of falsy values: False, None, 0, 0.0, 0j, empty sequences ("", [], ()), empty mappings ({}), and empty sets. Everything else is truthy. Objects can control their truthiness by implementing __bool__ (preferred) or __len__.

💡 Idiomatic Pattern: value = user_input or "default" assigns "default" when user_input is falsy. This is a common Python idiom, though for None-specifically, prefer value = user_input if user_input is not None else "default" to avoid swallowing 0 or "".

EXAMPLE 4.1 — Logical Operators, Short-Circuit, and Truthiness 
# ── and / or return an operand, not always a bool ─────────────────
print(5 and 10)        # 10   (both truthy → return last)
print(0 and 10)        # 0    (first falsy → short-circuit)
print(5 or  10)        # 5    (first truthy → short-circuit)
print(0 or  10)        # 10   (first falsy, return last)
print(0 or  None)      # None (both falsy → return last)
print(not 0)           # True
print(not 42)          # False

# ── Short-circuit prevents the error on the right ─────────────────
data = None
result = data and data[0]   # safe: right side never evaluated
print(f"\nSafe access: {result}")   # None

# ── Idiomatic default value pattern ───────────────────────────────
user_name = ""
display = user_name or "Anonymous"
print(f"Display name: {display}")   # Anonymous

# ── Truthiness: what counts as falsy? ─────────────────────────────
falsy_values = [False, None, 0, 0.0, "", [], {}, ()]
print("\nFalsy values:")
for val in falsy_values:
    print(f"  bool({val!r:8}) = {bool(val)}")

# ── Custom __bool__ ───────────────────────────────────────────────
class Wallet:
    def __init__(self, balance): self.balance = balance
    def __bool__(self): return self.balance > 0
    def __repr__(self): return f"Wallet({self.balance})"

w1, w2 = Wallet(50), Wallet(0)
print(f"\nbool({w1}) = {bool(w1)}")    # True
print(f"bool({w2}) = {bool(w2)}")    # False
if w1:
    print("Wallet has funds!")

5.0 BITWISE OPERATORS

5.1 Bit-Level Operations

Bitwise operators work directly on the binary representation of integers. Python supports six bitwise operators: AND (&), OR (|), XOR (^), NOT (~), left shift (<<), and right shift (>>). Unlike logical operators, bitwise operators operate on every individual bit of their integer operands and always return an integer.

5.2 Real-World Applications

Permission flags: Bitwise OR combines flags (READ | WRITE), AND checks for a flag (perms & READ), and XOR toggles a flag (perms ^ WRITE). Efficient multiplication/division by powers of two: x << 1 is equivalent to x * 2 and x >> 1 to x // 2 — though Python's compiler often handles this optimisation automatically. Hashing and checksums: XOR is used in many checksum algorithms because it is its own inverse: (a ^ b) ^ b == a.

5.3 Bitwise NOT and Two's Complement

The unary NOT operator ~x inverts all bits. For a positive integer n, this gives ~n == -(n+1), which follows from Python's use of two's complement representation. This means ~0 == -1, ~1 == -2, and so on. Python integers have arbitrary precision, so bitwise NOT on a positive integer produces a negative integer.

💡 Modern Usage: In Python 3.9+, the | operator was extended to merge dictionaries: merged = dict_a | dict_b. The |= augmented assignment updates a dict in place. This is a great example of how Python evolves operator semantics for new use cases.

EXAMPLE 5.1 — Bitwise Operators and Permission Flags 
# ── Bitwise fundamentals ──────────────────────────────────────────
a, b = 0b1010, 0b1100   # 10, 12
print(f"a      = {a:04b}  ({a})")
print(f"b      = {b:04b}  ({b})")
print(f"a & b  = {a&b:04b}  ({a&b})   AND")
print(f"a | b  = {a|b:04b}  ({a|b})   OR")
print(f"a ^ b  = {a^b:04b}  ({a^b})   XOR")
print(f"~a     = {~a}           NOT  (~10 = -(10+1))")
print(f"a << 1 = {a<<1:04b}  ({a<<1})   left shift  (×2)")
print(f"a >> 1 = {a>>1:04b}   ({a>>1})    right shift (÷2)")

# ── Permission flags ──────────────────────────────────────────────
READ    = 0b001   # 1
WRITE   = 0b010   # 2
EXECUTE = 0b100   # 4

user_perms = READ | WRITE      # 3 — read + write
print(f"\nUser perms: {user_perms:03b} ({user_perms})")

print(f"Can read?    {bool(user_perms & READ)}")     # True
print(f"Can write?   {bool(user_perms & WRITE)}")    # True
print(f"Can execute? {bool(user_perms & EXECUTE)}")  # False

user_perms ^= WRITE            # toggle write off
print(f"After toggle write: {user_perms:03b} ({user_perms})")  # 001 (1)

user_perms |= EXECUTE          # grant execute
print(f"After grant execute:{user_perms:03b} ({user_perms})")  # 101 (5)

# ── XOR swap (no temp variable) ───────────────────────────────────
x, y = 42, 99
x ^= y; y ^= x; x ^= y
print(f"\nAfter XOR swap: x={x}, y={y}")   # x=99, y=42

# ── Dict merge with | (Python 3.9+) ──────────────────────────────
defaults = {"color": "blue", "size": 12}
overrides = {"size": 16, "bold": True}
merged = defaults | overrides
print(f"\nMerged dict: {merged}")

6.0 ASSIGNMENT AND AUGMENTED ASSIGNMENT OPERATORS

6.1 Basic Assignment

The assignment operator = binds a name to an object in the current scope. Python assignment does not copy the object — it creates a reference. This means a = b makes a point to the same object as b. For immutable objects like integers and strings, this distinction rarely matters. For mutable objects like lists and dicts, understanding references is essential to avoid unintended aliasing.

6.2 Augmented Assignment

Augmented assignment operators combine an arithmetic or bitwise operation with assignment: +=, -=, *=, /=, //=, %=, **=, &=, |=, ^=, <<=, >>=. For immutable types, x += 1 creates a new object and rebinds the name. For mutable types that implement __iadd__, it modifies the object in place — meaning a += [1] modifies the list a refers to without rebinding the name.

6.3 Multiple and Unpacking Assignment

Python supports multiple assignment (a = b = 0), tuple unpacking (x, y = 1, 2), and extended unpacking with the star operator (first, *rest = [1, 2, 3, 4]). Swapping two variables is idiomatic and fast: a, b = b, a — Python evaluates the right side fully before any assignment occurs.

⚠ Mutable Aliasing: a = b = [] creates a single list and both names point to it. Appending via a changes what you see via b. If you want two independent lists, write a, b = [], [] or a = []; b = a.copy().

EXAMPLE 6.1 — Assignment, Augmented Assignment, and Unpacking 
# ── Augmented assignment operators ────────────────────────────────
n = 10
n += 5;  print(f"n += 5  → {n}")   # 15
n -= 3;  print(f"n -= 3  → {n}")   # 12
n *= 2;  print(f"n *= 2  → {n}")   # 24
n //= 5; print(f"n //= 5 → {n}")   # 4
n **= 3; print(f"n **= 3 → {n}")   # 64
n %= 10; print(f"n %= 10 → {n}")   # 4

# ── += on a list mutates in place (in-place __iadd__) ────────────
a = [1, 2]
b = a
a += [3]
print(f"\na is b after +=: {a is b}")  # True — same object
print(f"a = {a}, b = {b}")            # both [1, 2, 3]

# ── Tuple unpacking ───────────────────────────────────────────────
x, y, z = 1, 2, 3
print(f"\nx={x}, y={y}, z={z}")

# ── Swap without temp variable ────────────────────────────────────
x, y = y, x
print(f"After swap: x={x}, y={y}")   # x=2, y=1

# ── Extended unpacking with * ─────────────────────────────────────
first, *middle, last = [10, 20, 30, 40, 50]
print(f"\nfirst={first}, middle={middle}, last={last}")

# ── Nested unpacking ─────────────────────────────────────────────
(a, b), c = (1, 2), 3
print(f"a={a}, b={b}, c={c}")

# ── Mutable aliasing trap ─────────────────────────────────────────
p = q = []        # SAME list!
p.append(99)
print(f"\np is q: {p is q}")   # True
print(f"q = {q}")             # [99] — both names see the change

7.0 IDENTITY AND MEMBERSHIP OPERATORS

7.1 Identity: is and is not

The is operator tests object identity — it returns True only if both operands refer to the exact same object in memory (i.e., they have the same id()). This is fundamentally different from ==, which tests value equality. The canonical use of is is comparing against singletons: if x is None:, if result is True:. Never use is to compare integers, strings, or other value types, even though Python's integer and string interning may make it appear to work for small values.

7.2 Membership: in and not in

The in and not in operators test whether a value is a member of a sequence or collection. For lists, in performs a linear search — O(n). For sets and dict keys, it performs a hash lookup — O(1). For strings, it checks for substring membership. Custom classes can implement __contains__ to define membership semantics.

⚠ Common Bug: Due to Python's small integer cache (typically -5 to 256) and string interning, a is b may return True for small integers and simple strings even when they are assigned independently. This is a CPython implementation detail and must never be relied upon. Always use == for value equality.

EXAMPLE 7.1 — Identity and Membership Operators 
# ── is vs == ─────────────────────────────────────────────────────
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(f"a == b:  {a == b}")    # True  — same values
print(f"a is b:  {a is b}")    # False — different objects
print(f"a is c:  {a is c}")    # True  — same object
print(f"id(a)={id(a)}, id(b)={id(b)}, id(c)={id(c)}")

# ── Correct: use is only with singletons ──────────────────────────
value = None
if value is None:
    print("\nvalue is None — use 'is', not '=='")

# ── in / not in on different containers ──────────────────────────
fruits = ["apple", "banana", "cherry"]
fruit_set = {"apple", "banana", "cherry"}

print(f"\n'banana' in list:    {'banana' in fruits}")    # True  O(n)
print(f"'grape'  in set:     {'grape' in fruit_set}")    # False O(1)
print(f"'key1'   in dict:    {'key1' in {'key1': 1}}")   # True  O(1)
print(f"'nan'    in string:  {'nan' in 'banana'}")        # True  substring

# ── Custom __contains__ ──────────────────────────────────────────
class Range:
    def __init__(self, lo, hi): self.lo, self.hi = lo, hi
    def __contains__(self, x): return self.lo <= x <= self.hi
    def __repr__(self): return f"Range({self.lo}, {self.hi})"

teens = Range(13, 19)
print(f"\n15 in {teens}: {15 in teens}")    # True
print(f"25 in {teens}: {25 in teens}")    # False
print(f"10 not in {teens}: {10 not in teens}")  # True

8.0 OPERATOR PRECEDENCE AND ASSOCIATIVITY

8.1 Precedence Table (Highest to Lowest)

When multiple operators appear in an expression, Python uses a fixed precedence table to determine the order of evaluation. Operators with higher precedence bind more tightly and are evaluated first. Operators at the same level are evaluated left to right (left-associative), with the sole exception of the exponentiation operator **, which is right-associative: 2 ** 3 ** 2 is parsed as 2 ** (3 ** 2) = 512, not (2 ** 3) ** 2 = 64.

💡 Best Practice: When in doubt, use parentheses. They make intent explicit, eliminate ambiguity, and cost nothing in terms of runtime performance. Clarity always beats cleverness in production code.

EXAMPLE 8.1 — Precedence, Associativity, and Tricky Expressions 
# ── Exponentiation is right-associative ──────────────────────────
print(2 ** 3 ** 2)         # 512 → 2 ** (3**2) = 2**9
print((2 ** 3) ** 2)       # 64  → explicit left-assoc

# ── Mixed precedence ─────────────────────────────────────────────
print(2 + 3 * 4)           # 14  (* before +)
print((2 + 3) * 4)         # 20  (parens first)
print(8 >> 1 + 1)          # 2   (+ before >>: 8 >> 2)

# ── Comparison vs bitwise: common trap ───────────────────────────
x = 6
print(x & 3 == 2)          # False → x & (3==2) → 6 & False → 0
print((x & 3) == 2)        # True  → (6 & 3) == 2 → 2 == 2

# ── Logical operator precedence: not > and > or ──────────────────
print(True or False and False)   # True  (and before or)
print((True or False) and False) # False
print(not True or True)          # True  (not binds tighter than or)

# ── Chained assignment evaluates right to left ────────────────────
a = b = c = 0
print(f"\na={a}, b={b}, c={c}")

# ── Useful rule: use parens for anything beyond simple +/-/*/ ─────
result = (1 + 2) * (3 ** 2) - (4 // 3)
print(f"result = {result}")   # (3) * (9) - (1) = 26

9.0 THE WALRUS OPERATOR AND TERNARY EXPRESSIONS

9.1 The Walrus Operator (:=)

The walrus operator :=, introduced in Python 3.8, is an assignment expression. Unlike the regular = statement, := can appear inside expressions — including while conditions, list comprehensions, and if conditions. Its primary benefit is avoiding double evaluation: when you want to check and use the result of a function call in the same expression, := assigns the value to a variable and returns it, so the function is called only once.

9.2 The Ternary Conditional Expression

Python's ternary operator has the form value_if_true if condition else value_if_false. It evaluates only one of the two value expressions — the branch not selected is never evaluated. Ternary expressions are useful for concise inline conditionals but should be avoided when the condition or values are complex enough to hurt readability. For deeply nested ternaries, a regular if/elif/else block is always clearer.

💡 Walrus Best Use Cases: The walrus operator shines in three scenarios: (1) while chunk := f.read(8192): to read until EOF, (2) filtering and transforming in a single comprehension pass, and (3) caching a costly computation inside an if condition to reuse it in the body.

EXAMPLE 9.1 — Walrus Operator and Ternary Expressions 
# ── Ternary expression ────────────────────────────────────────────
age = 20
status = "adult" if age >= 18 else "minor"
print(f"Age {age} → {status}")          # adult

grade = 85
letter = ("A" if grade >= 90
          else "B" if grade >= 80
          else "C" if grade >= 70
          else "F")
print(f"Grade {grade} → {letter}")       # B

# ── Walrus in while loop (process items) ─────────────────────────
numbers = [4, 9, 16, 0, 25]
idx = 0
print("\nSquare roots until zero:")
while idx < len(numbers) and (n := numbers[idx]) != 0:
    print(f"  √{n} = {n**0.5:.2f}")
    idx += 1

# ── Walrus in comprehension: filter and reuse computed value ──────
import math

values = [-4, 9, -1, 16, 25]
# Without walrus: sqrt computed twice
roots_old = [math.sqrt(v) for v in values if v >= 0]

# With walrus: filter happens before sqrt, sqrt computed once
roots_new = [root for v in values if (root := math.sqrt(v)) > 3 if v >= 0]
print(f"\nRoots > 3: {roots_new}")   # [4.0, 5.0]

# ── Walrus in if: assign and test in one line ─────────────────────
import re
text = "Error 404: Page not found"
if (m := re.search(r"Error (\d+)", text)):
    print(f"\nError code found: {m.group(1)}")   # 404

# ── Walrus captures expensive call once ──────────────────────────
data = [3, 7, 1, 9, 2]
if (n := len(data)) > 3:

10.0 OPERATOR OVERLOADING

10.1 The Dunder Protocol

Python's operator overloading system is based entirely on dunder (double-underscore) methods. When Python evaluates a + b, it first tries type(a).__add__(a, b). If that returns NotImplemented, it tries the reflected version type(b).__radd__(b, a). This two-step protocol allows two different classes to cooperate: if Vector + Scalar doesn't know how to handle the scalar, the scalar's __radd__ gets a chance to respond. In-place operators like += call __iadd__ first, falling back to __add__.

10.2 The Matrix Multiplication Operator (@)

Python 3.5 introduced the @ operator (dunder __matmul__) specifically for matrix multiplication, used extensively in NumPy and scientific computing. The standard library does not implement @ for built-in types — it exists purely for third-party libraries and user-defined classes, making it a clean, unambiguous choice for any matrix or dot-product operation.

10.3 What to Overload (and What Not To)

Overload operators only when they create a natural, intuitive interface. Adding two Vector objects with + is intuitive. Adding two Employee objects with + is confusing. The Pythonic rule is: if you need to explain what the operator means for your type, it probably shouldn't be overloaded. Always return NotImplemented (not raise TypeError) when you can't handle the other operand's type — this allows the reflected method to be tried.

💡 Key Dunder Map: __add__/__radd__ for +, __sub__/__rsub__ for -, __mul__/__rmul__ for *, __truediv__ for /, __floordiv__ for //, __mod__ for %, __pow__ for **, __neg__ for unary -, __eq__ for ==, __lt__ for <, __and__ for &, __or__ for |, __lshift__ for <<.

EXAMPLE 10.1 — Full Operator Overloading: Fraction Class 
import math
import functools

@functools.total_ordering
class Fraction:
    """Exact rational number with full operator support."""

    def __init__(self, numerator: int, denominator: int = 1) -> None:
        if denominator == 0:
            raise ZeroDivisionError("Denominator cannot be zero.")
        g = math.gcd(abs(numerator), abs(denominator))
        sign = -1 if denominator < 0 else 1
        self.num = sign * numerator // g
        self.den = sign * denominator // g

    def __repr__(self) -> str:
        return f"Fraction({self.num}, {self.den})"

    def __str__(self) -> str:
        return f"{self.num}" if self.den == 1 else f"{self.num}/{self.den}"

    def __add__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return Fraction(self.num * other.den + other.num * self.den,
                       self.den * other.den)

    def __radd__(self, other): return self.__add__(other)

    def __sub__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return Fraction(self.num * other.den - other.num * self.den,
                       self.den * other.den)

    def __mul__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return Fraction(self.num * other.num, self.den * other.den)

    def __rmul__(self, other): return self.__mul__(other)

    def __truediv__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return Fraction(self.num * other.den, self.den * other.num)

    def __neg__(self): return Fraction(-self.num, self.den)
    def __abs__(self): return Fraction(abs(self.num), self.den)
    def __float__(self): return self.num / self.den
    def __eq__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return self.num == other.num and self.den == other.den
    def __lt__(self, other):
        if isinstance(other, int): other = Fraction(other)
        if not isinstance(other, Fraction): return NotImplemented
        return self.num * other.den < other.num * self.den
    def __hash__(self): return hash((self.num, self.den))

a = Fraction(1, 2)
b = Fraction(1, 3)

print(f"{a} + {b} = {a + b}")          # 5/6
print(f"{a} - {b} = {a - b}")          # 1/6
print(f"{a} * {b} = {a * b}")          # 1/6
print(f"{a} / {b} = {a / b}")          # 3/2
print(f"3 * {a}   = {3 * a}")           # 3/2  (radd triggered)
print(f"{a} > {b}: {a > b}")           # True
print(f"float({a}) = {float(a)}")      # 0.5
fracs = [Fraction(3,4), Fraction(1,6), Fraction(2,3)]
print(f"sorted: {sorted(fracs)}")

SAMPLE 2 — XOR Cipher with Bitwise Operators 
def xor_cipher(text: str, key: str) -> str:
    """Encrypt or decrypt text using XOR with a repeating key.

    XOR is its own inverse: encrypt twice → original text.
    """
    key_bytes = [ord(c) for c in key]
    return "".join(
        chr(ord(c) ^ key_bytes[i % len(key_bytes)])
        for i, c in enumerate(text)
    )

def show_binary(text: str, label: str, max_chars: int = 6) -> None:
    """Show the binary representation of each character."""
    print(f"\n  {label}:")

        print(f"    '{ch}' = {ord(ch):08b} ({ord(ch)})")
    if len(text) > max_chars:
        print(f"    ... ({len(text) - max_chars} more chars)")

SAMPLE PROGRAM 3 — Shopping Cart (Operator Overloading)

A shopping cart where individual Item objects can be added with +, compared with < and > by price, and where a Cart supports += to add items, len() to count items, and in to check membership. This example shows how operator overloading creates an expressive, domain-specific API that feels natural and Pythonic.

Concepts demonstrated: __add__, __iadd__, __lt__, __eq__, __len__, __contains__, @functools.total_ordering, augmented assignment.

SAMPLE 3 — Shopping Cart with Operator Overloading 
import functools

@functools.total_ordering
class Item:
    """A product with a name and price; supports addition and comparison."""

    def __init__(self, name: str, price: float, qty: int = 1) -> None:
        self.name  = name
        self.price = round(price, 2)
        self.qty   = qty

    def __repr__(self) -> str:
        return f"Item({self.name!r}, ${self.price:.2f} x{self.qty})"

    def __str__(self) -> str:
        return f"{self.name:<20} ${self.price:>7.2f} x{self.qty}"

    def __add__(self, other: "Item") -> "Item":
        """Combine two Items of the same product (merge quantities)."""
        if not isinstance(other, Item):
            return NotImplemented
        if self.name != other.name:
            raise ValueError(f"Cannot merge '{self.name}' with '{other.name}'")
        return Item(self.name, self.price, self.qty + other.qty)

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Item): return NotImplemented
        return self.name == other.name and self.price == other.price

    def __lt__(self, other: "Item") -> bool:
        if not isinstance(other, Item): return NotImplemented
        return self.total < other.total

    @property
    def total(self) -> float:
        return round(self.price * self.qty, 2)


class Cart:
    """Shopping cart supporting += to add items and len()."""

    def __init__(self) -> None:
        self._items: list[Item] = []

    def __iadd__(self, item: Item) -> "Cart":
        """cart += item"""
        for existing in self._items:
            if existing.name == item.name:
                existing.qty += item.qty
                return self
        self._items.append(item)
        return self

    def __len__(self) -> int:
        return sum(i.qty for i in self._items)

    def __contains__(self, name: str) -> bool:
        return any(i.name == name for i in self._items)

    def total(self) -> float:
        return round(sum(i.total for i in self._items), 2)

    def receipt(self) -> str:
        lines = ["\n  ── Receipt ──────────────────────────"]
        for item in sorted(self._items, reverse=True):
            lines.append(f"  {item}  = ${item.total:.2f}")
        lines.append(f"  {'─'*38}")
        lines.append(f"  {'TOTAL':<28} ${self.total():.2f}")
        lines.append(f"  {'Items in cart:':<28} {len(self)}")
        return "\n".join(lines)


cart = Cart()
cart += Item("Python Textbook",    49.99, 2)
cart += Item("USB-C Hub",          29.95, 1)
cart += Item("Mechanical Keyboard", 79.00, 1)
cart += Item("Python Textbook",    49.99, 1)   # merges into existing

print("'Python Textbook' in cart:", "Python Textbook" in cart)
print("'Webcam' in cart:", "Webcam" in cart)

cheapest = min(cart._items)
priciest = max(cart._items)
print(f"\nCheapest item: {cheapest.name}")
print(f"Priciest item: {priciest.name}")
print(cart.receipt())

12.0 COMMON PITFALLS AND HOW TO AVOID THEM

MistakeProblem DescriptionCorrect Approach
Using is for value equalityWriting if x is 5: or if s is "hello": relies on CPython's integer/string interning, which is an implementation detail and not guaranteed for all values. It will silently fail for large integers.Always use == for value comparison. Reserve is for singleton comparisons: if x is None:, if flag is True:.
Bitwise vs Logical in conditionsif x > 0 & y > 0: is parsed as if x > (0 & y) > 0: because & has higher precedence than >. This produces the wrong result silently.Use and for logical conditions: if x > 0 and y > 0:. Use & only for bitwise integer operations, always wrapping comparisons in parentheses: (x > 0) & (y > 0).
Forgetting / returns floatIn Python 3, 10 / 2 returns 5.0, not 5. Code expecting an integer index (e.g., arr[10/2]) raises TypeError.Use // for integer division: arr[10 // 2]. Use int(a / b) only when you want truncation toward zero for negative numbers.
Mutable augmented assignment aliasinga = b = []; a += [1] modifies the single shared list via __iadd__, so b also becomes [1] — surprising if you expected += to create a new list.Initialise independently: a, b = [], []. Understand that += on lists calls __iadd__ (in-place), while a = a + [1] creates a new list and rebinds a.
Returning TypeError instead of NotImplementedWhen overloading operators, raising TypeError in __add__ prevents Python from trying the reflected __radd__ on the other operand, breaking interoperability with other types.Return the singleton NotImplemented (not raise an exception) when your method cannot handle the other operand's type. Python will then try the reflected method.
Chaining None comparisonsWriting if None < x < 100: raises TypeError in Python 3 because None is not orderable against integers. Python 3 removed arbitrary ordering of unlike types.Always check for None first with is: if x is not None and 0 < x < 100:. Short-circuit evaluation ensures the range check only runs when x is not None.

13.0 KNOWLEDGE QUIZ — TEST YOUR UNDERSTANDING

1. What does -7 // 2 evaluate to, and why?
Click to reveal answer
-7 // 2 evaluates to -4. Python's floor division rounds toward negative infinity, not toward zero. The exact result is -3.5; the floor (largest integer ≤ -3.5) is -4. The modulo identity a == (a // b) * b + (a % b) holds: -7 == (-4) * 2 + 1, so -7 % 2 == 1.
2. Explain the difference between and and & in Python.
Click to reveal answer
and is a logical operator that short-circuits: it returns the first falsy operand or the last operand if all are truthy — and crucially, the right side is not evaluated if the left is falsy. & is a bitwise operator: it always evaluates both operands fully and returns an integer (the bitwise AND of both values). Use and for boolean logic, & for bit manipulation. Due to precedence differences, mixing them in comparisons requires explicit parentheses.
3. What is returned by NotImplemented in an overloaded operator, and why does it matter?
Click to reveal answer
Returning the built-in constant NotImplemented (not raising TypeError) signals to Python that the method cannot handle the given operand type. Python then tries the reflected version on the other operand — for example, if a.__add__(b) returns NotImplemented, Python tries b.__radd__(a). If that also returns NotImplemented, Python raises TypeError. This two-step mechanism enables cooperative interoperability between independently written classes.
4. What does the walrus operator (:=) do and when should you use it?
Click to reveal answer
The walrus operator := is an assignment expression: it assigns a value to a variable and simultaneously returns that value, allowing assignment inside expressions. Use it when: (1) a while condition depends on a value you want to use in the loop body (while chunk := f.read(4096):), (2) a list comprehension needs to filter and transform without computing the same expensive expression twice, or (3) an if condition produces a value needed inside the if block. Avoid it when it makes code less readable — it should always reduce redundancy, not just save a line.
5. What is the result of 2 ** 3 ** 2, and why?
Click to reveal answer
2 ** 3 ** 2 evaluates to 512. The exponentiation operator ** is right-associative, meaning it is parsed as 2 ** (3 ** 2). First 3 ** 2 = 9, then 2 ** 9 = 512. If it were left-associative like most operators, it would be (2 ** 3) ** 2 = 8 ** 2 = 64. Write explicit parentheses whenever the intended grouping matters.
