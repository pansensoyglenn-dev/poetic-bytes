🦭 Walrus Operator — := (Assignment Expressions)

python 
def walrus_demo():
    print("--- While loop with walrus ---")
    lines = ["first", "second", "third"]
    i = 0
    while (line := lines[i]) != "third":
        print(f"Processing: {line}")
        i += 1

    print("\n--- If statement with walrus ---")
    data = [1, 2, 3, 4, 5]
    if (n := len(data)) > 3:
        print(f"List has {n} elements")

    print("\n--- List comprehension with walrus ---")
    numbers = [10, 5, 8, 3, 12, 7]
    doubled = [value * 2 for x in numbers if (value := x) > 5]
    print(f"Original: {numbers}")
    print(f"Doubled (only values > 5): {doubled}")

📦 Operator Overloading — Customizing Operators for Your Classes

Python allows you to define how operators behave for your own classes using special methods.

python 
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)

    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

def overloading_demo():
    v1 = Vector(3, 4)
    v2 = Vector(1, 2)

    print(f"v1 = {v1}")
    print(f"v2 = {v2}")
    print(f"v1 + v2 = {v1 + v2}")
    print(f"v1 - v2 = {v1 - v2}")
    print(f"v1 * 2 = {v1 * 2}")

from functools import total_ordering

@total_ordering
class Book:
    def __init__(self, title, pages):
        self.title = title
        self.pages = pages

    def __eq__(self, other):
        return self.title == other.title

    def __lt__(self, other):
        return self.pages < other.pages

    def __repr__(self):
        return f'Book("{self.title}", {self.pages})'

def comparison_overloading_demo():
    book1 = Book("Python 101", 350)
    book2 = Book("Advanced Python", 520)
    book3 = Book("Python 101", 350)

    print(f"book1 = {book1}")
    print(f"book2 = {book2}")
    print(f"book3 = {book3}")

    print(f"book1 == book3: {book1 == book3}")
    print(f"book1 == book2: {book1 == book2}")
    print(f"book1 < book2: {book1 < book2}")
    print(f"book1 <= book2: {book1 <= book2}")
    print(f"book1 > book2: {book1 > book2}")

  • Step 3: Calculate subtotal using *.
  • Step 4: Determine discount rate based on quantity and customer type.
  • Step 5: Calculate discount amount using *.
  • Step 6: Calculate final price using -.
  • Step 7: Use logical operators to check eligibility.
  • Step 8: Call the function to execute the code.
python 
def discount_calculator():
    price = 29.99
    quantity = 5
    is_member = True

    subtotal = price * quantity
    print(f"Price: ${price:.2f}, Quantity: {quantity}, Subtotal: ${subtotal:.2f}")

    if quantity >= 5 and is_member:
        discount = 0.20
    elif quantity >= 5:
        discount = 0.10
    elif is_member:
        discount = 0.05
    else:
        discount = 0.00

    discount_amount = subtotal * discount
    final_price = subtotal - discount_amount

    print(f"Discount rate: {discount * 100:.0f}%")
    print(f"Discount amount: ${discount_amount:.2f}")
    print(f"Final price: ${final_price:.2f}")

🚀 12.3 Permission Check with Bitwise Operators

Use bitwise operators to manage permissions.

Step-by-step explanation:

  • Step 1: Define permission constants: READ = 1, WRITE = 2, EXECUTE = 4.
  • Step 2: Set user permissions using bitwise OR |.
  • Step 3: Check for permission using bitwise AND &.
  • Step 4: Grant a new permission using |=.
  • Step 5: Revoke a permission using &= and ~.
  • Step 6: Call the function to execute the code.
python 
def permission_demo():
    READ = 1
    WRITE = 2
    EXECUTE = 4

    user_perms = READ | WRITE
    print(f"User permissions: {user_perms} (binary: {bin(user_perms)})")

    has_read = user_perms & READ
    has_write = user_perms & WRITE
    has_execute = user_perms & EXECUTE

    print(f"Can read? {bool(has_read)}")
    print(f"Can write? {bool(has_write)}")
    print(f"Can execute? {bool(has_execute)}")

    user_perms |= EXECUTE
    print(f"After granting execute: {user_perms} (binary: {bin(user_perms)})")

    user_perms &= ~WRITE
    print(f"After revoking write: {user_perms} (binary: {bin(user_perms)})")

permission_demo()
✨ Bitwise permissions are efficient and compact. This pattern is used in Unix file permissions, database roles, and feature flags.

❓ Frequently Asked Questions — Operators

Common questions about Python operators, answered.

What is the difference between == and is?

+

== compares the values of two objects. is compares the object identity (whether they refer to the same object in memory). Use == for value comparison, is for None checks.

What does the // operator do?

+

// is the floor division operator. It divides two numbers and rounds down to the nearest integer. For positive numbers, it's the same as truncation. For negative numbers, it rounds toward negative infinity.

What is the walrus operator :=?

+

Introduced in Python 3.8, the walrus operator allows assignment within expressions. It's useful in while loops, if statements, and comprehensions to avoid computing the same value twice.

What is operator overloading?

+

Operator overloading allows you to define custom behavior for operators (like +, -, *) for your own classes. This is done by implementing special methods like __add__, __sub__, etc.

What is the precedence of and vs or?

+

and has higher precedence than or. In an expression like a or b and c, Python evaluates b and c first. Use parentheses for clarity.

How do I swap two variables in Python?

+

Use multiple assignment: x, y = y, x. This is the Pythonic way and works with any types, no temporary variable needed.

What is the difference between in and ==?

+

in checks membership — whether a value exists in a sequence or collection. == checks if two values are equal. They serve different purposes and are not interchangeable.

Can I overload operators for built-in types?

+

No, you cannot overload operators for built-in types. Operator overloading only works with user-defined classes. You can, however, use the numbers module to create numeric types that interoperate with built-in numbers.
