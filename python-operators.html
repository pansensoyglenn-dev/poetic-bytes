➕ Arithmetic Operators

Perform mathematical operations on numbers.

📝 1.1 Arithmetic Operators Overview

Python supports all standard arithmetic operators: +, -, *, /, //, %, **.

Step-by-step explanation:

  • Step 1: Addition: 10 + 3 adds two numbers.
  • Step 2: Subtraction: 10 - 3 subtracts the second from the first.
  • Step 3: Multiplication: 10 * 3 multiplies two numbers.
  • Step 4: Division: 10 / 3 returns a float.
  • Step 5: Floor division: 10 // 3 returns the integer quotient.
  • Step 6: Modulo: 10 % 3 returns the remainder.
  • Step 7: Exponentiation: 10 ** 3 raises to a power.
python 
a, b = 10, 3

print(f"a + b = {a + b}")
print(f"a - b = {a - b}")
print(f"a * b = {a * b}")
print(f"a / b = {a / b}")
print(f"a // b = {a // b}")
print(f"a % b = {a % b}")
print(f"a ** b = {a ** b}")
💡 Division / always returns a float. Use // for integer division.

📝 1.2 Modulo Operator (%)

The modulo operator returns the remainder of a division. It's useful for checking divisibility and cycling through ranges.

Step-by-step explanation:

  • Step 1: Check if a number is even: num % 2 == 0.
  • Step 2: Get the last digit: num % 10.
  • Step 3: Cycle through a range: index % 3 gives 0, 1, 2, 0, 1, 2...
python 
num = 17

print(f"num % 2 = {num % 2}")
print(f"{num} is even: {num % 2 == 0}")

print(f"num % 10 = {num % 10}")

for i in range(6):
    print(f"{i} % 3 = {i % 3}")

⚖️ Comparison Operators

Compare values and return True or False.

📝 2.1 Comparison Operators Overview

Python supports: ==, !=, >, <, >=, <=.

Step-by-step explanation:

  • Step 1: Equal to: x == y checks if values are equal.
  • Step 2: Not equal: x != y checks if values are different.
  • Step 3: Greater than: x > y.
  • Step 4: Less than: x < y.
  • Step 5: Greater than or equal to: x >= y.
  • Step 6: Less than or equal to: x <= y.
python 
x, y = 5, 10

print(f"x == y: {x == y}")
print(f"x != y: {x != y}")
print(f"x > y: {x > y}")
print(f"x < y: {x < y}")
print(f"x >= y: {x >= y}")
print(f"x <= y: {x <= y}")
💡 Comparison operators are used in if statements and loops to control program flow.

📝 2.2 Chaining Comparisons

Python allows chaining comparisons: a < b < c is equivalent to a < b and b < c.

Step-by-step explanation:

  • Step 1: Chain comparisons for cleaner code.
  • Step 2: Check if a value is within a range.
python 
x = 5

print(f"1 < x < 10: {1 < x < 10}")
print(f"0 <= x <= 10: {0 <= x <= 10}")

print(f"(1 < x) and (x < 10): {(1 < x) and (x < 10)}")

🔀 Logical Operators

Combine boolean values: and, or, not.

📝 3.1 Logical Operators Overview

Logical operators are used to combine conditional statements.

Step-by-step explanation:

  • Step 1: and returns True if both operands are true.
  • Step 2: or returns True if at least one operand is true.
  • Step 3: not returns the opposite boolean value.
python 
a, b = True, False

print(f"a and b: {a and b}")
print(f"a or b: {a or b}")
print(f"not a: {not a}")

x, y = 5, 10
print(f"x > 3 and y < 20: {x > 3 and y < 20}")
print(f"x > 10 or y > 5: {x > 10 or y > 5}")
💡 Logical operators short-circuit: and stops at the first False, or stops at the first True.

✏️ Assignment Operators

Assign values to variables, often combined with arithmetic.

📝 4.1 Assignment Operators Overview

Python supports: =, +=, -=, *=, /=, %=, //=, **=.

Step-by-step explanation:

  • Step 1: x = 10 assigns the value 10.
  • Step 2: x += 5 is equivalent to x = x + 5.
  • Step 3: x *= 2 is equivalent to x = x * 2.
python 
x = 10
print(f"x = {x}")

x += 3
print(f"x += 3 → x = {x}")

x -= 5
print(f"x -= 5 → x = {x}")

x *= 2
print(f"x *= 2 → x = {x}")

x /= 4
print(f"x /= 4 → x = {x}")

x //= 2
print(f"x //= 2 → x = {x}")

x **= 2
print(f"x **= 2 → x = {x}")

💻 Bitwise Operators

Work with numbers at the binary level: &, |, ^, ~, <<, >>.

📝 5.1 Bitwise Operators Overview

Bitwise operators are used for low-level manipulation of integer bits.

Step-by-step explanation:

  • Step 1: & (AND): 1 if both bits are 1.
  • Step 2: | (OR): 1 if at least one bit is 1.
  • Step 3: ^ (XOR): 1 if bits are different.
  • Step 4: ~ (NOT): Inverts all bits.
  • Step 5: << (Left shift): Shifts bits left.
  • Step 6: >> (Right shift): Shifts bits right.
python 
a, b = 6, 3

print(f"a = {a} (binary: {bin(a)})")
print(f"b = {b} (binary: {bin(b)})")
print(f"a & b = {a & b} ({bin(a & b)})")
print(f"a | b = {a | b} ({bin(a | b)})")
print(f"a ^ b = {a ^ b} ({bin(a ^ b)})")
print(f"~a = {~a} ({bin(~a)})")
print(f"a << 1 = {a << 1} ({bin(a << 1)})")
print(f"a >> 1 = {a >> 1} ({bin(a >> 1)})")

📝 5.2 Practical Use: Flags

Bitwise operators are commonly used to manage flags and permissions.

Step-by-step explanation:

  • Step 1: Define flags: READ = 1, WRITE = 2, EXEC = 4.
  • Step 2: Combine flags using |: READ | WRITE.
  • Step 3: Check if a flag is set using &.
python 
READ = 1
WRITE = 2
EXEC = 4

permissions = READ | WRITE
print(f"permissions: {permissions} (binary: {bin(permissions)})")

if permissions & READ:
    print("Read permission granted")
if permissions & WRITE:
    print("Write permission granted")
if permissions & EXEC:
    print("Execute permission granted")
else:
    print("Execute permission NOT granted")

🔍 Membership Operators

Check if a value exists in a sequence: in, not in.

📝 6.1 Membership Operators Overview

Membership operators are used to test if a value is present in a list, tuple, string, or other collection.

Step-by-step explanation:

  • Step 1: in returns True if the value is found.
  • Step 2: not in returns True if the value is not found.
python 
fruits = ["apple", "banana", "cherry"]

print(f"'apple' in fruits: {'apple' in fruits}")
print(f"'grape' in fruits: {'grape' in fruits}")
print(f"'grape' not in fruits: {'grape' not in fruits}")

text = "Hello, world!"
print(f"'world' in text: {'world' in text}")
print(f"'Python' in text: {'Python' in text}")

🆔 Identity Operators

Compare object identity: is, is not.

📝 7.1 Identity Operators Overview

Identity operators check if two variables refer to the same object in memory.

Step-by-step explanation:

  • Step 1: is returns True if both variables point to the same object.
  • Step 2: is not returns True if they point to different objects.
  • Step 3: == compares values, is compares identity.
python 
a = [1, 2, 3]
b = a
c = [1, 2, 3]

print(f"a = {a}, id(a): {id(a)}")
print(f"b = {b}, id(b): {id(b)}")
print(f"c = {c}, id(c): {id(c)}")

print(f"a is b: {a is b}")
print(f"a is c: {a is c}")
print(f"a == c: {a == c}")

x = 10
y = 10
print(f"x is y (integers): {x is y}")
⚠️ Use is for None, booleans, and checking object identity. Use == for value comparisons.

📊 Operator Precedence

Python follows specific rules to determine the order in which operators are evaluated.

📝 8.1 Precedence Rules

Use parentheses to make your code explicit and avoid confusion.

Step-by-step explanation:

  • Step 1: Exponentiation has the highest precedence among arithmetic operators.
  • Step 2: Multiplication and division are evaluated before addition and subtraction.
  • Step 3: Comparison operators have lower precedence than arithmetic.
  • Step 4: Logical operators have lower precedence than comparisons.
  • Step 5: Use parentheses to override default precedence.
python 
print("Without parentheses:")
print(f"10 + 5 * 2 = {10 + 5 * 2}")
print(f"(10 + 5) * 2 = {(10 + 5) * 2}")

print("\nComparison and logical:")
x, y, z = 5, 10, 15
print(f"x < y and y < z: {x < y and y < z}")
print(f"x < y and y > z: {x < y and y > z}")
print(f"x < y or y > z: {x < y or y > z}")
💡 When in doubt, use parentheses! They make your intention clear and improve readability.

❓ Frequently Asked Questions — Python Operators

Common questions about operators in Python, answered.

What is the difference between == and is?

+

== compares the values of two objects, while is checks if they refer to the same object in memory. Use == for value equality, and is for identity (e.g., None).

What does the % operator do?

+

The modulo operator % returns the remainder of a division. For example, 10 % 3 returns 1. It's useful for checking divisibility, cycling through ranges, and extracting digits.

What are bitwise operators used for?

+

Bitwise operators work at the binary level. They are used for low-level programming, flag manipulation, permissions, packing data, and optimization.

What is operator precedence?

+

Operator precedence determines the order in which operators are evaluated in an expression. For example, multiplication is evaluated before addition. Use parentheses to override the default precedence.

What is short-circuit evaluation?

+

In logical expressions, Python stops evaluating as soon as the result is known. For and, if the first operand is False, it doesn't evaluate the second. For or, if the first operand is True, it doesn't evaluate the second.

Can I use operators with strings?

+

Yes! + concatenates strings, * repeats strings, and in checks for substring membership. Comparison operators work lexicographically.

What is the difference between / and //?

+

/ performs floating-point division and always returns a float. // performs floor division and returns the integer quotient, discarding the remainder.
