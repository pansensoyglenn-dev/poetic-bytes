⌨️ The input() Function — Your Gateway to User Interaction

Every interactive Python program starts here. input() reads a line from the user and returns it as a string.

📝 1.1 Basic input() — Prompt and Capture

The simplest form: call input() with an optional prompt string. The program pauses until the user types something and presses Enter. The returned value is always a string.

Step-by-step explanation:

  • Step 1: Define the function basic_input_demo() — this groups our code for reuse.
  • Step 2: Call input("What is your name? ") — this displays the prompt and waits for the user to type something and press Enter.
  • Step 3: The value typed by the user is stored in the variable name as a string.
  • Step 4: Use an f-string with print() to display a personalized greeting: f"Hello, {name}!".
  • Step 5: Call input("How old are you? ") to capture the user's age — again stored as a string.
  • Step 6: Display the age back to the user using an f-string.
  • Step 7: Use type(age) to show that input() always returns a string — even for numbers.
  • Step 8: Call the function basic_input_demo() to execute the code.
python 
def basic_input_demo():
    name = input("What is your name? ")
    print(f"Hello, {name}! Nice to meet you.")

    age = input("How old are you? ")
    print(f"You are {age} years old.")
    print(f"Type of age: {type(age)}")

📝 1.3 Empty Input — Handling ''

If the user just presses Enter without typing anything, input() returns an empty string ''. Always plan for this — many bugs come from assuming input is non-empty.

def handle_empty():

    if response == "":
        print("You pressed Enter without typing anything!")
    else:
        print(f"You typed: {response}")

    print(f"Length: {len(response)} characters")

✅ Input Validation — Trust Nothing, Verify Everything

Users make mistakes. Malicious users exploit input. Validate early and often.

🛡️ 3.1 Basic Validation with try/except

Wrap conversion in a try/except block to catch invalid input gracefully.

Step-by-step explanation:

  • Step 1: Define a function validate_integer() to handle validation.
  • Step 2: Inside a try block, call input() and convert with int().
  • Step 3: If conversion succeeds, print a success message and return the value.
  • Step 4: In the except ValueError block, print an error message and return None.
  • Step 5: Call the function and check if the result is not None before using it.
  • Step 6: If valid, double the number and display it.
python 
def validate_integer():
    try:
        value = int(input("Enter a whole number: "))
        print(f"✅ You entered: {value}")
        return value
    except ValueError:
        print("❌ That's not a valid whole number!")
        return None

result = validate_integer()
if result is not None:
    print(f"Double it: {result * 2}")

🛡️ 3.2 Loop Until Valid — Robust User Experience

Keep asking until the user provides valid input. This pattern is essential for interactive CLI tools.

Step-by-step explanation:

  • Step 1: Define get_positive_number(prompt) that loops until valid input is received.
  • Step 2: Inside an infinite while True loop, attempt to convert the input to float.
  • Step 3: If conversion succeeds and the value is positive, return the value (exits the function).
  • Step 4: If the value is not positive, show an error message and continue the loop.
  • Step 5: If conversion fails with ValueError, show an error and continue.
  • Step 6: In main(), call the function with a prompt and calculate the area of a circle.
  • Step 7: Display the calculated area with 2 decimal places.
python 
def get_positive_number(prompt: str) -> float:
    while True:
        try:
            value = float(input(prompt))
            if value > 0:
                return value
            print("❌ Please enter a positive number.")
        except ValueError:
            print("❌ Invalid input. Please enter a number.")

def main():
    radius = get_positive_number("Enter radius: ")
    area = 3.14159 * radius ** 2
    print(f"✅ Area of circle: {area:.2f}")

main()
💡 The while True + return pattern is the cleanest way to loop until valid input is received. No messy flag variables.

🛡️ 3.3 Range Validation — Check Bounds

Sometimes you need a number within a specific range (e.g., 1–100, 0–10). Add bounds checking after conversion.

Step-by-step explanation:

  • Step 1: Define get_score() that validates a score between 0 and 100.
  • Step 2: Inside the loop, attempt to convert input to int.
  • Step 3: Check if the score is between 0 and 100 inclusive — if so, return it.
  • Step 4: If out of range, show a message and continue the loop.
  • Step 5: Define grade_score() that returns a letter grade based on the score.
  • Step 6: Call get_score() to get a valid score, then pass it to grade_score().
  • Step 7: Display the score and its corresponding grade.
python 
def get_score() -> int:
    while True:
        try:
            score = int(input("Enter a score (0–100): "))
            if 0 <= score <= 100:
                return score
            print("❌ Score must be between 0 and 100.")
        except ValueError:
            print("❌ Please enter a whole number.")

def grade_score(score: int) -> str:
    if score >= 90: return "A"
    if score >= 80: return "B"
    if score >= 70: return "C"
    if score >= 60: return "D"
    return "F"

score = get_score()
print(f"✅ Score {score} → Grade {grade_score(score)}")

🛡️ 3.4 String Validation — Check Format and Content

Validate strings for length, allowed characters, or pattern matching with regex.

Step-by-step explanation:

  • Step 1: Import the re module for regular expression matching.
  • Step 2: Define validate_name() that loops until a valid name is entered.
  • Step 3: Use len() to check that the name is between 2 and 30 characters.
  • Step 4: Use re.match() with a pattern to ensure only letters, spaces, hyphens, and apostrophes.
  • Step 5: Define validate_email() that loops until a valid email is entered.
  • Step 6: Use a regex pattern to check the email format (basic validation).
  • Step 7: Call both validation functions and display a welcome message.
python 
import re

def validate_name() -> str:
    while True:
        name = input("Enter your name (2–30 letters): ").strip()

        if len(name) < 2 or len(name) > 30:
            print("❌ Name must be 2–30 characters.")
            continue

        if not re.match(r"^[A-Za-z\s\-'']+$", name):
            print("❌ Name contains invalid characters.")
            continue

        return name

def validate_email() -> str:
    while True:
        email = input("Enter your email: ").strip()
        if re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", email):
            return email
        print("❌ Invalid email format.")

name = validate_name()
email = validate_email()
print(f"✅ Welcome, {name}! Email: {email}")
✨ Use str.strip() to remove accidental leading/trailing spaces before validation.

📦 Multiple Inputs — Collecting Several Values at Once

Sometimes you need to collect multiple values in a single line, or ask for a list of items.

📦 4.1 split() — Parse Space-Separated Input

Use str.split() to break a single input line into multiple pieces. Then convert each piece individually.

Step-by-step explanation:

  • Step 1: Call input() to capture a line with three numbers separated by spaces.
  • Step 2: Use split() to break the string into a list of parts.
  • Step 3: Check that exactly three parts were entered — if not, show an error and return.

  • Step 6: Display the results with 2 decimal places.
python 
def parse_multiple_numbers():
    raw = input("Enter three numbers separated by spaces: ")
    parts = raw.split()
    print(f"Parts: {parts}")

    if len(parts) != 3:
        print("❌ Please enter exactly three numbers.")
        return

    try:
        a, b, c = float(parts[0]), float(parts[1]), float(parts[2])
        print(f"✅ Sum: {a + b + c:.2f}")
        print(f"✅ Average: {(a + b + c) / 3:.2f}")
    except ValueError:
        print("❌ All inputs must be numbers.")

📦 4.2 Collecting an Arbitrary Number of Items

Use a loop to collect items until the user signals they are done (e.g., with a blank line or a sentinel value).

Step-by-step explanation:

  • Step 1: Define collect_items() that returns a list of strings.
  • Step 2: Print instructions for the user: enter items one per line, press Enter twice to finish.
  • Step 3: Inside an infinite loop, call input() to get each line.
  • Step 4: If the line is empty (""), break out of the loop.
  • Step 5: Otherwise, append the line to the items list.
  • Step 6: In main(), display the collected items with numbers.
  • Step 7: Call the function to run the interactive collection.
python 
def collect_items() -> list:
    items = []
    print("Enter items one per line. Press Enter twice to finish.")

    while True:
        line = input("> ")
        if line == "":
            break
        items.append(line)

    return items

def main():
    items = collect_items()
    print(f"\nYou entered {len(items)} items:")
    for i, item in enumerate(items, 1):
        print(f"  {i}. {item}")

main()
💡 The double-Enter pattern is a common UX pattern for CLI tools that collect lists of items. It's clean, intuitive, and works everywhere.

📂 5.2 Simple Argument Parser — Manual Parsing

⚙️ argparse — Professional Command-Line Parsing

For robust, production-ready CLI tools, argparse is the standard library module that handles flags, options, subcommands, and help text automatically.

⚙️ 6.1 Basic argparse Setup

Define arguments, flags, and options with type checking and automatic help generation.

Step-by-step explanation:

  • Step 1: Import the argparse module.
  • Step 2: Create an ArgumentParser instance with a description.
  • Step 3: Add a positional argument: "name" — this is required.
  • Step 4: Add an optional argument: "--age" with type int and default 18.
  • Step 5: Add a flag: "--verbose" with action="store_true".
  • Step 6: Parse arguments with parser.parse_args(), passing an explicit list of fake CLI arguments rather than reading real argv.
  • Step 7: Access values via args.name, args.age, args.verbose.
  • Step 8: Display the parsed values and check if the user is an adult.
python 
import argparse

def build_parser():
    parser = argparse.ArgumentParser(
        description="A simple CLI tool with argparse"
    )

    parser.add_argument("name", help="Your name")
    parser.add_argument("--age", type=int, default=18, help="Your age (default: 18)")
    parser.add_argument("--verbose", action="store_true", help="Enable verbose output")

    return parser

def main():
    parser = build_parser()
    args = parser.parse_args(["Alice", "--age", "25", "--verbose"])

    if args.verbose:
        print("🔍 Verbose mode enabled")

    print(f"Hello, {args.name}!")
    print(f"Age: {args.age}")

    if args.age >= 18:
        print("✅ You are an adult.")
    else:
        print("🔞 You are a minor.")

main()
💡 argparse automatically generates --help output, validates types, and provides error messages for invalid input. Note we pass an explicit list to parse_args() instead of reading real sys.argv — this keeps the demo self-contained and avoids SystemExit surprises.

⚙️ 6.2 Positional vs Optional Arguments

Positional arguments are required and ordered. Optional arguments are prefixed with -- and can appear anywhere.

Step-by-step explanation:

  • Step 1: Create a parser with a description for file operations.
  • Step 2: Add "source" as a required positional argument.
  • Step 3: Add "dest" as an optional positional with nargs="?" and a default value.
  • Step 4: Add "-v" / "--verbose" as a flag (store_true).
  • Step 5: Add "-f" / "--force" as a flag (store_true).
  • Step 6: Parse arguments with an explicit fake-argument list.
  • Step 7: Display all parsed values to show how each argument is handled.
python 
import argparse

def create_parser():
    parser = argparse.ArgumentParser(description="File operations")

    parser.add_argument("source", help="Source file path")
    parser.add_argument("dest", nargs="?", default=".", help="Destination path (default: current dir)")

    parser.add_argument("-v", "--verbose", action="store_true", help="Verbose output")
    parser.add_argument("-f", "--force", action="store_true", help="Overwrite without asking")

    return parser

def main():
    parser = create_parser()
    args = parser.parse_args(["report.txt", "/backup/", "--verbose"])

    print(f"Source: {args.source}")
    print(f"Destination: {args.dest}")
    print(f"Verbose: {args.verbose}")
    print(f"Force: {args.force}")

main()

🔒 Secure Input — getpass for Passwords

Never echo passwords to the screen. Use getpass for secure, hidden input.

🔒 7.1 getpass.getpass() — Hidden Input

This function works like input() but doesn't echo characters to the console. Perfect for passwords, API keys, and sensitive data.

Step-by-step explanation:

  • Step 1: Import the getpass module.
  • Step 2: Use input() for the username (visible input).
  • Step 3: Use getpass.getpass() for the password (hidden input — in this browser sandbox it's mocked with a JS prompt dialog).
  • Step 4: Display the username and the length of the password (not the password itself).
  • Step 5: Check if the credentials match a hardcoded pair and display success or failure.
  • Step 6: Call the function to test secure login.
python 
import getpass

def secure_login():
    username = input("Username: ")
    password = getpass.getpass("Password: ")

    print(f"\nUsername: {username}")
    print(f"Password length: {len(password)} characters (hidden)")

    if username == "admin" and password == "secret123":
        print("✅ Login successful!")
    else:
        print("❌ Invalid credentials.")

secure_login()
⚠️ In this browser sandbox, getpass is mocked using a JavaScript prompt() dialog (it still doesn't echo into the page output). In a local terminal, the real getpass hides typed characters entirely.

🔒 7.2 Password Confirmation Pattern

When creating new passwords, ask twice and compare. This prevents typos and ensures accuracy.

Step-by-step explanation:

  • Step 1: Import getpass for hidden input.
  • Step 2: Ask for the current password using getpass().
  • Step 3: Check if the current password matches the expected value.
  • Step 4: Ask for the new password twice using getpass().
  • Step 5: Compare the two new passwords — if they don't match, show an error.
  • Step 6: Check if the new password is at least 8 characters long.
  • Step 7: If all checks pass, confirm the password change.
python 
import getpass

def change_password():
    print("🔐 Change Password")
    print("-" * 30)

    current = getpass.getpass("Current password: ")
    if current != "secret123":
        print("❌ Incorrect current password.")
        return

    new1 = getpass.getpass("New password: ")
    new2 = getpass.getpass("Confirm new password: ")

    if new1 != new2:
        print("❌ Passwords do not match.")
        return

    if len(new1) < 8:
        print("❌ Password must be at least 8 characters.")
        return

    print("✅ Password changed successfully!")

change_password()

Sometimes input comes from a file, not from a user typing. Learn to read structured data from CSV, JSON, and text files.

📄 8.1 Reading Text Files

Use open() with a context manager to read files safely. Always specify encoding.

def sentinel_loop():

            break

            num = float(raw)

    if count > 0:

🛡️ 10.1 Comprehensive Validation Function

Build a reusable validation function that handles multiple failure modes and provides clear feedback.

  • Step 2: Define safe_int_input() with parameters for prompt, min, max, and retries.
  • Step 3: Loop for the specified number of retries.
  • Step 4: Inside each attempt, try to convert the input to int.
  • Step 5: Check if the value is within the specified min and max bounds.
  • Step 6: If valid, return the value immediately.
  • Step 7: If invalid or conversion fails, show an error and continue.
  • Step 8: After all retries, return None.
  • Step 9: Test the function with age and score inputs.
python 
from typing import Optional

def safe_int_input(
    prompt: str,
    min_val: Optional[int] = None,
    max_val: Optional[int] = None,
    retries: int = 3
) -> Optional[int]:
    for attempt in range(1, retries + 1):
        try:
            raw = input(f"{prompt} ")
            value = int(raw)

            if min_val is not None and value < min_val:
                print(f"❌ Value must be at least {min_val}.")
                continue

            if max_val is not None and value > max_val:
                print(f"❌ Value must be at most {max_val}.")
                continue

            return value

        except ValueError:
            print(f"❌ Invalid integer. ({retries - attempt} attempts left)")

    print("❌ Too many attempts. Returning None.")
    return None

age = safe_int_input("Enter your age:", min_val=1, max_val=120)
if age is not None:
    print(f"✅ Age: {age}")

score = safe_int_input("Enter a score (0-100):", min_val=0, max_val=100)
if score is not None:
    print(f"✅ Score: {score}")

Collect structured data from the user with validation and confirmation.

import re

def collect_person_data() -> dict:
    print("\n📝 Enter person details:")

    while True:
        name = input("Full name: ").strip()
        if len(name) >= 2 and re.match(r"^[A-Za-z\s\-'']+$", name):
            break
        print("❌ Please enter a valid name (2+ letters, no numbers).")

    while True:
        email = input("Email: ").strip()
        if re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", email):
            break
        print("❌ Invalid email format.")

    while True:
        age_str = input("Age: ")
        try:
            age = int(age_str)
            if 1 <= age <= 150:
                break
            print("❌ Age must be between 1 and 150.")
        except ValueError:
            print("❌ Please enter a valid number.")

    role = input("Role (developer, designer, manager): ").strip().lower()
    if role not in ("developer", "designer", "manager"):
        role = "other"
        print(f"Role set to {role}")

    return {"name": name, "email": email, "age": age, "role": role}

import argparse

def build_subcommand_parser():
    parser = argparse.ArgumentParser(description="Task manager CLI")
    subparsers = parser.add_subparsers(dest="command", required=True)

    add_parser = subparsers.add_parser("add", help="Add a task")
    add_parser.add_argument("title", help="Task title")
    add_parser.add_argument("--priority", choices=["low", "medium", "high"], default="medium")

    list_parser = subparsers.add_parser("list", help="List tasks")
    list_parser.add_argument("--filter", choices=["all", "active", "done"], default="all")

    done_parser = subparsers.add_parser("done", help="Mark task as done")
    done_parser.add_argument("task_id", type=int, help="Task ID")

    return parser

def main():
    parser = build_subcommand_parser()
    args = parser.parse_args(["add", "Buy milk", "--priority", "high"])

    if args.command == "add":
        print(f"✅ Added task: '{args.title}' (priority: {args.priority})")
    elif args.command == "list":
        print(f"📋 Listing tasks (filter: {args.filter})")
    elif args.command == "done":
        print(f"✅ Task {args.task_id} marked as done")

❓ Frequently Asked Questions — Input Handling

Common questions about Python input, answered.

Why does input() always return a string?

+

Python is strongly typed and does not auto-convert. This design makes the program's behaviour predictable and explicit. You — the programmer — decide what type the input should be and convert it accordingly.

How do I get a number without ValueError?

+

Wrap int() or float() in a try/except block. For user-facing programs, loop until valid input is received. Never assume input will be correct.

What is the difference between sys.argv and argparse?

+

sys.argv is a raw list of arguments. argparse is a full-featured parser that handles flags, options, type conversion, help generation, and error messages. Use argparse for any non-trivial CLI tool.

How do I read input that has spaces?

+

input() reads the entire line, including spaces. For multiple values on one line, use split() and then process each piece. For multi-line input, collect in a loop until a sentinel value is entered.

Is getpass safe to use in production?

+

getpass is safe for hiding input from shoulder surfers, but it does not encrypt the input. Use it for passwords in local scripts. For production applications, use environment variables or dedicated secret management tools.

How do I handle Unicode input?

+

Python 3 handles Unicode strings by default. input() returns a str that can contain any Unicode character. When reading from files, always specify encoding="utf-8".

What's the best way to validate email addresses?

+

Use a regular expression for basic format validation. For production, consider using a dedicated library like email-validator. Never try to fully validate an email address with regex alone — send a confirmation link to verify.
