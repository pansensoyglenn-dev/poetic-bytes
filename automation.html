📜 History of Automation

From Mechanical Looms to Python Scripts — A Brief History

The desire to automate has driven human innovation for centuries. Here's a timeline that shows how we arrived at modern Python automation.

1801 — Jacquard Loom

Joseph Marie Jacquard invented a loom that used punched cards to automate the weaving of complex patterns. This was one of the first examples of a programmable machine.

1936 — Turing Machine

Alan Turing's theoretical machine formalised the concept of computation, laying the foundation for all automation.

1950s — Batch Processing

Mainframes introduced batch processing, allowing repetitive jobs to run without human intervention. This was the dawn of computer automation.

1970s — Shell Scripting

Unix shells brought scripting to the masses, enabling users to automate system tasks with simple text files.

1990s — Web Automation

As the web grew, tools like Perl and later Python were used to automate web requests, scraping, and data extraction.

2000s — Python and Selenium

Python's simplicity and vast ecosystem made it the go‑to language for automation. Selenium brought true browser automation to the mainstream.

2020s — AI‑Driven Automation

Today, Python powers everything from RPA (Robotic Process Automation) to AI orchestration, making it the undisputed king of automation.

📂 Working with Files and Directories

Automation often starts with files. Learn to navigate, create, and manipulate the filesystem using Python's os and pathlib modules.

📂 1.1 Listing Files in a Directory

Use os.listdir() or Path.iterdir() to get a list of files and folders.

Step-by-step explanation:

  • Step 1: Import the os module for operating system interfaces.
  • Step 2: Use os.getcwd() to get the current working directory.
  • Step 3: Call os.listdir('.') to list all files and directories in the current folder.
  • Step 4: Loop through the list and print each item.
  • Step 5: Use os.path.isfile() and os.path.isdir() to distinguish files from directories.
python 
import os

def list_files():
    cwd = os.getcwd()
    print(f"Current directory: {cwd}")

    items = os.listdir(".")
    for item in items:
        full_path = os.path.join(".", item)
        if os.path.isfile(full_path):
            print(f"📄 {item} (file)")
        else:
            print(f"📁 {item} (folder)")

list_files()
💡 os.path.join() is platform‑agnostic — it uses the correct path separator for your OS.

📂 1.2 Creating and Removing Directories

Use os.mkdir() and os.rmdir() to manage folders. Always check existence first to avoid errors.

Step-by-step explanation:

  • Step 1: Import os and os.path.
  • Step 2: Define a function manage_dirs().
  • Step 3: Use os.path.exists() to check if a folder exists.
  • Step 4: If not, create it with os.mkdir().
  • Step 5: If it exists, remove it with os.rmdir() (only if empty).
  • Step 6: Print status messages for each action.
python 
import os

def manage_dirs():
    folder = "test_auto"
    if not os.path.exists(folder):
        os.mkdir(folder)
        print(f"✅ Created folder: {folder}")
    else:
        os.rmdir(folder)
        print(f"🗑️ Removed folder: {folder}")

⚠️ os.rmdir() only works on empty folders. Use shutil.rmtree() to delete non‑empty directories.

📂 2.2 Batch File Renaming

Rename multiple files at once using a pattern. Perfect for organizing photos, logs, or documents.

Step-by-step explanation:

  • Step 1: Import os and glob.
  • Step 2: Use glob.glob() to find files matching a pattern.
  • Step 3: Loop through each file and build a new name.
  • Step 4: Use os.rename() to rename the file.
  • Step 5: Print the renaming details.
python 
import os, glob

def batch_rename():
    for i in range(3):
        with open(f"file_{i}.txt", "w") as f:
            f.write(f"Content {i}")

    for filepath in glob.glob("file_*.txt"):
        base = os.path.basename(filepath)
        new_name = f"data_{base.split('_')[1]}"
        new_path = os.path.join(os.path.dirname(filepath), new_name)
        os.rename(filepath, new_path)
        print(f"Renamed {filepath} → {new_path}")

    for f in glob.glob("data_*.txt"):
        os.remove(f)
    print("✅ Cleaned up")

batch_rename()

📊 Automating CSV and Excel Processing

Data often comes in spreadsheets. Learn to read, process, and write CSV and Excel files with Python.

📊 3.1 Reading and Writing CSV

The csv module makes working with CSV files a breeze.

Step-by-step explanation:

  • Step 1: Import csv and io.StringIO (for in‑memory demo).
  • Step 2: Define a sample CSV string.
  • Step 3: Use csv.reader to parse the data.
  • Step 4: Process each row (e.g., calculate average of a column).
  • Step 5: Write a new CSV with the processed data.
python 
import csv
from io import StringIO

def csv_demo():
    sample = """name,age,score
Alice,30,85
Bob,25,90
Charlie,35,78
"""
    reader = csv.DictReader(StringIO(sample))
    total = 0
    count = 0
    for row in reader:
        total += int(row["score"])
        count += 1
    avg = total / count
    print(f"Average score: {avg:.2f}")

    output = StringIO()
    writer = csv.writer(output)
    writer.writerow(["name", "score", "status"])
    for row in csv.DictReader(StringIO(sample)):
        status = "Pass" if int(row["score"]) >= 80 else "Fail"
        writer.writerow([row["name"], row["score"], status])
    print("New CSV content:")
    print(output.getvalue())

csv_demo()
💡 For large files, process row by row instead of loading everything into memory.

Extract data from websites by parsing HTML. We'll use html.parser and simulate a page.

🕷️ 4.1 Parsing HTML and Extracting Data

Use BeautifulSoup to navigate and extract information from HTML documents.

Step-by-step explanation:

  • Step 1: Import BeautifulSoup from bs4 (simulated).
  • Step 2: Define a sample HTML string (a product listing).
  • Step 3: Parse the HTML with BeautifulSoup(html, "html.parser").
  • Step 4: Use .find_all() to locate elements by tag and class.
  • Step 5: Extract text and attributes (e.g., product names and prices).
python 
from bs4 import BeautifulSoup

def scrape_demo():
    html = """<div class="product">
        <h2>Laptop</h2>
        <span class="price">$999.99</span>
    </div>
    <div class="product">
        <h2>Phone</h2>
        <span class="price">$599.00</span>
    </div>"""
    soup = BeautifulSoup(html, "html.parser")
    for product in soup.find_all("div", class_="product"):
        name = product.find("h2").text
        price = product.find("span", class_="price").text
        print(f"Product: {name}, Price: {price}")

scrape_demo()

class SimulatedBrowser:
    def __init__(self):
        self.state = {"page": "login", "logged_in": False}

    def find_element(self, element_id):
        return {"id": element_id, "value": ""}

    def type_text(self, element_id, text):
        self.state[element_id] = text

    def click(self, element_id):
        if element_id == "login_button":

                self.state["logged_in"] = True

def browser_demo():

    browser.type_text("password", "secret")
    browser.click("login_button")
    print(f"Logged in: {browser.state['logged_in']}")
    print(f"Current page: {browser.state['page']}")

💻 Building a CLI Automation Tool with argparse

Create a command‑line utility that performs automation tasks, with flags and arguments.

💻 10.1 A File Cleaner CLI Tool

We'll build a tool that cleans temporary files based on age and pattern, with command‑line options.

Step-by-step explanation:

  • Step 1: Import argparse, os, and glob.
  • Step 2: Define a parser with arguments: --path, --pattern, --days.
  • Step 3: Parse arguments and use them to find and delete files older than N days.
  • Step 4: Add a --dry-run flag to preview deletions.
  • Step 5: Print a summary of actions.
python 
import argparse, os, glob, time

def clean_files(path, pattern, days, dry_run=False):
    now = time.time()
    cutoff = now - days * 86400
    files = glob.glob(os.path.join(path, pattern))
    print(f"Found {len(files)} files matching {pattern} in {path}")
    for f in files:
        if os.path.getmtime(f) < cutoff:
            if dry_run:
                print(f"[DRY RUN] Would delete {f}")
            else:
                os.remove(f)
                print(f"Deleted {f}")

def main():
    parser = argparse.ArgumentParser(description="Clean old files")
    parser.add_argument("--path", default=".", help="Directory to scan")
    parser.add_argument("--pattern", default="*.tmp", help="File pattern")
    parser.add_argument("--days", type=int, default=7, help="Delete files older than N days")
    parser.add_argument("--dry-run", action="store_true", help="Preview without deleting")
    args = parser.parse_args([])
    clean_files(args.path, args.pattern, args.days, args.dry_run)

main()
💡 This pattern forms the basis of many system administration tools. Add logging and error handling for production use.

import os, shutil, datetime, glob

def backup(source_dir, dest_dir, keep=3):
    timestamp = datetime.datetime.now().strftime("%Y%m%d_%H%M%S")
    archive_name = f"backup_{timestamp}"
    print(f"Creating backup {archive_name} from {source_dir}")
    with open(f"{dest_dir}/{archive_name}.zip", "w") as f:
        f.write("Backup content")
    print(f"✅ Backup created: {dest_dir}/{archive_name}.zip")

    backups = sorted(glob.glob(f"{dest_dir}/backup_*.zip"))
    if len(backups) > keep:
        for old in backups[:-keep]:
            os.remove(old)
            print(f"🗑️ Removed old backup: {old}")

def main():
    src = "source_folder"
    dst = "backup_folder"
    os.makedirs(src, exist_ok=True)
    os.makedirs(dst, exist_ok=True)
    with open(f"{src}/dummy.txt", "w") as f:
        f.write("Important data")
    backup(src, dst, keep=2)
    shutil.rmtree(src)
    shutil.rmtree(dst)

main()

❓ Frequently Asked Questions — Automation

Common questions about Python automation, answered.

What is the best Python library for automation?

+

It depends on the task: os/shutil for files, requests for APIs, BeautifulSoup/Selenium for web, schedule for timers, subprocess for system commands. Python's standard library covers many basics, and the ecosystem is huge.

Is automation only for sysadmins?

+

No! Automation benefits developers, data scientists, QA engineers, marketers, and even managers. Any repetitive task can be automated, regardless of your role.

Can I run automation scripts in the cloud?

+

Yes, you can deploy Python scripts to cloud functions (AWS Lambda, Google Cloud Functions) or run them on virtual machines with cron jobs. Automation scales easily in the cloud.

How do I handle errors in automation?

+

Use try/except blocks, log errors, and implement retry logic for transient failures. Always have a fallback or notify someone when a critical automation fails.

Is Selenium the only way to automate browsers?

+

No, there's also Playwright and Puppeteer (Node.js). In Python, Selenium is the most mature, but Playwright has a Python version that's gaining popularity.

How do I schedule a Python script on Windows?

+

Use Task Scheduler. On macOS/Linux, use cron. Alternatively, use a Python scheduler like schedule with an infinite loop, but system schedulers are more reliable for production.
