📂 Working with Paths — os and pathlib

Every automation script starts with navigating the filesystem. Learn the modern way with pathlib.

📂 1.1 pathlib — Object‑Oriented Paths

pathlib provides an object‑oriented interface to filesystem paths. It's cleaner, more readable, and cross‑platform.

Step-by-step explanation:

  • Step 1: Import Path from pathlib.
  • Step 2: Create a Path object pointing to the current directory using Path.cwd().
  • Step 3: Use path.exists(), path.is_dir(), and path.is_file() to check properties.
  • Step 4: List all files in the current directory using path.iterdir().
  • Step 5: Use path / "subfolder" / "file.txt" to build paths — the / operator is overloaded.
  • Step 6: Access the parent directory with .parent, file name with .name, and extension with .suffix.
python 
from pathlib import Path

def pathlib_demo():
    cwd = Path.cwd()
    print(f"Current directory: {cwd}")

    print(f"Exists: {cwd.exists()}")
    print(f"Is directory: {cwd.is_dir()}")

    print("\nContents:")
    for item in cwd.iterdir():
        print(f"  {item.name} ({'dir' if item.is_dir() else 'file'})")

    sub = cwd / "data" / "config.json"
    print(f"\nBuilt path: {sub}")
    print(f"Parent: {sub.parent}")
    print(f"Name: {sub.name}")
    print(f"Suffix: {sub.suffix}")

pathlib_demo()
💡 pathlib handles path separators automatically — your code works on Windows, macOS, and Linux without modification.

📂 1.2 os Module — Traditional Path Operations

The os module provides low‑level system functions. Still widely used in legacy code and for operations not yet in pathlib.

Step-by-step explanation:

  • Step 1: Import os — the standard library for operating system interfaces.
  • Step 2: Use os.getcwd() to get the current working directory (string).
  • Step 3: Use os.listdir() to list files in a directory.
  • Step 4: Use os.path.join() to build paths — this is cross‑platform.
  • Step 5: Use os.path.exists(), os.path.isdir(), and os.path.isfile() to check properties.
  • Step 6: Use os.environ to access environment variables — useful for configuration.
python 
import os

def os_demo():
    cwd = os.getcwd()
    print(f"Current directory: {cwd}")

    print("\nFiles (os.listdir):")
    for name in os.listdir(cwd):
        path = os.path.join(cwd, name)
        if os.path.isdir(path):
            print(f"  📁 {name}")
        else:
            print(f"  📄 {name}")

    config_path = os.path.join(cwd, "config", "settings.ini")
    print(f"\nBuilt path: {config_path}")
    print(f"Exists: {os.path.exists(config_path)}")

    print("\nEnvironment variables (first 3):")
    env = os.environ
    for i, (k, v) in enumerate(env.items()):
        if i >= 3: break
        print(f"  {k}={v[:30]}...")

os_demo()
✨ Prefer pathlib for new code — it's more intuitive and less error‑prone. Use os when you need specific system calls.

📄 File Operations — Create, Read, Write, Delete

Automation is mostly about files. Learn to create, read, write, move, and delete files programmatically.

📄 2.1 Reading and Writing Text Files

Use open() with context managers to read and write text files safely.

Step-by-step explanation:

  • Step 1: Define a filename and some sample content.
  • Step 2: Use with open(...) to write to the file ("w" mode).
  • Step 3: Use with open(...) to read the file ("r" mode).
  • Step 4: Read the entire content with read() or line by line with readlines().
  • Step 5: Append to a file with "a" mode.
  • Step 6: Delete the file using Path.unlink().
python 
from pathlib import Path

def file_rw_demo():
    filename = "automation_demo.txt"
    path = Path(filename)

    content = "Hello from Python automation!\nLine 2 of content.\n"
    with open(path, "w", encoding="utf-8") as f:
        f.write(content)
    print(f"✅ Wrote {len(content)} characters to {filename}")

    with open(path, "r", encoding="utf-8") as f:
        data = f.read()
    print(f"📄 Read content: {repr(data)}")

    with open(path, "a", encoding="utf-8") as f:
        f.write("Appended line!\n")
    print("✅ Appended a line")

    with open(path, "r", encoding="utf-8") as f:
        lines = f.readlines()
    print(f"📄 Lines: {lines}")

    path.unlink(missing_ok=True)
    print("🧹 Deleted file")

📄 2.2 Working with Directories

Create, rename, list, and delete directories — essential for any automation script.

Step-by-step explanation:

  • Step 1: Create a directory with Path.mkdir().
  • Step 2: Check if a directory exists with Path.exists().
  • Step 3: Create nested directories with parents=True.
  • Step 4: List directory contents with iterdir().
  • Step 5: Remove an empty directory with Path.rmdir().
  • Step 6: Remove a directory and all its contents with shutil.rmtree().
python 
from pathlib import Path
import shutil

def dir_demo():
    base = Path.cwd() / "automation_test"
    nested = base / "sub" / "subsub"

    nested.mkdir(parents=True, exist_ok=True)
    print(f"✅ Created: {nested}")

    (base / "sub" / "hello.txt").write_text("Hello nested dir!")

    print("\n📂 Directory tree:")
    for item in base.rglob("*"):
        indent = "  " * (len(item.relative_to(base).parts) - 1)
        if item.is_file():
            print(f"{indent}📄 {item.name}")
        else:
            print(f"{indent}📁 {item.name}")

    shutil.rmtree(base, ignore_errors=True)
    print(f"\n🧹 Deleted: {base}")

📦 shutil — High‑Level File Operations

shutil provides copy, move, archive, and delete operations that work across filesystems.

📦 3.1 Copy, Move, and Rename

Copy files and directories, move them, or rename them — all with shutil.

Step-by-step explanation:

  • Step 1: Import shutil and pathlib.
  • Step 2: Create a source file with some content.
  • Step 3: Use shutil.copy2() to copy with metadata preservation.
  • Step 4: Use shutil.move() to move or rename a file.
  • Step 5: Copy an entire directory tree with shutil.copytree().
  • Step 6: Clean up by removing created files and directories.
python 
from pathlib import Path
import shutil

def shutil_demo():
    src = Path.cwd() / "source.txt"
    src.write_text("Original content")

    dest = Path.cwd() / "copy.txt"
    shutil.copy2(src, dest)
    print(f"✅ Copied {src} → {dest}")

    renamed = Path.cwd() / "renamed.txt"
    shutil.move(dest, renamed)
    print(f"✅ Moved {dest} → {renamed}")

    print(f"📄 {renamed} contains: {renamed.read_text()}")

    src.unlink()
    renamed.unlink()
    print("🧹 Cleaned up")

📦 3.2 Archiving — ZIP and TAR

Create and extract archives programmatically — perfect for backups and distribution.

Step-by-step explanation:

  • Step 1: Create a directory with some files to archive.
  • Step 2: Use shutil.make_archive() to create a ZIP archive.
  • Step 3: Use shutil.unpack_archive() to extract the archive.
  • Step 4: List the contents of the extracted directory.
  • Step 5: Clean up the temporary files.
python 
from pathlib import Path
import shutil

def archive_demo():
    base = Path.cwd() / "archive_test"
    base.mkdir(exist_ok=True)

    (base / "file1.txt").write_text("Content 1")
    (base / "file2.txt").write_text("Content 2")
    sub = base / "sub"
    sub.mkdir()
    (sub / "file3.txt").write_text("Content 3")

    archive_name = shutil.make_archive("my_archive", "zip", base)
    print(f"✅ Created archive: {archive_name}")

    extract_dir = Path.cwd() / "extracted"
    shutil.unpack_archive(archive_name, extract_dir)
    print(f"✅ Extracted to: {extract_dir}")

    print("\n📂 Extracted contents:")
    for item in extract_dir.rglob("*"):
        if item.is_file():
            print(f"  📄 {item.relative_to(extract_dir)}")

    shutil.rmtree(base, ignore_errors=True)
    shutil.rmtree(extract_dir, ignore_errors=True)
    Path(archive_name).unlink(missing_ok=True)
    print("\n🧹 Cleaned up")

archive_demo()
✨ shutil.make_archive() supports "zip", "tar", "gztar", and "bztar" formats.

💻 Running External Commands — subprocess

Automation often means orchestrating other programs. Learn to run, control, and capture output from external commands.

💻 4.1 subprocess.run() — The Modern Way

subprocess.run() is the recommended way to run external commands in Python 3.5+.

Step-by-step explanation:

  • Step 1: Import subprocess.
  • Step 2: Use subprocess.run() with a list of arguments.
  • Step 3: Capture output with capture_output=True.
  • Step 4: Access result.stdout and result.stderr as bytes or strings with text=True.
  • Step 5: Check the return code with result.returncode.
  • Step 6: Use check=True to raise an exception on non‑zero exit codes.
python 
import subprocess

def subprocess_demo():
    result = subprocess.run(["echo", "Hello from subprocess!"], capture_output=True, text=True)
    print(f"✅ Command output: {result.stdout.strip()}")
    print(f"Return code: {result.returncode}")

    try:
        subprocess.run(["false"], check=True)
    except subprocess.CalledProcessError as e:
        print(f"❌ Command failed with exit code: {e.returncode}")

    result = subprocess.run(["ls", "/nonexistent"], capture_output=True, text=True)
    print(f"stderr: {result.stderr.strip()}")

subprocess_demo()
💡 For shell‑like behavior (pipes, wildcards), use shell=True — but be careful with user input to avoid injection.

💻 4.2 Piping and Advanced Control

Chain commands, set environment variables, and control timeouts.

Step-by-step explanation:

  • Step 1: Use subprocess.Popen() for more control.
  • Step 2: Pipe output from one command to another.
  • Step 3: Set environment variables with the env parameter.
  • Step 4: Set a timeout to prevent hanging.
  • Step 5: Handle the timeout exception.
python 
import subprocess
import os

def advanced_subprocess():
    p1 = subprocess.Popen(["ls"], stdout=subprocess.PIPE, text=True)
    p2 = subprocess.Popen(["grep", ".py"], stdin=p1.stdout, stdout=subprocess.PIPE, text=True)
    p1.stdout.close()
    output = p2.communicate()[0]
    print("🔍 Python files:")
    print(output)

    env = os.environ.copy()
    env["MY_VAR"] = "automation"
    result = subprocess.run(["printenv", "MY_VAR"], env=env, capture_output=True, text=True)
    print(f"✅ MY_VAR = {result.stdout.strip()}")

⏰ Scheduling Tasks — Run on a Schedule

Automation runs on its own schedule. Learn to schedule tasks with Python's schedule library and time.

⏰ 5.1 Basic Scheduling with schedule

The schedule library provides a human‑friendly API for scheduling tasks.

Step-by-step explanation:

  • Step 1: Import schedule and time.
  • Step 2: Define a function to run on schedule.
  • Step 3: Use schedule.every(). syntax to set the schedule.
  • Step 4: In an infinite loop, call schedule.run_pending().
  • Step 5: Use time.sleep(1) to avoid busy‑waiting.
  • Step 6: Run the scheduler for a limited number of iterations for demo purposes.
python 
import schedule
import time

def scheduled_task():
    print(f"⏰ Task ran at {time.strftime('%H:%M:%S')}")

def scheduler_demo():
    print("⏳ Starting scheduler (runs for 5 seconds)")

    schedule.every(2).seconds.do(scheduled_task)

    start = time.time()
    while time.time() - start < 5:
        schedule.run_pending()
        time.sleep(0.5)

    print("✅ Scheduler stopped")

scheduler_demo()
✨ The schedule library supports seconds, minutes, hours, days, weeks, and specific times of day.

⏰ 5.2 Cron‑Like Scheduling with apscheduler

apscheduler provides cron‑like scheduling with more advanced features.

Step-by-step explanation:

  • Step 1: Import BackgroundScheduler from apscheduler.schedulers.background.
  • Step 2: Import CronTrigger for cron‑style scheduling.
  • Step 3: Define a function to run on schedule.
  • Step 4: Create a BackgroundScheduler and add a job.
  • Step 5: Start the scheduler.
  • Step 6: Keep the program running for a few seconds.
  • Step 7: Shut down the scheduler.
python 
from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.triggers.cron import CronTrigger
import time

def cron_task():
    print(f"⏰ Cron task ran at {time.strftime('%Y-%m-%d %H:%M:%S')}")

def cron_demo():
    scheduler = BackgroundScheduler()
    scheduler.add_job(cron_task, CronTrigger(second="*/2"))
    scheduler.start()
    print("⏳ Cron scheduler running for 6 seconds...")

    time.sleep(6)
    scheduler.shutdown()
    print("✅ Cron scheduler stopped")

🌐 6.2 Parsing HTML with BeautifulSoup

Use BeautifulSoup to parse HTML and extract structured data.

Step-by-step explanation:

  • Step 1: Import BeautifulSoup from bs4.
  • Step 2: Create a sample HTML string.
  • Step 3: Parse the HTML with BeautifulSoup(html, "html.parser").
  • Step 4: Use soup.find() and soup.find_all() to extract elements.
  • Step 5: Access attributes with tag.get("href").

from bs4 import BeautifulSoup

def soup_demo():
    html = """
    <html>
        <body>
            <h1>Automation Tutorial</h1>
            <ul class="links">
                <li><a href="/lesson1">Lesson 1</a></li>
                <li><a href="/lesson2">Lesson 2</a></li>
                <li><a href="/lesson3">Lesson 3</a></li>
            </ul>
            <p class="desc">Learn Python automation.</p>
        </body>
    </html>
    """

    title = soup.find("h1").text
    print(f"📄 Title: {title}")

    for link in links:
        print(f"  - {link.text} → {link.get('href')}")

    desc = soup.find("p", class_="desc")
    print(f"📝 Description: {desc.text if desc else 'Not found'}")

🖥️ System Monitoring — psutil

Monitor CPU, memory, disk, and network usage with psutil — the cross‑platform system monitoring library.

🖥️ 9.1 System Metrics with psutil

Get real‑time system information: CPU, RAM, disk, and processes.

Step-by-step explanation:

  • Step 1: Import psutil.
  • Step 2: Use psutil.cpu_percent() to get CPU usage.
  • Step 3: Use psutil.virtual_memory() for RAM details.
  • Step 4: Use psutil.disk_usage() for disk space.
  • Step 5: Use psutil.net_io_counters() for network stats.
  • Step 6: Use psutil.process_iter() to list running processes.
python 
import psutil

def system_monitor():
    print("🖥️ System Monitoring")

    cpu = psutil.cpu_percent(interval=0.5)
    print(f"CPU: {cpu}%")

    mem = psutil.virtual_memory()
    print(f"RAM: {mem.percent}% used ({mem.used // (1024**2)} MB / {mem.total // (1024**2)} MB)")

    disk = psutil.disk_usage("/")
    print(f"Disk: {disk.percent}% used ({disk.used // (1024**3)} GB / {disk.total // (1024**3)} GB)")

    net = psutil.net_io_counters()
    print(f"Network: {net.bytes_sent // (1024**2)} MB sent, {net.bytes_recv // (1024**2)} MB received")

    processes = list(psutil.process_iter(["pid", "name"]))[:5]
    print(f"\n📋 Top 5 processes:")
    for p in processes:
        print(f"  PID {p.info['pid']}: {p.info['name']}")

🚀 11.1 Backup Script

A complete backup script that creates timestamped archives of a directory.

Step-by-step explanation:

  • Step 1: Import shutil, pathlib, and datetime.
  • Step 2: Define source and backup directories.
  • Step 3: Generate a timestamp for the archive name.
  • Step 4: Use shutil.make_archive() to create the backup.
  • Step 5: Print success message with the backup location.
python 
from pathlib import Path
import shutil
from datetime import datetime

def backup_demo():
    source = Path.cwd() / "data_to_backup"
    source.mkdir(exist_ok=True)

    (source / "file1.txt").write_text("Backup this!")
    (source / "file2.txt").write_text("And this too!")

    backup_dir = Path.cwd() / "backups"
    backup_dir.mkdir(exist_ok=True)

    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    archive_name = backup_dir / f"backup_{timestamp}"

    shutil.make_archive(str(archive_name), "zip", source)
    print(f"✅ Backup created: {archive_name}.zip")

    shutil.rmtree(source, ignore_errors=True)
    Path(f"{archive_name}.zip").unlink(missing_ok=True)
    backup_dir.rmdir()
    print("🧹 Cleaned up")

🚀 11.2 File Organizer

Sort files in a directory by extension into subfolders.

Step-by-step explanation:

  • Step 1: Define a mapping of extensions to folder names.
  • Step 2: Iterate over all files in the directory.
  • Step 3: Get the file extension with Path.suffix.
  • Step 4: Determine the target folder from the mapping.
  • Step 5: Create the folder if it doesn't exist.
  • Step 6: Move the file to the target folder with shutil.move().
python 
from pathlib import Path
import shutil

def organize_files_demo():
    base = Path.cwd() / "to_organize"
    base.mkdir(exist_ok=True)

    for ext in [".txt", ".pdf", ".jpg", ".py", ".docx"]:
        (base / f"file{ext}").write_text(f"Sample {ext} file")

    mapping = {
        ".txt": "text_files",
        ".pdf": "pdf_files",
        ".jpg": "images",
        ".png": "images",
        ".py": "python_scripts",
        ".docx": "documents",
    }

    print("📂 Organizing files...")
    for item in base.iterdir():
        if item.is_file():
            ext = item.suffix.lower()
            target = mapping.get(ext, "other")
            target_dir = base / target
            target_dir.mkdir(exist_ok=True)
            shutil.move(str(item), str(target_dir / item.name))
            print(f"  Moved {item.name} → {target}/")

    print("\n📂 Organized structure:")
    for dir_name in ["text_files", "pdf_files", "images", "python_scripts", "documents", "other"]:
        dir_path = base / dir_name
        if dir_path.exists():
            files = list(dir_path.glob("*"))
            print(f"  {dir_name}/: {len(files)} files")

    shutil.rmtree(base, ignore_errors=True)
    print("\n🧹 Cleaned up")

organize_files_demo()

python 
import psutil

def health_check():
    thresholds = {
        "cpu": 80,
        "ram": 80,
        "disk": 85
    }

    cpu = psutil.cpu_percent(interval=0.5)
    mem = psutil.virtual_memory()
    disk = psutil.disk_usage("/")

    alerts = []
    if cpu > thresholds["cpu"]:
        alerts.append(f"CPU at {cpu}% (threshold: {thresholds['cpu']}%)")
    if mem.percent > thresholds["ram"]:
        alerts.append(f"RAM at {mem.percent}% (threshold: {thresholds['ram']}%)")
    if disk.percent > thresholds["disk"]:
        alerts.append(f"Disk at {disk.percent}% (threshold: {thresholds['disk']}%)")

    print("🖥️ System Health Check")
    print(f"CPU: {cpu}%")
    print(f"RAM: {mem.percent}%")
    print(f"Disk: {disk.percent}%")

    if alerts:
        print("\n⚠️ ALERTS:")
        for alert in alerts:
            print(f"  ❌ {alert}")
    else:
        print("\n✅ All systems healthy!")

health_check()

❓ Frequently Asked Questions — Python Automation

Common questions about automating with Python, answered.

What's the difference between shutil and os for file operations?

+

os provides low‑level system functions. shutil builds on os to provide high‑level operations like copying entire directory trees, moving files across filesystems, and creating archives. Use shutil for common tasks, os for fine‑grained control.

How do I run a Python script on a schedule?

+

Use the schedule library for in‑process scheduling, or use your operating system's scheduler: cron on Linux/macOS, Task Scheduler on Windows. For production, consider APScheduler or a dedicated orchestration tool like Celery.

Is web scraping legal?

+

Check the website's robots.txt and terms of service. Scraping public data for personal use is usually fine, but avoid overwhelming servers (use delays) and respect copyright. Always identify your bot with a custom User-Agent.

What's the best library for Excel automation?

+

openpyxl is the most popular for reading/writing .xlsx files. For older .xls files, use xlrd (read) and xlwt (write). For data‑heavy workflows, consider pandas which uses openpyxl or xlrd under the hood.

How do I keep my automation scripts secure?

+

Never hard‑code credentials. Use environment variables, .env files, or a secrets manager. Validate all inputs, especially when using subprocess with shell=True. Run automation scripts with the least privilege necessary.

What should I automate first?

+

Start with tasks you do repeatedly: file renaming, data entry, report generation, or system checks. The best automation is the one that saves you the most time. Start small and iterate.

Can I use Python to automate Excel on macOS/Linux?

+

Yes! openpyxl works on all platforms because it reads/writes the file format directly — no Excel installation needed. For Windows‑specific COM automation (controlling the Excel application itself), you'd need win32com which only works on Windows.
