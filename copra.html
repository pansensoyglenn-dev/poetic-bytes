01 — The Context

Why Copra & Rubber, Why Python?

🥥 Building a Philippine Copra & Rubber Trading Post in Python

Copra and rubber are generational livelihoods across Quezon, Davao, Cotabato, and Basilan. A trading post — locally a bagsakan — buys raw copra and rubber from farmers, then sells it onward to processors and exporters at a higher price. That buy-sell spread is the business.

This tutorial is built around one complete Python program, but the real subject is formal abstraction — using Python's built-in abc module to define an interface that the language itself enforces, not just a convention developers agree to follow.

Every code example here is fully interactive — copy or run each snippet right in your browser using the Pyodide Python runtime. No setup required.

From Informal to Formal Abstraction

The rice.html tutorial hid complexity inside a method — informal abstraction. Here, we go further: TradableProduct(ABC) defines a contract with @abstractmethod that every subclass must fulfill. Forget to implement buy_price and Python refuses to create the object — a TypeError, not a silent bug found later.

Definition: Formal abstraction means defining an interface that the language enforces at runtime — not just naming a function and hoping callers respect the intent behind it.

Three-Class Architecture

The trading post uses an abstract base plus two concrete product types — a minimal hierarchy that shows inheritance and formal contracts working together.

  • TradableProduct — the abstract base. Cannot be instantiated. Requires buy_price and sell_price.
  • CopraProduct / RubberProduct — concrete subclasses with commodity-specific data.
  • CopraRubberTradingPost — orchestrates the buy/sell workflow and receipts.

Why an abstract base? It guarantees every product — present or future — exposes both prices, letting the receipt engine trust the interface without knowing the concrete type.

02 — Full Source Code

The Complete copra_rubber_trading.py

This is the complete Python program. Copy it, run it, modify it — every part is explained below.

python 
from abc import ABC, abstractmethod
from datetime import datetime


class TradableProduct(ABC):
    """Abstract base -- enforces buy_price, sell_price, and display()."""

    def __init__(self, code, name, unit, origin, grade):
        self.code   = code
        self.name   = name
        self.unit   = unit
        self.origin = origin
        self.grade  = grade

    @property
    @abstractmethod
    def buy_price(self): ...

    @property
    @abstractmethod
    def sell_price(self): ...

    @abstractmethod
    def display(self): ...

    def margin(self):
        return self.sell_price - self.buy_price


class CopraProduct(TradableProduct):
    def __init__(self, code, name, origin, grade, buy, sell):
        super().__init__(code, name, "kg", origin, grade)
        self._buy, self._sell = buy, sell

    @property
    def buy_price(self):  return self._buy

    @property
    def sell_price(self): return self._sell

    def display(self):
        print(f"  [{self.code}] {self.name:<28} | Grade: {self.grade:<8} | "
              f"Buy: ₱{self._buy:>7.2f}/kg  Sell: ₱{self._sell:>7.2f}/kg")


class RubberProduct(TradableProduct):
    def __init__(self, code, name, origin, grade, buy, sell, rubber_type):
        super().__init__(code, name, "kg", origin, grade)
        self._buy, self._sell = buy, sell
        self.rubber_type = rubber_type

    @property
    def buy_price(self):  return self._buy

    @property
    def sell_price(self): return self._sell

    def display(self):
        print(f"  [{self.code}] {self.name:<28} | Type: {self.rubber_type:<10} | "
              f"Buy: ₱{self._buy:>7.2f}/kg  Sell: ₱{self._sell:>7.2f}/kg")


class CopraRubberTradingPost:
    def __init__(self):
        self.catalog = {
            "C1": CopraProduct("C1", "Sun-Dried Copra",  "Quezon",    "Grade A", 28.00, 32.00),
            "C2": CopraProduct("C2", "Kiln-Dried Copra", "Davao",     "Grade A", 30.00, 35.00),
            "C3": CopraProduct("C3", "Ball Copra",       "Laguna",    "Grade B", 24.00, 28.00),
            "R1": RubberProduct("R1", "Cup Lump Standard", "Cotabato", "Grade 1", 52.00, 58.00, "Cup Lump"),
            "R2": RubberProduct("R2", "RSS-1 Smoked Sheet", "Zamboanga", "RSS-1", 68.00, 75.00, "RSS"),
        }
        self.transaction_log = []

    def show_catalog(self):
        print("\n  📦  CURRENT PRICE LIST:")
        for code, p in self.catalog.items():
            p.display()

    def _encode_items(self, mode):
        # mode = 'buy' (we pay a seller) or 'sell' (we charge a customer)
        items = []
        while True:
            code = input("Enter code (DONE to finish): ").strip().upper()
            if code == "DONE": break
            if code not in self.catalog:
                print("❌ Invalid code."); continue
            try:
                qty = float(input(f"Quantity in kg for {self.catalog[code].name}? "))
                price = self.catalog[code].buy_price if mode == "buy" else self.catalog[code].sell_price
                items.append((code, qty, price, price * qty))
                print(f"✅ {qty}kg × ₱{price:.2f} = ₱{price*qty:.2f}")
            except ValueError:
                print("❌ Invalid quantity.")
        return items

    def print_transaction(self, party, mode, items):
        if not items: print("🛒 No items encoded."); return
        total = sum(s for _, _, _, s in items)
        label = "BUYING RECEIPT" if mode == "buy" else "SELLING RECEIPT"
        print(f"\n{label} — {party} — {datetime.now():%b %d, %Y}")
        for code, qty, price, sub in items:
            p = self.catalog[code]
            print(f"  {p.name:<28} {qty:>6.1f}kg  ₱{sub:>9.2f}")
        print(f"  {'TOTAL':<28} {'':>6}    ₱{total:>9.2f}")
        self.transaction_log.append({"party": party, "mode": mode, "total": total})

    def run(self):
        self.show_catalog()
        choice = input("\n[1] BUY from seller  [2] SELL to customer: ").strip()
        mode   = "buy" if choice == "1" else "sell"
        party  = input("Name: ").strip().title()
        items  = self._encode_items(mode)
        self.print_transaction(party, mode, items)


if __name__ == "__main__":
    post = CopraRubberTradingPost()
    post.run()
03 — Step by Step

Code Explanation

Step 1: Abstract Base Class

class TradableProduct(ABC):
    @property
    @abstractmethod
    def buy_price(self): ...

    @property
    @abstractmethod
    def sell_price(self): ...
📝 Why This Code Was Used & How It Works
  1. Why ABC? Inheriting from ABC makes TradableProduct uninstantiable on its own. TradableProduct() raises TypeError — it only exists to be subclassed.
  2. Why @abstractmethod? Any subclass that forgets to implement buy_price or sell_price can't be instantiated either — the contract is enforced by the interpreter, not documentation.
  3. Why stack it with @property? It forces subclasses to expose the value as an attribute — product.buy_price, not product.buy_price() — a uniform interface across every product type.

Step 2: Subclasses & super().__init__()

class CopraProduct(TradableProduct):
    def __init__(self, code, name, origin, grade, buy, sell):
        super().__init__(code, name, "kg", origin, grade)
        self._buy, self._sell = buy, sell
📝 Why This Code Was Used & How It Works
  1. Why super().__init__()? It delegates the five shared fields to the parent constructor. The child only sets its own data — no duplicated initialization logic between CopraProduct and RubberProduct.
  2. Why the underscore in _buy? A leading underscore signals "internal use" by convention. External code reads product.buy_price through the property, not the raw attribute.
  3. Adding a new commodity: A future CharcoalProduct(TradableProduct) only needs its own __init__ and the two required properties — the rest of the system supports it automatically.

Step 3: One Method, Two Modes

def _encode_items(self, mode):
    price = self.catalog[code].buy_price if mode == "buy" else self.catalog[code].sell_price
    items.append((code, qty, price, price * qty))
📝 Why This Code Was Used & How It Works
  1. Why one method for both modes? Buying and selling only differ in which price applies. A single mode parameter avoids duplicating the entire input loop — DRY (Don't Repeat Yourself).
  2. The ternary expression: a if cond else b is Python's inline conditional — a compact, readable way to pick between two values.
  3. Tuple storage: Each encoded item is a (code, qty, price, subtotal) tuple — an immutable snapshot of a completed calculation, ready for the receipt.

Step 4: Generator Total & Transaction Log

total = sum(s for _, _, _, s in items)
self.transaction_log.append({"party": party, "mode": mode, "total": total})
📝 Why This Code Was Used & How It Works
  1. Generator expression: sum(s for _,_,_,s in items) unpacks each tuple and streams the subtotal straight into sum() — memory-efficient and Pythonic.
  2. Why log transactions? Appending a summary dict after every receipt builds a running record of the day's business — the seed of a real reporting feature.

💡 margin() on the base class: TradableProduct.margin() returns sell_price - buy_price as a concrete method. Because both abstract properties are guaranteed on every subclass, this calculation is safe to write once — shared logic belongs on the parent, specific data belongs on the child.

08 — Glossary

Key Terms

abc.ABCAbstract Base Class. Inheriting from it blocks direct instantiation — TradableProduct() raises TypeError. It exists only to be subclassed.
@abstractmethodMarks a method as required in any concrete subclass. Omitting it prevents that subclass from being instantiated at all.
@propertyConverts a method into a read-only attribute. product.buy_price looks like plain data access but runs a method behind the scenes.
super().__init__()Calls the parent class's constructor so shared attributes are set before the subclass adds its own.
isinstance()Returns True if an object belongs to a class or its subclasses. Used to separate copra from rubber when both live in one dictionary.
margin()A concrete method on the abstract base: sell_price - buy_price. Safe to define once because both properties are guaranteed on every subclass.
f-stringFormatted string literal. f"₱{price:.2f}" embeds variables with format specifiers directly in the string.
generator expressionA lazy iterator that computes values on demand: sum(s for _,_,_,s in items). More memory-efficient than building a full list first.
CopraDried coconut meat, the primary raw material for coconut oil. Graded by drying method — sun-dried, kiln-dried, ball, or split.
Cup Lump / RSSTwo natural rubber grades. Cup Lump forms by coagulation in collection cups; Ribbed Smoked Sheet (RSS) is rolled and smoked for higher quality.
