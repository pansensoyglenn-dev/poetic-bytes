What is Web Development?

The art of building websites and web applications

Web development is the process of building websites and web applications that run in a browser. It's divided into frontend (what users see — HTML, CSS, JavaScript) and backend (what powers the site — Python, databases, servers). This course covers both sides, with Python as the backend language of choice.

The Web Stack

Frontend: HTML (structure) → CSS (style) → JavaScript (interactivity). Backend: Python (Flask/Django) → Database → Server. Full‑stack: combines both.

HTML — Structure

The foundation of every web page — elements, attributes, and semantic markup

🏗️ Basic HTML Document Structure

Every HTML document has a <!DOCTYPE>, <html>, <head>, and <body>. This is the skeleton of every web page.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is a paragraph of text on my first web page.</p>
    <a href="https://example.com">Click here to visit Example.com</a>
</body>
</html>
[HTML page rendered: "Welcome to My Website" with a paragraph and link]
💡 The <head> contains metadata (title, charset, viewport). The <body> contains visible content.

CSS — Styling

Making websites beautiful — colors, layout, animations, and responsive design

🎨 CSS Basics — Selectors and Properties

CSS selects HTML elements and applies styles. Common properties: color, background, font-size, margin, padding.

/* Element selector */
body {
    font-family: Arial, sans-serif;
    background-color: #f4f4f4;
    color: #333;
}

/* Class selector */
.container {
    max-width: 800px;
    margin: 0 auto;
    padding: 20px;
    background: white;
    border-radius: 8px;
    box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}

/* ID selector */
#header {
    background: #4B8BBE;
    color: white;
    padding: 20px;
    text-align: center;
}

/* Descendant selector */
.container p {
    line-height: 1.6;
    margin-bottom: 1rem;
}

/* Pseudo-classes */
a:hover {
    color: #FFD966;
    text-decoration: underline;
}

button:active {
    transform: scale(0.95);
}
[CSS styles defined: body, container, header, hover effects]
💡 Use class selectors (.classname) for reusable styles. ID selectors (#id) should be unique per page.

📐 Flexbox — Modern Layout

Flexbox makes one‑dimensional layouts easy — aligning items horizontally or vertically with powerful control.

/* Flex container */
.flex-container {
    display: flex;
    flex-direction: row;        /* row | column | row-reverse | column-reverse */
    justify-content: center;    /* flex-start | center | space-between | space-around */
    align-items: center;        /* flex-start | center | stretch | baseline */
    gap: 1rem;
    flex-wrap: wrap;            /* nowrap | wrap | wrap-reverse */
}

/* Flex items */
.flex-item {
    flex: 1;                   /* Grow to fill available space */
    min-width: 150px;
    padding: 1rem;
    background: #4B8BBE;
    color: white;
    border-radius: 4px;
    text-align: center;
}

/* Centering with flex */
.centered {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 200px;
    background: #1a2535;
}
[Flexbox layout: items centered with gap and wrap]
💡 Flexbox is the standard for modern layouts. flex: 1 distributes space equally among items.

📐 CSS Grid — Two‑Dimensional Layout

Grid creates complex, two‑dimensional layouts — rows and columns with complete control.

/* Grid container */
.grid-container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;  /* 3 columns with different widths */
    grid-template-rows: 100px 200px;      /* Row heights */
    gap: 1rem;
}

/* Common grid patterns */
.grid-2col {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 2rem;
}

.grid-3col {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 1.5rem;
}

/* Responsive grid */
.grid-responsive {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: 1.5rem;
}

/* Grid item spanning */
.grid-item-2col {
    grid-column: span 2;
}

.grid-item-full {
    grid-column: 1 / -1;  /* Span all columns */
}
[CSS Grid layouts: 2‑col, 3‑col, responsive grid, and spanning items]
💡 auto-fit and minmax() create responsive grids that adapt to screen size automatically.

📱 Responsive Design — Media Queries

Media queries adapt styles to different screen sizes — essential for mobile‑friendly websites.

/* Mobile-first approach — default styles for small screens */
body { font-size: 16px; }
.container { padding: 1rem; }

/* Tablets (768px and up) */
@media (min-width: 768px) {
    body { font-size: 18px; }
    .container { padding: 2rem; }
    .grid-responsive {
        grid-template-columns: repeat(2, 1fr);
    }
}

/* Desktops (1024px and up) */
@media (min-width: 1024px) {
    body { font-size: 20px; }
    .container { max-width: 1200px; margin: 0 auto; }
    .grid-responsive {
        grid-template-columns: repeat(3, 1fr);
    }
}

/* Hide on small screens */
@media (max-width: 480px) {
    .desktop-only {
        display: none;
    }
    .mobile-only {
        display: block;
    }
}
[Responsive CSS: mobile-first with breakpoints at 768px and 1024px]
💡 Mobile‑first design starts with small screens and adds styles for larger screens — recommended practice.

📋 DOM Manipulation

The Document Object Model (DOM) represents the page structure — JavaScript can change it dynamically.

// Select elements
const heading = document.querySelector("h1");
const paragraphs = document.querySelectorAll("p");
const button = document.getElementById("my-button");
const container = document.querySelector(".container");

// Change content
heading.textContent = "New Heading";
heading.innerHTML = "<span style='color:red'>Red Heading</span>";

// Change styles
heading.style.color = "#4B8BBE";
heading.style.fontSize = "2rem";

// Add/remove classes
heading.classList.add("highlight");
heading.classList.remove("old-class");
heading.classList.toggle("active");

// Create new elements
const newDiv = document.createElement("div");
newDiv.textContent = "I'm new!";
container.appendChild(newDiv);
[DOM manipulation: selecting, modifying, and creating elements]
💡 document.querySelector() selects the first matching element — querySelectorAll() returns all matches.

🖱️ Event Handling

JavaScript responds to user actions — clicks, form submissions, key presses, and more.

// Click event
button.addEventListener("click", function() {
    alert("Button clicked!");
});

// Arrow function
button.addEventListener("click", () => {
    console.log("Button clicked");
});

// Form submission
const form = document.querySelector("form");
form.addEventListener("submit", (event) => {
    event.preventDefault();  // Prevent page reload
    const formData = new FormData(form);
    console.log("Form data:", formData);
});

// Mouse events
element.addEventListener("mouseover", () => {});
element.addEventListener("mouseout", () => {});

// Keyboard events
document.addEventListener("keydown", (e) => {
    if (e.key === "Enter") {
        console.log("Enter pressed");
    }
});

// Input events
input.addEventListener("input", (e) => {
    console.log(e.target.value);
});
[Event listeners: click, submit, mouseover, keydown, input]
💡 event.preventDefault() stops default behavior (like form submission refreshing the page).

🌐 Fetching Data — AJAX and APIs

JavaScript can fetch data from APIs without reloading the page — the foundation of modern web apps.

// Fetch API (modern)
fetch("https://api.example.com/data")
    .then(response => response.json())
    .then(data => {
        console.log("Data:", data);
        // Update DOM with data
        document.querySelector("#results").innerHTML = data;
    })
    .catch(error => console.error("Error:", error));

// Async/await (cleaner)
async function fetchData() {
    try {
        const response = await fetch("https://api.example.com/data");
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error(error);
    }
}

// POST request
async function postData() {
    const response = await fetch("/api/users", {
        method: "POST",
        headers: {
            "Content-Type": "application/json"
        },
        body: JSON.stringify({ name: "Alice", age: 25 })
    });
    const data = await response.json();
    return data;
}
[Fetch API: GET and POST requests with async/await]
💡 Fetch returns a Promise — use async/await or .then() chains. Always handle errors with .catch() or try/catch.

Flask Web Apps

Serving HTML, templates, static files, and building full web applications

🏗️ Flask with Templates (Jinja2)

Flask uses Jinja2 templates to generate HTML dynamically — perfect for building web pages with Python data.

from flask import Flask, render_template, request

app = Flask(__name__)

@app.route("/")
def home():
    return render_template("index.html", title="Home")

@app.route("/user/<name>")
def user(name):
    user_data = {
        "name": name,
        "age": 30,
        "city": "Berlin",
        "hobbies": ["coding", "reading", "hiking"]
    }
    return render_template("user.html", user=user_data)

# templates/index.html
# <!DOCTYPE html>
# <html>
# <head><title>{{ title }}</title></head>
# <body>
#   <h1>Welcome to {{ title }}</h1>
#   <a href="/user/Alice">View User</a>
# </body>
# </html>
[Flask app: renders index.html and user.html with dynamic data]
💡 Jinja2 uses {{ variable }} for output and {% for item in items %} for loops — powerful and clean.

📁 Serving Static Files

Flask serves static files (CSS, JS, images) from the static/ folder — essential for styling and interactivity.

/* static/style.css */
body {
    font-family: 'Syne', sans-serif;
    background: '#0B121C';
    color: '#E2E8F0';
}
.container { max-width: 800px; margin: 0 auto; padding: 2rem; }

/* templates/base.html (inheritance) */
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My App{% endblock %}</title>
    <link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">
</head>
<body>
    <div class="container">
        {% block content %}{% endblock %}
    </div>
    <script src="{{ url_for('static', filename='script.js') }}"></script>
</body>
</html>
[Static files served: style.css, script.js from static/ folder]
💡 url_for('static', filename='style.css') generates the correct URL for static files — even in production.

📝 Flask Forms and Handling POST Data

Handle form submissions — validate input, process data, and return a response.

from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)

@app.route("/contact", methods=["GET", "POST"])
def contact():
    if request.method == "POST":
        name = request.form.get("name")
        email = request.form.get("email")
        message = request.form.get("message")
        
        # Process data (save to DB, send email, etc.)
        print(f"Name: {name}, Email: {email}, Message: {message}")
        
        return render_template("thank_you.html", name=name)
    
    return render_template("contact.html")

# templates/contact.html
# <form method="POST">
#   <input type="text" name="name" placeholder="Name" required>
#   <input type="email" name="email" placeholder="Email" required>
#   <textarea name="message" placeholder="Message"></textarea>
#   <button type="submit">Send</button>
# </form>
[Flask form: GET for display, POST for processing]
💡 Use request.form for form data (POST), request.args for query parameters (GET).

Django Web Apps

Full‑stack framework with built‑in admin, ORM, authentication, and templates

🏛️ Django Templates

Django's template language is similar to Jinja2 but with different syntax — {{ }} for variables, {% %} for logic.

# views.py
from django.shortcuts import render
from .models import Post

def home(request):
    posts = Post.objects.all().order_by("-created_at")[:5]
    return render(request, "blog/home.html", {"posts": posts})

def post_detail(request, post_id):
    post = Post.objects.get(id=post_id)
    return render(request, "blog/detail.html", {"post": post})

# templates/blog/home.html
{% extends "base.html" %}

{% block content %}
    <h1>Blog Posts</h1>
    {% for post in posts %}
        <article>
            <h2>{{ post.title }}</h2>
            <p>By {{ post.author }} on {{ post.created_at|date:"M d, Y" }}</p>
            <p>{{ post.content|truncatewords:30 }}</p>
            <a href="{% url 'post_detail' post.id %}">Read more</a>
        </article>
    {% empty %}
        <p>No posts yet.</p>
    {% endfor %}
{% endblock %}
[Django templates: loop over posts, display with filters and URLs]
💡 Django's template filters (|date, |truncatewords) transform data before display — no need to preprocess in views.

🔐 Django Admin

Django's built‑in admin panel is one of its killer features — automatically generated CRUD interfaces for models.

# admin.py
from django.contrib import admin
from .models import Post, Comment

class PostAdmin(admin.ModelAdmin):
    list_display = ("title", "author", "created_at", "published")
    list_filter = ("published", "created_at")
    search_fields = ("title", "content")
    prepopulated_fields = {"slug": ("title",)}
    date_hierarchy = "created_at"

class CommentAdmin(admin.ModelAdmin):
    list_display = ("post", "author", "created_at")
    list_filter = ("created_at",)

admin.site.register(Post, PostAdmin)
admin.site.register(Comment, CommentAdmin)
[Django admin: custom list_display, filters, search, and prepopulated fields]
💡 admin.site.register() makes models visible in the admin panel. Customize with ModelAdmin subclasses.

Full‑Stack Integration

Connecting frontend and backend — API calls, WebSockets, and server‑side rendering

🔗 API Integration — Frontend & Backend

Modern web apps separate frontend and backend. The frontend fetches data from the backend API and renders it dynamically.

# backend: Flask API
@app.route("/api/posts")
def api_posts():
    posts = [
        {"id": 1, "title": "First Post", "content": "Hello world!"},
        {"id": 2, "title": "Second Post", "content": "Learning web dev"}
    ]
    return jsonify(posts)

# frontend: JavaScript
async function loadPosts() {
    const response = await fetch("/api/posts");
    const posts = await response.json();
    const container = document.getElementById("posts");
    posts.forEach(post => {
        const div = document.createElement("div");
        div.innerHTML = `<h3>${post.title}</h3><p>${post.content}</p>`;
        container.appendChild(div);
    });
}
loadPosts();
[API integration: Flask backend + JavaScript frontend fetching posts]
💡 This pattern (separate frontend and backend) is the foundation of modern SPAs (Single Page Applications).

🔌 WebSockets with Python

WebSockets enable real‑time communication — chat apps, live notifications, and collaborative tools.

# FastAPI WebSocket example
from fastapi import FastAPI, WebSocket

app = FastAPI()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    try:
        while True:
            data = await websocket.receive_text()
            await websocket.send_text(f"Server: {data}")
    except:
        await websocket.close()

// JavaScript client
const ws = new WebSocket("ws://localhost:8000/ws");
ws.onopen = () => console.log("Connected");
ws.onmessage = (event) => console.log("Received:", event.data);
ws.send("Hello from browser!");
[WebSocket server and client: real-time bidirectional communication]
💡 WebSockets are ideal for real‑time features — chat, live notifications, and collaborative editing.

Web Development FAQ

Quick answers to common web development questions

What's the difference between frontend and backend?

+

Frontend is what users see and interact with — HTML, CSS, JavaScript. Backend handles data processing, authentication, databases, and business logic — Python with Flask, Django, or FastAPI.

Do I need JavaScript to build web apps?

+

You can build web apps with Python alone using server‑side rendering (Flask/Django templates). But modern web apps use JavaScript on the frontend for better interactivity and dynamic content.

Should I learn Flask or Django first?

+

Start with Flask — it's minimal and teaches you core concepts. Django is more powerful but has a steeper learning curve. Both are valuable for different projects.

What is a full‑stack developer?

+

A full‑stack developer works on both frontend (HTML, CSS, JavaScript) and backend (Python, databases, servers). They can build complete web applications from end to end.

How do I deploy my web app?

+

Use platforms like Render, Railway, or Vercel for easy deployment. For more control, use a VPS (DigitalOcean, AWS) with Gunicorn and Nginx.
