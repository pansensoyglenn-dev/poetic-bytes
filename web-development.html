📜 History of Web Development

From Static Pages to Dynamic Full‑Stack Applications

Web development has evolved from simple static pages to complex, interactive applications. Understanding this evolution helps you appreciate the tools you're about to master.

1991 — The First Website

Tim Berners‑Lee launches the first website — a simple text‑based page. HTML is born.

1995 — CSS and JavaScript

CSS introduces styling and layout. JavaScript adds interactivity to web pages. The modern web begins.

1997 — Dynamic Content

PHP, ASP, and JSP enable server‑side scripting, allowing websites to generate dynamic content from databases.

2005 — Web 2.0

AJAX enables asynchronous requests, making web applications more responsive. Gmail and Google Maps lead the way.

2010 — Frontend Frameworks

Angular, React, and Vue.js revolutionise frontend development, enabling single‑page applications (SPAs).

2015 — Full‑Stack JavaScript & Python

Node.js brings JavaScript to the server. Python's Flask and Django dominate backend development for data‑driven apps.

Today — Modern Full‑Stack

Developers use a combination of Python backends, modern frontend frameworks, and cloud infrastructure to build scalable, feature‑rich applications.

🖌️ HTML Fundamentals — The Structure of the Web

HTML (HyperText Markup Language) is the foundation of every web page. Learn to structure content with semantic HTML.

🖌️ 2.1 HTML Tags, Elements, and Attributes

HTML uses tags to define elements. Each element has a purpose — headings, paragraphs, links, images, lists, and more.

Step-by-step explanation:

  • Step 1: Start with the <!DOCTYPE html> declaration.
  • Step 2: Use the <html> element as the root.
  • Step 3: Add a <head> with metadata (title, charset, viewport).
  • Step 4: Add a <body> with the visible content.
  • Step 5: Use semantic tags: <header>, <nav>, <main>, <section>, <article>, <footer>.
html 
<!DOCTYPE html>

⚡ JavaScript — Making Your Website Interactive

JavaScript adds interactivity to web pages — handling clicks, form submissions, API calls, and dynamic content updates.

⚡ 4.1 JavaScript DOM Manipulation and Events

JavaScript can access and modify the Document Object Model (DOM), reacting to user actions and updating the page in real time.

Step-by-step explanation:

  • Step 1: Select elements with document.querySelector() or document.getElementById().
  • Step 2: Add event listeners with addEventListener().
  • Step 3: Manipulate the DOM: change text, styles, or add/remove elements.
  • Step 4: Use fetch() to make API calls.
  • Step 5: Handle asynchronous operations with async/await.
html + javascript 
<!-- Add this to your HTML body -->
<div id="app">
    <h2>Interactive Counter</h2>
    <p id="counter">0</p>
    <button id="incrementBtn">+1</button>
    <button id="resetBtn">Reset</button>
    <hr>
    <input type="text" id="nameInput" placeholder="Enter your name">
    <button id="greetBtn">Greet</button>
    <p id="greeting"></p>
</div>

<script>
    // Select elements
    const counterEl = document.getElementById('counter');
    const incrementBtn = document.getElementById('incrementBtn');
    const resetBtn = document.getElementById('resetBtn');
    const nameInput = document.getElementById('nameInput');
    const greetBtn = document.getElementById('greetBtn');
    const greetingEl = document.getElementById('greeting');

    let count = 0;

    // Event listeners
    incrementBtn.addEventListener('click', function() {
        count++;
        counterEl.textContent = count;
    });

    resetBtn.addEventListener('click', function() {
        count = 0;
        counterEl.textContent = count;
    });

    greetBtn.addEventListener('click', function() {
        const name = nameInput.value.trim();
        if (name) {
            greetingEl.textContent = `Hello, ${name}! Welcome to interactive web development.`;
        } else {
            greetingEl.textContent = 'Please enter your name.';
        }
    });

    // Keyboard support
    nameInput.addEventListener('keypress', function(event) {
        if (event.key === 'Enter') {
            greetBtn.click();
        }
    });

    console.log('JavaScript is running!');
</script>
JavaScript is the language of the browser. With modern JavaScript (ES6+), you can build complex, interactive applications entirely on the frontend.

🧪 Flask Templates — Jinja2

Flask uses Jinja2 as its templating engine. Templates allow you to generate dynamic HTML with Python variables and logic.

🧪 5.1 Jinja2 Templates in Flask

Jinja2 templates use special syntax: {{ variable }} for output, {% for %} for loops, and {% if %} for conditionals.

Step-by-step explanation:

  • Step 1: Create a templates folder in your Flask project.
  • Step 2: Create HTML templates with Jinja2 syntax.
  • Step 3: Use render_template() to render templates with context variables.
  • Step 4: Use template inheritance with {% extends %} and {% block %}.
  • Step 5: Use filters like | length, | join, | safe.
python + jinja2 
# Flask application with templates
from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def home():
    return render_template("index.html", title="Home")

@app.route("/users")
def users():
    users_data = [
        {"name": "Alice", "age": 30, "city": "NYC"},
        {"name": "Bob", "age": 25, "city": "LA"},
        {"name": "Charlie", "age": 35, "city": "Chicago"}
    ]
    return render_template("users.html", title="Users", users=users_data)

if __name__ == "__main__":
    app.run(debug=True)

# templates/base.html
'''
<!DOCTYPE html>
<html>
<head>
    <title>{{ title }} | My Flask App</title>
    <link rel="stylesheet" href="/static/style.css">
</head>
<body>
    <header>
        <h1>My Flask App</h1>
        <nav>
            <a href="/">Home</a>
            <a href="/users">Users</a>
        </nav>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2026 Flask App</p>
    </footer>
</body>
</html>
'''

# templates/index.html
'''
{% extends "base.html" %}

{% block content %}
<h2>Welcome to the Flask App!</h2>
<p>This page is rendered using Jinja2 templates.</p>
<p>Current time: {{ now }}</p>
{% endblock %}
'''

# templates/users.html
'''
{% extends "base.html" %}

{% block content %}
<h2>User List</h2>
<p>Total users: {{ users|length }}</p>
<ul>
    {% for user in users %}
    <li>{{ user.name }} - {{ user.age }} years old ({{ user.city }})</li>
    {% endfor %}
</ul>
{% if users|length == 0 %}
<p>No users found.</p>
{% endif %}
{% endblock %}
'''
Template inheritance is a powerful feature of Jinja2. It lets you define a base template with common structure and extend it in child templates for specific content.

🏗️ Django Templates — Powerful Templating

Django has its own templating language with similar concepts to Jinja2, with additional features like template tags and filters.

🏗️ 6.1 Django Template System

Django templates use {{ variable }} for output, {% tag %} for logic, and support inheritance.

Step-by-step explanation:

  • Step 1: Create a templates folder in your Django app.
  • Step 2: Create base templates with {% block %}.
  • Step 3: Extend base templates with {% extends %}.
  • Step 4: Use built‑in tags: {% for %}, {% if %}, {% url %}.
  • Step 5: Use filters: | length, | date, | default.

# myapp/views.py
from django.shortcuts import render

    return render(request, "home.html", {"now": timezone.now()})

        {"name": "Alice", "age": 30, "city": "NYC"},
        {"name": "Bob", "age": 25, "city": "LA"},
    ]
    return render(request, "users.html", {"users": users})

# myapp/urls.py
from django.urls import path

    path("", views.home, name="home"),

]

📁 Static Files — CSS, JavaScript, and Images

Static files are assets that don't change dynamically: CSS, JavaScript, images, and fonts. Learn to serve them in Flask and Django.

📁 7.1 Serving Static Files in Flask and Django

Both Flask and Django have built‑in support for static files. In development, they're served automatically. In production, use a dedicated web server.

Step-by-step explanation:

  • Step 1: In Flask, create a static folder and use url_for('static', filename='...').
  • Step 2: In Django, use {% load static %} and {% static '...' %}.
  • Step 3: Organise static files by type: css/, js/, images/.
  • Step 4: Use a CDN for popular libraries (Bootstrap, jQuery) to improve load times.
  • Step 5: For production, use whitenoise (Flask) or Django's collectstatic.
project structure

📝 Forms and User Input

Forms are how users interact with your web applications — submitting data, signing up, logging in, and more.

📝 8.1 Building and Processing Forms

Learn to build HTML forms, handle GET and POST requests, and validate user input in Flask and Django.

Step-by-step explanation:

  • Step 1: Create an HTML form with <form> and input fields.
  • Step 2: Set method="POST" and action to the endpoint.
  • Step 3: In Flask, use request.form to access submitted data.
  • Step 4: In Django, use request.POST and Django forms.
  • Step 5: Validate data and handle errors gracefully.
python + html 
# Flask form example
from flask import Flask, request, render_template, redirect, url_for

app = Flask(__name__)

@app.route("/contact", methods=["GET", "POST"])
def contact():
    if request.method == "POST":
        name = request.form.get("name")
        email = request.form.get("email")
        message = request.form.get("message")

        if not name or not email or not message:
            return render_template("contact.html", error="All fields are required")

        # Process the data (save to database, send email, etc.)
        print(f"New contact: {name} ({email}) - {message}")
        return render_template("contact.html", success="Thank you for your message!")

    return render_template("contact.html")

🍪 Sessions and Cookies — Managing State

HTTP is stateless — sessions and cookies allow you to maintain state across requests, remembering users and their data.

🍪 9.1 Working with Sessions in Flask and Django

Both Flask and Django have built‑in session support. Sessions store data on the server, while cookies store data on the client.

Step-by-step explanation:

  • Step 1: In Flask, use session from flask with a SECRET_KEY.
  • Step 2: In Django, use request.session (requires django.contrib.sessions).
  • Step 3: Store data: session['key'] = value.
  • Step 4: Retrieve data: value = session.get('key', default).
  • Step 5: Clear sessions: session.clear() or request.session.flush().
python 
# Flask session example
from flask import Flask, session, render_template, request, redirect, url_for

app = Flask(__name__)
app.secret_key = "your-secret-key-here"  # Required for sessions

@app.route("/")
def home():
    # Get visit count from session
    visits = session.get("visits", 0)
    visits += 1
    session["visits"] = visits
    return render_template("home.html", visits=visits)

@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        username = request.form.get("username")
        # In a real app, verify credentials
        session["user"] = username
        return redirect(url_for("profile"))
    return render_template("login.html")

@app.route("/profile")
def profile():
    user = session.get("user")
    if not user:
        return redirect(url_for("login"))
    return render_template("profile.html", user=user)

@app.route("/logout")
def logout():
    session.clear()
    return redirect(url_for("home"))

🔌 10.1 Fetching Data from APIs in Flask and Django

Use requests or httpx to make API calls from your backend. You can also use JavaScript fetch() on the frontend.

Step-by-step explanation:

  • Step 1: Install requests: pip install requests.
  • Step 2: Use requests.get(url) to fetch data from an API.
  • Step 3: Handle errors and timeouts.
  • Step 4: Parse JSON responses with .json().
  • Step 5: Use the data in your templates.
python 
# Flask API integration example
from flask import Flask, render_template
import requests

app = Flask(__name__)

@app.route("/weather")
def weather():
    # Example: fetch weather data from a public API
    api_key = "your-api-key"  # Get from environment variable in production
    city = "London"
    url = f"http://api.openweathermap.org/data/2.5/weather?q={city}&appid={api_key}&units=metric"

    try:
        response = requests.get(url, timeout=5)
        response.raise_for_status()
        data = response.json()
        weather_data = {
            "city": city,
            "temperature": data["main"]["temp"],
            "description": data["weather"][0]["description"],
            "humidity": data["main"]["humidity"],
            "icon": data["weather"][0]["icon"]
        }
        return render_template("weather.html", weather=weather_data, error=None)
    except requests.RequestException as e:
        return render_template("weather.html", weather=None, error="Failed to fetch weather data")

    <p>Conditions: {{ weather.description }}</p>

⚛️ Frontend Frameworks — Building Modern UIs

Frontend frameworks like Vue.js, React, and Alpine.js help you build interactive, component‑based user interfaces.

⚛️ 11.1 Vue.js with a Python Backend

Vue.js is a progressive framework that can be easily integrated with any Python backend. It's perfect for building single‑page applications (SPAs).

Step-by-step explanation:

  • Step 1: Include Vue.js via CDN: <script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>.
  • Step 2: Create a Vue app with new Vue() and bind data.
  • Step 3: Use directives: v-bind, v-for, v-if, v-on.
  • Step 4: Use fetch() to call your Python API from Vue.
  • Step 5: Render dynamic content with {{ }}.
html + vue.js 
<!-- Vue.js single-page application with Flask backend -->
<!DOCTYPE html>
<html>
<head>
    <title>Vue + Flask</title>
    <script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
    <style>
        body { font-family: 'Segoe UI', sans-serif; max-width: 800px; margin: 0 auto; padding: 20px; }
        .user-card { border: 1px solid #ddd; padding: 15px; margin: 10px 0; border-radius: 8px; }
        .btn { padding: 8px 16px; border: none; border-radius: 4px; cursor: pointer; }
        .btn-primary { background: #2c3e50; color: white; }
        .btn-danger { background: #e74c3c; color: white; }
        input, select { padding: 8px; margin: 5px; border: 1px solid #ddd; border-radius: 4px; }
    </style>
</head>
<body>
    <div id="app">
        <h1>User Management (Vue + Flask)</h1>

        <!-- Form -->
        <div>
            <h3>Add User</h3>
            <input v-model="newUser.name" placeholder="Name">
            <input v-model="newUser.age" placeholder="Age" type="number">
            <input v-model="newUser.city" placeholder="City">
            <button @click="addUser" class="btn btn-primary">Add</button>
        </div>

        <!-- User list -->
        <div>
            <h3>Users ({{ users.length }})</h3>
            <div v-for="(user, index) in users" :key="user.id" class="user-card">
                <strong>{{ user.name }}</strong> - {{ user.age }} years old ({{ user.city }})
                <button @click="deleteUser(index)" class="btn btn-danger">Delete</button>
            </div>
            <div v-if="users.length === 0">No users yet.</div>
        </div>

        <!-- Status message -->
        <div v-if="message" :style="{ color: messageType === 'error' ? 'red' : 'green' }">
            {{ message }}
        </div>
    </div>

    <script>
        new Vue({
            el: '#app',
            data: {
                users: [],
                newUser: { name: '', age: '', city: '' },
                message: '',
                messageType: ''
            },
            mounted() {
                this.fetchUsers();
            },
            methods: {
                async fetchUsers() {
                    try {
                        const response = await fetch('/api/users');
                        if (response.ok) {
                            this.users = await response.json();
                        }
                    } catch (error) {
                        this.showMessage('Error fetching users', 'error');
                    }
                },
                async addUser() {
                    if (!this.newUser.name || !this.newUser.age || !this.newUser.city) {
                        this.showMessage('All fields are required', 'error');
                        return;
                    }
                    try {
                        const response = await fetch('/api/users', {
                            method: 'POST',
                            headers: { 'Content-Type': 'application/json' },
                            body: JSON.stringify(this.newUser)
                        });
                        if (response.ok) {
                            const user = await response.json();
                            this.users.push(user);
                            this.newUser = { name: '', age: '', city: '' };
                            this.showMessage('User added successfully', 'success');
                        }
                    } catch (error) {
                        this.showMessage('Error adding user', 'error');
                    }
                },
                async deleteUser(index) {
                    const user = this.users[index];
                    try {
                        const response = await fetch(`/api/users/${user.id}`, {
                            method: 'DELETE'
                        });
                        if (response.ok) {
                            this.users.splice(index, 1);
                            this.showMessage('User deleted', 'success');
                        }
                    } catch (error) {
                        this.showMessage('Error deleting user', 'error');
                    }
                },
                showMessage(msg, type) {
                    this.message = msg;
                    this.messageType = type;
                    setTimeout(() => { this.message = ''; }, 3000);
                }
            }
        });
    </script>
</body>
</html>
Vue.js is a progressive framework — you can start with a simple CDN script and gradually build up to a full single‑page application. It pairs beautifully with Python backends.

❓ Frequently Asked Questions — Web Development

Common questions about Python web development, answered.

Flask or Django — which should I choose?

+

Flask is lightweight and flexible — great for microservices and APIs. Django is "batteries‑included" — great for full‑featured web applications with admin interfaces, ORM, and authentication built in.

Do I need to know JavaScript to do web development?

+

Yes, JavaScript is essential for frontend interactivity. However, you can start with Python backend and add JavaScript as you progress. This curriculum covers both.

What's the difference between frontend and backend?

+

Frontend is what users see and interact with (HTML, CSS, JavaScript in the browser). Backend is the server‑side logic (Python, databases, APIs) that processes requests and serves data.

How do I handle user authentication?

+

Both Flask and Django have authentication libraries. Flask uses Flask‑Login or JWT. Django has built‑in authentication with django.contrib.auth. Always use HTTPS in production.

What's a REST API?

+

A REST API is a web service that uses HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources. It's the standard way for frontend and backend to communicate.

How do I deploy my web app?

+

Use platforms like Render, Heroku, or Railway. They handle server management, SSL, and scaling. For more control, use VPS providers like DigitalOcean or AWS with Nginx and Gunicorn.
