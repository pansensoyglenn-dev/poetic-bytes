📖 What Is a Python Function?

A function is a reusable block of code that performs a specific task. Functions are the fundamental building blocks of modular, maintainable Python programs — they allow you to write once, call many times, and they are the primary mechanism for abstraction, reuse, and organisation in Python.

🔤 Function Basics – Definition, Calls, and Return

The foundation everything else in this tutorial builds on

📝 Defining and Calling Functions

The def Keyword and Function Calls

A function is defined with the def keyword, followed by the function name, parentheses, and a colon. The function body is indented. A function is called by using its name followed by parentheses.

# Example 1: defining a simple function
def say_hello():
    print("Hello, world!")

# Example 2: calling the function
say_hello()

# Example 3: functions with parameters
def greet(name):
    print(f"Hello, {name}!")

greet("Glenn")

# Example 4: functions with a return value
def add(a, b):
    return a + b

result = add(3, 5)
print(result)
🔍 Why this code is used: Defining and calling functions is the most fundamental operation in Python — every program uses functions.

⚙️ How it operates: def creates a function object and assigns it to the function name. The code inside the function body runs only when the function is called. Parameters are variables that receive arguments when the function is called.

📊 How it affects the program: Example 4 shows that a function can return a value using the return statement — the caller can then use that value.

🎯 How it stimulates the outcome: Functions without an explicit return statement return None by default — a subtle but important point covered later.

📌 Significance: The def statement is the primary mechanism for code organisation in Python — everything else is built on top of it.
📤 Return Values

The return Statement and None

# Example 5: returning a single value
def square(n):
    return n ** 2

print(square(5))

# Example 6: returning multiple values as a tuple
def min_max(numbers):
    return min(numbers), max(numbers)

low, high = min_max([1, 2, 3, 4, 5])
print(f"min: {low}, max: {high}")

# Example 7: functions return None by default
def no_return():
    print("I don't return anything")

result = no_return()
print(result)  # None

# Example 8: early return
def is_even(n):
    if n % 2 == 0:
        return True
    return False

print(is_even(4))
print(is_even(5))
🔍 Why this code is used: Functions return values to communicate results back to the caller — this is how data flows through your program.

⚙️ How it operates: The return statement exits the function and sends a value back to the caller. If no return is encountered, the function returns None. A function can return multiple values as a tuple (Example 6), which the caller can unpack.

📊 How it affects the program: Example 7 demonstrates that a function without a return statement still returns something — None. This is a frequent source of confusion for beginners.

🎯 How it stimulates the outcome: Example 8 shows that return can be used early — it exits the function immediately, which is useful for guard clauses and early exits.

📌 Significance: The return statement is how functions communicate — mastering it is essential for writing useful functions.
📝 Docstrings

Documenting Functions with Docstrings

# Example 9: a function with a docstring
def calculate_area(length: float, width: float) -> float:
    """Calculate the area of a rectangle.

    Args:
        length: The length of the rectangle.
        width: The width of the rectangle.

    Returns:
        The area of the rectangle (length * width).
    """
    return length * width

print(calculate_area(5, 3))
print(calculate_area.__doc__)
🔍 Why this code is used: Docstrings are the standard way to document functions — they are read by help(), IDEs, and documentation generators.

⚙️ How it operates: A docstring is a string literal that appears as the first statement in a function. It is stored in the function's __doc__ attribute and is displayed by help().

📊 How it affects the program: Example 9 shows that the docstring is accessible via calculate_area.__doc__ — this is how IDEs show function documentation.

🎯 How it stimulates the outcome: Writing docstrings is a professional habit — it makes your code more usable and maintainable, especially in team settings.

📌 Significance: Docstrings are the primary documentation mechanism in Python — they are not optional in professional code.

📥 Function Parameters — Positional, Keyword, Default, *args, **kwargs

Every way to pass data into a function

📥 Positional and Keyword Arguments

Positional, Keyword, and Default Parameters

# Example 10: positional arguments — order matters
def describe_person(name, age, city):
    return f"{name} is {age} years old and lives in {city}"

print(describe_person("Glenn", 30, "Diplahan"))

# Example 11: keyword arguments — order doesn't matter
print(describe_person(age=30, city="Diplahan", name="Glenn"))

# Example 12: default parameters
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Ana"))
print(greet("Bob", "Hola"))

# Example 13: default parameters must come after non-default
# def bad(name="Glenn", age):  # SyntaxError
#     pass

def good(age, name="Glenn"):
    pass
🔍 Why this code is used: Parameters are how functions receive data — understanding the different types gives you flexibility in how you call functions.

⚙️ How it operates: Positional arguments are matched by position — the first argument goes to the first parameter. Keyword arguments are matched by name — order doesn't matter. Default parameters provide a fallback value if the caller doesn't supply one.

📊 How it affects the program: Example 11 shows that keyword arguments make function calls more readable and self-documenting — describe_person(age=30, city="Diplahan", name="Glenn") is clearer than positional ordering.

🎯 How it stimulates the outcome: Default parameters are ideal for optional values — they let you call a function with fewer arguments when the defaults are sufficient.

📌 Significance: Mastering positional, keyword, and default parameters gives you control over how functions are called and how they behave.
⭐ *args and **kwargs

Variable-Length Arguments

# Example 14: *args collects extra positional arguments as a tuple
def sum_all(*args):
    return sum(args)

print(sum_all(1, 2, 3, 4, 5))

# Example 15: **kwargs collects extra keyword arguments as a dict
def print_kwargs(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_kwargs(name="Glenn", age=30, city="Diplahan")

# Example 16: combining positional, *args, and **kwargs
def flexible(required, *args, **kwargs):
    print(f"Required: {required}")
    print(f"Extra args: {args}")
    print(f"Extra kwargs: {kwargs}")

flexible("must have", 1, 2, 3, name="Glenn", age=30)

# Example 17: unpacking a list into *args
numbers = [1, 2, 3, 4, 5]
print(sum_all(*numbers))

# Example 18: unpacking a dict into **kwargs
data = {"name": "Glenn", "age": 30}
print_kwargs(**data)
🔍 Why this code is used: *args and **kwargs make functions flexible — they can accept any number of arguments, which is essential for decorators, wrappers, and generic functions.

⚙️ How it operates: *args collects any extra positional arguments into a tuple. **kwargs collects any extra keyword arguments into a dictionary. The names args and kwargs are conventions, not keywords — you could use *things and **stuff.

📊 How it affects the program: Example 16 shows the complete signature pattern — required parameters, then *args, then **kwargs.

🎯 How it stimulates the outcome: Examples 17 and 18 show the "unpacking" pattern — using * and ** to pass a list or dictionary as arguments to a function.

📌 Significance: *args and **kwargs are used heavily in libraries and frameworks — they enable the flexible, extensible APIs that Python is known for.
🔀 Keyword-Only and Positional-Only Parameters

Python 3.8+ Parameter Syntax

# Example 19: positional-only parameters using /
def pos_only(a, b, /, c):
    return a + b + c

print(pos_only(1, 2, 3))
# print(pos_only(1, 2, c=3))  # Valid
# print(pos_only(a=1, b=2, c=3))  # TypeError — a and b are positional-only

# Example 20: keyword-only parameters using *
def kw_only(a, *, b, c):
    return a + b + c

print(kw_only(1, b=2, c=3))
# print(kw_only(1, 2, 3))  # TypeError — b and c are keyword-only

# Example 21: combined positional-only and keyword-only
def combined(a, b, /, c, *, d, e):
    return a + b + c + d + e

print(combined(1, 2, 3, d=4, e=5))
🔍 Why this code is used: Positional-only and keyword-only parameters give you fine-grained control over how functions can be called — they're useful for API design and preventing ambiguity.

⚙️ How it operates: Parameters before / are positional-only — they cannot be passed as keywords. Parameters after * are keyword-only — they cannot be passed positionally.

📊 How it affects the program: Example 19 shows that a and b are positional-only — calling pos_only(a=1, b=2, c=3) raises TypeError.

🎯 How it stimulates the outcome: Positional-only parameters are used in the Python standard library to maintain backward compatibility — they allow parameter names to be changed without breaking existing code.

📌 Significance: These syntax features give API designers precise control over function interfaces — they're a sign of a mature, carefully designed language.

🌐 Variable Scope — The LEGB Rule

Understanding where variables are defined and accessed

🌐 LEGB Rule

Local, Enclosing, Global, Built-in

Python resolves variable names in a specific order: Local → Enclosing → Global → Built-in. Understanding this order prevents confusing UnboundLocalError bugs.

# Example 22: local scope
def local_example():
    x = 10  # local variable
    print(x)

local_example()
# print(x)  # NameError — x is not defined globally

# Example 23: global scope
y = 20  # global variable

def global_example():
    print(y)  # reads the global variable

global_example()

# Example 24: modifying a global variable with 'global'
z = 30

def modify_global():
    global z
    z = 40

modify_global()
print(z)

# Example 25: enclosing scope (closure)
def outer():
    outer_var = "outer"

    def inner():
        print(outer_var)  # accesses the enclosing scope

    inner()

outer()

# Example 26: modifying an enclosing variable with 'nonlocal'
def counter():
    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

inc = counter()
print(inc())
print(inc())
🔍 Why this code is used: Scope determines where variables are visible — understanding it prevents a whole category of bugs.

⚙️ How it operates: Python looks up variable names in this order: Local (inside the current function), Enclosing (in outer functions), Global (at the module level), Built-in (Python's built-in names). The global keyword lets you modify a global variable from inside a function. The nonlocal keyword lets you modify a variable in an enclosing (but not global) scope.

📊 How it affects the program: Example 22 shows that variables defined inside a function are not visible outside it. Example 24 shows that you need global to modify a global variable.

🎯 How it stimulates the outcome: The nonlocal pattern in Example 26 is used to create function factories and closures — a powerful functional programming technique.

📌 Significance: The LEGB rule is one of the most important concepts in Python — it explains why variables behave the way they do in different contexts.

λ Lambda Functions — Anonymous Functions

Concise, single-expression functions for callbacks and functional programming

λ Lambda Basics

lambda — Functions Without a Name

A lambda function is a small, anonymous function defined with the lambda keyword. It can have any number of parameters but only one expression — the result of which is returned automatically.

# Example 27: a simple lambda
square = lambda x: x ** 2
print(square(5))

# Example 28: lambda with multiple parameters
add = lambda a, b: a + b
print(add(3, 5))

# Example 29: lambda used with map()
numbers = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, numbers))
print(doubled)

# Example 30: lambda used with filter()
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(evens)

# Example 31: lambda used with sorted() key
people = [{"name": "Bob", "age": 30}, {"name": "Ana", "age": 25}]
sorted_by_age = sorted(people, key=lambda p: p["age"])
print(sorted_by_age)
🔍 Why this code is used: Lambdas are ideal for short, single-use functions — especially as callbacks for map(), filter(), sorted(), and similar functions.

⚙️ How it operates: lambda args: expression creates a function object. The expression is evaluated and returned when the lambda is called. Lambdas are restricted to a single expression — they cannot contain statements or multiple lines.

📊 How it affects the program: Example 29 shows that map() with a lambda is more concise than defining a separate function for doubling numbers.

🎯 How it stimulates the outcome: The sorted() example (31) is a common real-world use — sorting a list of dictionaries by a specific key using a lambda.

📌 Significance: Lambdas are a key feature of functional programming in Python — they enable a more concise, expressive style for certain operations.

🔄 Recursion — Functions That Call Themselves

Elegant problem-solving for naturally recursive problems

🔄 Recursion Basics

Base Cases and Recursive Cases

A recursive function is a function that calls itself. Every recursive function must have a base case that stops the recursion, and a recursive case that calls itself with a smaller or simpler input.

# Example 32: factorial with recursion
def factorial(n):
    if n <= 1:
        return 1  # base case
    return n * factorial(n - 1)  # recursive case

print(factorial(5))

# Example 33: Fibonacci with recursion
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(10))

# Example 34: recursion with memoization for performance
def fib_memo(n, memo={}):
    if n <= 1:
        return n
    if n not in memo:
        memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]

print(fib_memo(50))

# Example 35: recursion depth limit
def deep(n):
    if n == 0:
        return "done"
    return deep(n - 1)

# print(deep(10000))  # RecursionError: maximum recursion depth exceeded
print(deep(1000))  # works
🔍 Why this code is used: Recursion is the natural way to solve problems that have a recursive structure — tree traversal, factorial, Fibonacci, and many mathematical and algorithmic problems.

⚙️ How it operates: Each recursive call creates a new stack frame. The base case stops the recursion and begins returning values back up the call stack. Python has a default recursion limit of 1000 — deeper recursion raises RecursionError.

📊 How it affects the program: Example 34 shows memoization — caching results to avoid exponential time complexity. Without memoization, fibonacci(50) would take an impossibly long time.

🎯 How it stimulates the outcome: Recursion is elegant but can be memory-intensive — for very deep recursion, an iterative solution is often better.

📌 Significance: Recursion is a fundamental technique in computer science — mastering it opens up a whole new way of thinking about problems.

✨ Decorators — Wrapping and Extending Functions

One of Python's most powerful and elegant features

✨ Decorator Basics

Functions that Modify Other Functions

A decorator is a function that takes another function as an argument and returns a new function that wraps the original. This is a powerful pattern for adding behaviour (logging, timing, authentication, caching) without modifying the original function's code.

# Example 36: a simple decorator
def uppercase_decorator(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result.upper()
    return wrapper

@uppercase_decorator
def greet(name):
    return f"Hello, {name}"

print(greet("Glenn"))

# Example 37: a timing decorator
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} took {end - start:.6f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(0.1)
    return "done"

slow_function()

# Example 38: decorators with arguments (factory)
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def say_hello():
    print("Hello!")

say_hello()

# Example 39: preserving metadata with functools.wraps
import functools

def log(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log
def add(a, b):
    """Add two numbers."""
    return a + b

print(add.__name__)
print(add.__doc__)
🔍 Why this code is used: Decorators are used extensively in frameworks (Flask routes, Django views, pytest fixtures) and for cross-cutting concerns like logging, timing, authentication, and caching.

⚙️ How it operates: @decorator is syntactic sugar for func = decorator(func). The decorator function takes func as an argument and returns a new function that wraps it. Decorator factories (Example 38) are functions that return decorators, allowing parameters to be passed to the decorator.

📊 How it affects the program: Without functools.wraps (Example 39), the wrapper function would lose the original function's name and docstring — wraps preserves this metadata.

🎯 How it stimulates the outcome: Example 37's timer decorator is a real-world pattern — it adds timing behaviour to any function without modifying its code.

📌 Significance: Decorators are one of Python's most distinctive and powerful features — they enable clean, reusable behaviour modification across your codebase.

⏱️ Project 1 – Function Timer Decorator

Measuring Function Performance with a Decorator

Demonstrates decorator creation, *args/**kwargs, and timing — a practical tool for performance profiling.

import time
import functools

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} took {end - start:.6f}s")
        return result
    return wrapper

@timer
def compute(n):
    return sum(i ** 2 for i in range(n))

print("Result:", compute(1000000))
🔍 Why this code is used: Timing functions is a common need in performance optimisation — this decorator makes it reusable.

⚙️ How it operates: The timer decorator wraps any function, measuring and printing the elapsed time. functools.wraps preserves the original function's metadata.

📊 How it affects the program: The decorator can be applied to any function with a simple @timer annotation — no code duplication.

🎯 How it stimulates the outcome: This pattern scales to real performance profiling — you can time specific operations without cluttering your business logic.

📌 Significance: This is a real-world use of decorators — adding cross-cutting behaviour cleanly and reusably.
📝 Project 2 – Input Validation with *args and **kwargs

Flexible Input Validation with Variable-Length Arguments

def validate_user(required_fields, *args, **kwargs):
    """Validate that all required fields are present in kwargs."""
    missing = [field for field in required_fields if field not in kwargs]
    if missing:
        raise ValueError(f"Missing required fields: {missing}")
    if args:
        print(f"Extra positional arguments ignored: {args}")
    return {field: kwargs[field] for field in required_fields}

# Example 40: valid call
result = validate_user(["name", "email"], name="Glenn", email="g@example.com")
print(result)

# Example 41: missing field
try:
    result = validate_user(["name", "email"], name="Glenn")
except ValueError as e:
    print(f"Error: {e}")
🔍 Why this code is used: Validation of function inputs is a common need — this pattern shows how to validate keyword arguments flexibly.

⚙️ How it operates: The function takes a list of required fields and uses **kwargs to collect all keyword arguments. It checks which required fields are missing and raises an error.

📊 How it affects the program: This pattern is useful for API endpoints, configuration validation, and any function that receives structured data.

🎯 How it stimulates the outcome: The function ignores extra positional arguments (collected in *args) — this is a clean way to handle unexpected input.

📌 Significance: This pattern demonstrates how *args and **kwargs enable flexible, robust function interfaces.
🔁 Project 3 – Memoization with a Decorator

Caching Expensive Function Results

import functools

def memoize(func):
    cache = {}
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        key = (args, tuple(kwargs.items()))
        if key not in cache:
            cache[key] = func(*args, **kwargs)
        return cache[key]
    return wrapper

@memoize
def expensive(n):
    """A computationally expensive function."""
    print(f"Computing expensive({n})...")
    return sum(i ** 2 for i in range(n))

print(expensive(1000))
print(expensive(1000))  # cached — no computation
🔍 Why this code is used: Memoization caches the results of expensive function calls — a classic optimisation technique.

⚙️ How it operates: The memoize decorator stores results in a dictionary keyed by the function arguments. If the same arguments are passed again, the cached result is returned immediately.

📊 How it affects the program: The expensive function is only called once for each unique set of arguments — subsequent calls are instant.

🎯 How it stimulates the outcome: Python's built-in functools.lru_cache provides this functionality — this example shows how it works under the hood.

📌 Significance: Memoization is one of the most common uses of decorators in real-world Python code.

❓ Frequently Asked Questions

Common questions about Python functions, answered

What is the difference between a parameter and an argument?

+

A parameter is a variable in the function definition — it's a placeholder for the value that will be passed. An argument is the actual value that is passed to the function when it's called. In def greet(name):, name is a parameter. In greet("Glenn"), "Glenn" is an argument.

What does return do in a function?

+

The return statement exits the function and sends a value back to the caller. If no return is used, the function returns None. A function can have multiple return statements (e.g., for early exits), but only one is executed per call.

What is *args and **kwargs?

+

*args collects extra positional arguments into a tuple. **kwargs collects extra keyword arguments into a dictionary. They make functions flexible — able to accept any number of arguments. The names args and kwargs are conventions, not keywords.

What is the LEGB rule?

+

LEGB stands for Local, Enclosing, Global, Built-in — it's the order in which Python looks up variable names. Python checks the local scope first, then any enclosing (outer) scopes, then the global scope, and finally the built-in scope.

What is a lambda function?

+

A lambda is a small, anonymous function defined with lambda. It can have any number of parameters but only one expression. Lambdas are ideal for short, single-use callbacks in map(), filter(), and sorted().

What is recursion in Python?

+

Recursion is when a function calls itself. It's useful for problems with a recursive structure (tree traversal, factorial, Fibonacci). Every recursive function needs a base case to stop recursion and a recursive case that moves toward the base case.

What is a decorator and why would I use one?

+

A decorator is a function that wraps another function, adding behaviour without modifying its code. Use decorators for cross-cutting concerns like logging, timing, authentication, and caching. They're a key feature in frameworks like Flask, Django, and pytest.

What is the difference between global and nonlocal?

+

global allows you to modify a variable at the module (global) level from inside a function. nonlocal allows you to modify a variable in an enclosing (but not global) scope — used in nested functions and closures.

What is a docstring?

+

A docstring is a string literal that appears as the first statement in a function, class, or module. It's used for documentation and is accessible via __doc__ and help(). Writing docstrings is a professional practice.

Can a function return multiple values?

+

Yes — a function can return multiple values as a tuple. The caller can unpack the tuple into separate variables: a, b = func(). This is a common Python pattern.

🧪 Exercise 2 – Write a Function Using *args

    # Your code here
    pass

# Test

def product(*args) -> int:
    result = 1
    for n in args:
        if n != 0:
            result *= n
    return result

🧪 Exercise 3 – Write a Recursive Function

    # Your code here
    pass

def sum_to_n(n: int) -> int:
    if n <= 0:
        return 0
    return n + sum_to_n(n - 1)

# Explanation:
# 1. Base case: n <= 0 returns 0
