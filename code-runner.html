✅ What Works in the Browser Runner

Pyodide runs real CPython 3.12 via WebAssembly — here is exactly what you can and cannot do

This runner uses Pyodide — a genuine port of CPython to WebAssembly. It runs real Python 3.12 in your browser, not a simplified simulation. Almost everything that works in a local Python environment works here, with a small set of browser‑sandbox exceptions.

✅ Fully Supported

All of the following work exactly as they do in your local Python 3.12 environment:

  • ✓Full standard library — math, statistics, collections, itertools, functools, json, csv, re, datetime, pathlib, io, abc, dataclasses, typing, enum, copy, random, string, textwrap, hashlib, base64, struct, and 280+ more modules
  • ✓All Python language features — classes, decorators, generators, context managers, async/await (top-level), match statements, type hints, f-strings, walrus operator :=, structural pattern matching
  • ✓Scientific libraries — NumPy, SciPy, Pandas, Matplotlib, scikit‑learn are pre‑bundled in Pyodide and can be imported directly
  • ✓File simulation — use io.StringIO and io.BytesIO as in‑memory file objects. open() reads/writes to a virtual in‑memory filesystem
  • ✓Full tracebacks — complete Python error tracebacks with line numbers
  • ✓micropip — install additional pure‑Python packages at runtime with import micropip; await micropip.install('package-name')

⚠️ Not Available in the Browser Sandbox

  • ✗Network requests — requests, urllib.request, and socket cannot make outbound network calls
  • ✗input() is disabled — interactive keyboard input via input() does not block correctly. Replace with hardcoded values
  • ✗Real filesystem — files live in a virtual in‑memory filesystem and do not persist between sessions. Use io.StringIO
  • ✗subprocess & multiprocessing — spawning new processes is not supported in WebAssembly. asyncio coroutines work fully

📦 Installing Extra Packages with micropip

Pyodide pre‑bundles the most popular data science packages. For additional pure‑Python packages, use micropip.

# micropip — install extra packages at runtime
import micropip
await micropip.install("colorama")                 # pure‑Python ✓
await micropip.install(["rich", "pydantic"])       # install multiple

# Pre‑included — import directly, no install needed:
import numpy as np        # numerical arrays
import pandas as pd       # DataFrames
import scipy               # scientific computing
import sklearn              # machine learning
import matplotlib.pyplot as plt  # plotting
💡 micropip.install() is an async function — you must await it. This works fine because the runner uses top‑level runPythonAsync().

Paste this into the runner and change any value to see what happens:

print(f"Unique    : {unique}")

🐛 Debugging: Add print() to trace your code

When code behaves unexpectedly, add print() at each step to inspect values:

def process(items):
    result = []
    for i, item in enumerate(items):
        print(f"  Step {i}: item={item!r}")    # trace each step
        if isinstance(item, (int, float)) and item > 0:
            result.append(item * 2)
        elif isinstance(item, str):
            result.append(item.upper())
        print(f"         result so far: {result}")
    return result

data = [1, "hello", -3, "world", 7.5, None, 0]
print("Output:", process(data))
💡 Remove debug print() calls once you understand the code, or switch to Python's logging module.

functools

lru_cache, partial, reduce, wraps for higher‑order functions.

❓ Code Runner FAQ

Answers to the most common questions about running Python in the browser

Why does the first run take so long?

+

The first click loads Pyodide — a full Python 3.12 runtime compiled to WebAssembly — from jsDelivr CDN into your browser (~12MB). Every run after the first is near‑instant.

Is my code sent to any server?

+

No. Your code runs entirely inside your browser using WebAssembly. It is never transmitted to any server.

Can I install packages with pip?

+

Not with pip directly. Instead use micropip: import micropip; await micropip.install('package-name'). Popular data science packages (NumPy, Pandas) are pre‑bundled.

Why doesn't input() work?

+

input() requires blocking thread execution, which conflicts with the browser's event loop. Replace calls with hardcoded values in the runner.

Does my code persist between page reloads?

+

Yes — your code is automatically saved to sessionStorage. If you reload the page, the editor content is restored.

Can I run async Python code?

+

Yes. Pyodide executes all code with runPythonAsync(), so top‑level await works without wrapping in asyncio.run().

What Python version is this exactly?

+

Pyodide 0.26.2 ships with CPython 3.12.1. All Python 3.10+ language features work.

How does this compare to other online Python runners?

+

Most online runners execute your code on a remote server. This runner runs locally in your browser — no rate limits, complete privacy, but no outbound networking.
