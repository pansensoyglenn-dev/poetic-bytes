PYTHON NUMBERS: A COMPLETE GUIDE (2026)

1.0 THE THEORETICAL AND PRACTICAL FOUNDATIONS OF NUMERIC TYPES

Numbers are the fundamental data type in all of computing. Python provides a uniquely rich and flexible numeric system that balances performance with correctness. Unlike lower-level languages where integer overflow is a constant threat or floating-point precision is rigidly fixed, Python offers arbitrary-precision integers, hardware-accelerated floats, and specialized modules for exact decimal and rational arithmetic. This comprehensive guide explores every facet of Python's numeric ecosystem from a university-level perspective, blending computer science theory with practical, production-ready patterns.

Understanding Python numbers is not merely about memorizing operators. It requires grasping the distinction between an int (unbounded precision, ideal for counting and cryptography) and a float (IEEE 754 binary64, ideal for scientific computing). It requires knowing when to reach for the decimal module to satisfy financial auditing requirements, or the fractions module to preserve exact rational relationships. This guide will equip you with that deep, actionable knowledge.

1.1 Why This Guide Exists — Our Educational Philosophy

This resource has been designed as a comprehensive educational tool for computer science students, self-directed learners, and professional developers seeking to deepen their understanding of Python's numeric types. Every explanation emphasizes not just the "how" but the "why" behind each concept, connecting abstract mathematical principles (two's complement, IEEE 754, rational approximation) to concrete Python code.

1.2 Prerequisites and Learning Path

This guide assumes familiarity with Python functions, variables, and basic data types. Some exposure to basic algebra is helpful for complex numbers and bitwise operations. If you have not yet studied Python fundamentals, we recommend reviewing the POETIC BYTES Python Basics Guide first.

2.0 INTEGERS (int) — UNBOUNDED PRECISION AND INTERNAL REPRESENTATION

2.1 The Absence of Overflow

In languages like C, C++, or Java, the int type is typically a 32-bit signed integer. Adding 1 to 2,147,483,647 results in an overflow to -2,147,483,648 (silent wraparound) or an exception. Python eliminates this class of bugs entirely. The int type has arbitrary precision. The only limit is the available virtual memory of the system. This makes Python exceptionally well-suited for algorithmic number theory, cryptography (RSA key generation), and large factorial calculations.

Under the hood (CPython), an integer is stored as an array of 30-bit digits (on 64-bit builds) in a C struct called PyLongObject. This is why memory usage grows linearly with the number of digits. Operations like addition are O(n), while multiplication is O(n^1.58) using the Karatsuba algorithm for large numbers.

2.2 Literals, Bases, and Readability

Python 3.6+ supports underscores in numeric literals for visual grouping. You can express integers in binary (0b or 0B), octal (0o or 0O), decimal, or hexadecimal (0x or 0X). The built-in functions bin(), oct(), and hex() return string representations.

💡 Key Distinction: In Python 3, int and long are unified. There is no trailing L suffix. All integers are simply int.

2.3 Boolean Subclass

In a fascinating design choice, bool is a subclass of int. True evaluates to 1 and False to 0 in arithmetic contexts. While convenient for counting (sum(condition for item in list)), it's a historical quirk to be aware of when using strict type checking.

EXAMPLE 2.1 — Integer Literals, Bases, and Arbitrary Precision 
# Integer representations with visual separators
population = 8_200_000_000  # 8.2 billion
binary     = 0b11111101010    # 2026 in binary
hex_color  = 0xFF5733          # Hex color code
permissions = 0o755            # Unix file permissions

print(f"Population: {population:_}")
print(f"Binary {binary} = {bin(binary)}")
print(f"Hex {hex_color} = {hex(hex_color)}")
print(f"Octal {permissions} = {oct(permissions)}")

# Arbitrary precision: 2^1000 has 302 digits
large_power = 2 ** 1000
print(f"\n2**1000 has {len(str(large_power))} digits")
print(f"First 50 digits: {str(large_power)[:50]}...")

# Boolean as subclass of int
print(f"\nTrue + True + False = {True + True + False}")
print(f"issubclass(bool, int): {issubclass(bool, int)}")

# Converting between bases
print(f"\nint('1010', base=2) = {int('1010', 2)}")
print(f"int('FF', base=16) = {int('FF', 16)}")

# Memory usage of integers
import sys
print(f"\nMemory: 0 uses {sys.getsizeof(0)} bytes")
print(f"Memory: 2**1000 uses {sys.getsizeof(2**1000)} bytes")

3.0 FLOATING-POINT NUMBERS (float) — IEEE 754 BINARY64

3.1 The IEEE 754 Double-Precision Standard

Python's float type is implemented as a C double, adhering to the IEEE 754 binary64 standard. This provides 53 bits of mantissa precision (approximately 15-17 decimal digits) and 11 bits of exponent (range ~1e-308 to 1e308). Floats are stored in hardware floating-point units (FPUs), making them extremely fast for vectorized operations (e.g., in NumPy).

The critical educational takeaway is that floats are finite binary approximations of real numbers. Just as 1/3 cannot be represented exactly in decimal (0.33333...), 1/10 cannot be represented exactly in binary (0.0001100110011...). This leads to the classic pitfall: 0.1 + 0.2 != 0.3.

3.2 Mitigating Floating-Point Errors

Never compare floats for exact equality with ==. Instead, use math.isclose(a, b, rel_tol=1e-9, abs_tol=0.0) which checks if the numbers are within a relative or absolute tolerance. For summation of many small numbers, use math.fsum() which uses the Shewchuk algorithm to track partial sums and minimize accumulated error.

3.3 Special Values and Rounding

IEEE 754 defines special bit patterns for infinity (float('inf')), negative infinity (float('-inf')), and Not a Number (float('nan')). Operations involving inf behave as expected in limits. nan is toxic: any arithmetic operation with nan results in nan, and nan == nan evaluates to False. Always use math.isnan() to detect it.

Python's round() uses "Banker's Rounding" (round half to even) as defined in IEEE 754. This minimizes statistical bias in repeated rounding operations, but surprises developers expecting "round half up" taught in elementary school.

EXAMPLE 3.1 — Float Precision, Comparisons, and Special Values 
import math
import sys

# The classic floating-point surprise
a = 0.1
b = 0.2
c = 0.3
print(f"0.1 + 0.2 == 0.3 ? {a + b == c}")
print(f"0.1 + 0.2 actual value: {a + b:.17f}")
print(f"0.3 actual value: {c:.17f}")

# Correct comparison using tolerance
print(f"\nUsing math.isclose: {math.isclose(a + b, c)}")

# Accurate summation with math.fsum
numbers = [0.1] * 10
sum_normal = sum(numbers)
sum_accurate = math.fsum(numbers)
print(f"\nSum of ten 0.1s: normal={sum_normal:.17f}, accurate={sum_accurate:.17f}")

# System float information
print(f"\nFloat info: max={sys.float_info.max}")
print(f"Machine epsilon: {sys.float_info.epsilon}")

# Special values
inf = float('inf')
neg_inf = float('-inf')
nan = float('nan')

print(f"\ninf > 1e308? {inf > 10**308}")
print(f"inf / inf = {inf / inf}")  # nan
print(f"nan == nan? {nan == nan}")
print(f"math.isnan(nan)? {math.isnan(nan)}")

# Banker's Rounding demonstration
print(f"\nround(2.5): {round(2.5)}")
print(f"round(3.5): {round(3.5)}")
print(f"round(4.5): {round(4.5)}")

4.0 COMPLEX NUMBERS — NATIVE SUPPORT FOR IMAGINARY MATH

4.1 Syntax and Internal Representation

Python is one of the few mainstream languages with native syntax for complex numbers. A complex number is written as a + bj (or J), where a is the real part and b is the imaginary part. The internal storage uses a pair of machine-level floats (Py_complex struct), making complex math fast and intuitive.

Access the real and imaginary attributes via .real and .imag (both are read-only floats). The cmath module provides functions for complex trigonometry, logarithms, and polar coordinate conversion. Note that some math operations (like math.sqrt(-1)) raise a ValueError; you must use cmath.sqrt(-1) to get a complex result.

4.2 Polar Coordinates and Euler's Identity

The abs(z) function returns the magnitude (modulus) sqrt(z.real**2 + z.imag**2). cmath.phase(z) returns the argument (angle) in radians, in the range [-π, π]. cmath.rect(r, phi) converts polar coordinates back to rectangular form. This is essential for signal processing and electrical engineering applications.

EXAMPLE 4.1 — Complex Arithmetic and Polar Coordinates 
import cmath
import math

z1 = 3 + 4j
z2 = 1 - 2j

print(f"z1 = {z1}, z2 = {z2}")
print(f"Addition: {z1 + z2}")
print(f"Multiplication: {z1 * z2}")
print(f"Division: {z1 / z2}")
print(f"Conjugate of z1: {z1.conjugate()}")
print(f"z1^2: {z1 ** 2}")

# Real and imaginary parts
print(f"\nReal part: {z1.real}, Imag part: {z1.imag}")

# Magnitude and Phase (Polar Coordinates)
mag = abs(z1)          # Pythagorean theorem: sqrt(3^2 + 4^2)
phase = cmath.phase(z1) # Angle in radians (atan2(y, x))
print(f"\nMagnitude (abs): {mag}")
print(f"Phase (radians): {phase:.4f} (degrees: {math.degrees(phase):.1f}°)")

# Convert back from polar
z_rect = cmath.rect(mag, phase)
print(f"Rect from polar: {z_rect}")

# Euler's identity: e^(i*pi) + 1 = 0
result = cmath.exp(complex(0, cmath.pi)) + 1
print(f"\ne^(i*π) + 1 = {result}")
print(f"Is it effectively zero? {cmath.isclose(result, 0, abs_tol=1e-15)}")

# Complex roots
print(f"\nSquare root of -1: {cmath.sqrt(-1)}")
print(f"Natural log of -1: {cmath.log(-1)}")
Output

5.0 THE DECIMAL MODULE — EXACT DECIMAL ARITHMETIC FOR FINANCE

5.1 The Problem with Float for Money

Financial applications cannot tolerate the binary rounding errors inherent in float. A difference of one cent in a million transactions is a $10,000 error. The decimal module implements IBM's General Decimal Arithmetic Specification (IEEE 754-2008 decimal floating-point), providing exact representation of decimal numbers and user-controlled precision and rounding.

Critical Rule: Always construct Decimal objects from strings or integers. Decimal(0.1) first creates an approximate float 0.1, then converts that approximation to Decimal, inheriting the binary error. Decimal('0.1') is exact.

5.2 Context, Precision, and Rounding Modes

The getcontext() function retrieves the current thread-local context. You can set the precision (prec) which limits the total number of significant digits in results, and the rounding algorithm (ROUND_HALF_UP, ROUND_DOWN, ROUND_CEILING, etc.). The localcontext() context manager allows temporary precision changes without affecting global state. This is essential for meeting specific financial regulations (e.g., European Central Bank rounding).

5.3 Signals and Traps

The Decimal module includes advanced features like "signals" for error conditions (DivisionByZero, Inexact, Overflow). By default, these are flagged but do not raise exceptions. You can configure the context to trap these signals and raise Python exceptions, providing fine-grained control over numeric error handling.

EXAMPLE 5.1 — Financial Calculations with Decimal 
from decimal import Decimal, getcontext, ROUND_HALF_UP, localcontext

# Set global precision and rounding
getcontext().prec = 10
getcontext().rounding = ROUND_HALF_UP

# Construct correctly from strings
price = Decimal('19.99')
tax_rate = Decimal('0.08')
quantity = Decimal('3')

subtotal = price * quantity
tax = subtotal * tax_rate
total = subtotal + tax

print(f"Price: {price}")
print(f"Subtotal (3 items): {subtotal}")
print(f"Tax (8%): {tax}")
print(f"Total: {total}")

# The float vs Decimal trap
float_sum = 0.1 + 0.1 + 0.1 - 0.3
dec_sum = Decimal('0.1') + Decimal('0.1') + Decimal('0.1') - Decimal('0.3')
print(f"\nFloat 0.1*3 - 0.3 = {float_sum}")
print(f"Decimal 0.1*3 - 0.3 = {dec_sum}")

# NEVER do this: Decimal(0.1) -> loses precision!
bad_dec = Decimal(0.1)
print(f"\nBad Decimal from float(0.1): {bad_dec}")
print(f"Good Decimal from '0.1': {Decimal('0.1')}")

# Local context for specific calculation
with localcontext() as ctx:
    ctx.prec = 3
    print(f"\nLow precision (3 sig figs): {Decimal('1.2345') * Decimal('2.3456')}")

# Quantization (rounding to specific decimal places)
value = Decimal('123.4567')
rounded = value.quantize(Decimal('0.01'), rounding=ROUND_HALF_UP)
print(f"\n123.4567 rounded to cents: {rounded}")
Output

6.0 THE FRACTIONS MODULE — EXACT RATIONAL NUMBERS

6.1 Rational Arithmetic Without Loss

The fractions.Fraction type represents rational numbers as a numerator/denominator pair stored as Python integers. It automatically reduces fractions to their lowest terms using math.gcd(). This provides exactness for operations that cannot be represented finitely in decimal or binary (e.g., 1/3, 1/7).

Fractions are invaluable in mathematical education, symbolic computation, and anywhere recurring decimals would introduce unacceptable cumulative error (e.g., summing geometric series). They also serve as an excellent bridge between floating approximations and exact algebra.

6.2 Constructors and Conversions

Fractions can be constructed from: two integers Fraction(1, 3), a string Fraction('1/3'), a float string Fraction('0.25'), or another Fraction. Avoid Fraction(0.25) as it converts the binary float approximation. Fraction.from_float(0.25) is a safer alternative that uses an algorithm to find the nearest rational with a limited denominator.

The limit_denominator(max_denominator) method is extremely useful for finding a human-friendly fraction that approximates a float (e.g., finding that 0.33333333 ≈ 1/3).

EXAMPLE 6.1 — Exact Rational Arithmetic with Fractions 
from fractions import Fraction

# Creating fractions
f1 = Fraction('1/3')
f2 = Fraction(2, 5)   # 2/5
f3 = Fraction('0.125') # Converts from string float exactly (1/8)

print(f"f1 = {f1}, f2 = {f2}, f3 = {f3}")
print(f"f1 + f2 = {f1 + f2}")
print(f"f1 * f2 = {f1 * f2}")
print(f"f1 / f2 = {f1 / f2}")

# Automatic reduction
f4 = Fraction(6, 8)
print(f"\nFraction(6, 8) reduces to: {f4}")

# Converting to float approximation
print(f"f1 as float: {float(f1):.6f}")

# Mixed operations (int auto-converted to Fraction)
result = f1 * 3
print(f"\n1/3 * 3 = {result} (Numerator: {result.numerator})")

# Getting close to famous constants
pi_float = 3.1415926535
pi_frac = Fraction(pi_float).limit_denominator(1000)
print(f"\nπ ≈ {pi_float}")
print(f"Best fraction with denominator ≤ 1000: {pi_frac} = {float(pi_frac)}")

# The classic 355/113 approximation of π
print(f"Famous approximation: {Fraction(355, 113)} ≈ {float(Fraction(355, 113))}")

# Working with repeating decimals
repeating = Fraction(1, 7) + Fraction(2, 7)
print(f"\n1/7 + 2/7 = {repeating}")
Output

7.0 BITWISE OPERATIONS — MANIPULATING BITS

7.1 Two's Complement with Infinite Precision

Python integers support bitwise operators that act on the two's complement binary representation. Because Python ints have infinite length, these operators act as if the number is padded with an infinite series of sign bits (0 for positive, 1 for negative). This is a mathematically elegant extension of finite-width bitwise operations.

Common operators: & (AND), | (OR), ^ (XOR), ~ (NOT/complement), << (left shift), >> (arithmetic right shift). Bitwise operations are essential for systems programming, cryptography, graphics (RGB color manipulation), network protocol implementation, and performance optimization.

7.2 Practical Bit Manipulation Patterns

Setting a bit: value |= (1 << n)
Clearing a bit: value &= ~(1 << n)
Toggling a bit: value ^= (1 << n)
Testing a bit: bool(value & (1 << n))

Left shift << is equivalent to multiplication by 2^n. Right shift >> is equivalent to floor division by 2^n. For non-negative numbers, these are significantly faster than arithmetic operations (though CPython's peephole optimizer already converts * 2 to a shift).

EXAMPLE 7.1 — Bitwise Operators and Permission Flags 
# Basic bitwise operations
a = 0b1010  # 10
b = 0b1100  # 12

print(f"a = {a} ({bin(a)}), b = {b} ({bin(b)})")
print(f"a & b  = {a & b} ({bin(a & b)})    # AND")
print(f"a | b  = {a | b} ({bin(a | b)})   # OR")
print(f"a ^ b  = {a ^ b} ({bin(a ^ b)})    # XOR")
print(f"~a     = {~a} ({bin(~a)})           # NOT (two's complement)")

# Shifts (multiplication/division by powers of 2)
print(f"\na << 2 = {a << 2} (Equivalent to a * 4)")
print(f"b >> 1 = {b >> 1} (Equivalent to b // 2)")

# Negative number shift (arithmetic shift preserves sign)
neg = -16
print(f"\n-16 >> 2 = {neg >> 2} (Equivalent to -16 // 4)")

# Practical: File permission flags (Unix style)
flags = 0
READ  = 0b100  # 4 (r)
WRITE = 0b010  # 2 (w)
EXEC  = 0b001  # 1 (x)

flags |= READ          # Set read bit
flags |= WRITE         # Set write bit
print(f"\nFlags after setting READ|WRITE: {bin(flags)} (value: {flags})")

flags &= ~WRITE        # Clear write bit
print(f"Flags after clearing WRITE: {bin(flags)} (value: {flags})")

flags ^= EXEC          # Toggle exec bit
print(f"Flags after toggling EXEC: {bin(flags)} (value: {flags})")

# Test bits
print(f"\nHas READ permission? {bool(flags & READ)}")
print(f"Has WRITE permission? {bool(flags & WRITE)}")
print(f"Has EXEC permission? {bool(flags & EXEC)}")

# RGB color manipulation
red   = 0xFF0000
green = 0x00FF00
blue  = 0x0000FF
purple = red | blue
print(f"\nPurple (Red | Blue): {hex(purple)}")
Output

8.0 THE MATH AND RANDOM MODULES

8.1 The Math Module — C Library Wrappers

The math module provides access to the underlying C standard math library functions. It includes constants (pi, e, tau, inf, nan), number-theoretic functions (gcd, lcm, isqrt, comb, perm), power and logarithmic functions (exp, log, log2, log10, pow, sqrt), trigonometric functions (sin, cos, tan and their inverses), and hyperbolic functions.

Python 3.8+ introduced math.comb(n, k) and math.perm(n, k) for combinatorics. Python 3.11+ added math.exp2(x) for 2^x. The math.isqrt(n) function returns the integer square root (floor of exact square root) without floating-point conversion, crucial for large integer algorithms.

8.2 The Random Module — Pseudorandom Number Generation

The random module implements a Mersenne Twister pseudorandom number generator (PRNG) with a period of 2^19937-1. It provides functions for generating random floats (random()), integers in a range (randint(a, b), randrange(start, stop, step)), random selections (choice(), choices(), sample()), and random distributions (gauss(), expovariate(), etc.).

Critical Security Warning: The random module is deterministic (based on a seed) and not cryptographically secure. For security-sensitive applications (password generation, authentication tokens, cryptographic keys), use the secrets module instead. secrets uses the operating system's cryptographically secure random number generator (CSPRNG).

EXAMPLE 8.1 — Math Functions and Random Distributions 
import math
import random

# Math constants
print(f"π: {math.pi}")
print(f"e: {math.e}")
print(f"τ (2π): {math.tau}")

# Number theory and combinatorics
print(f"\nGCD of 48 and 18: {math.gcd(48, 18)}")
print(f"LCM of 12 and 18: {math.lcm(12, 18)}")
print(f"Combinations C(5,2): {math.comb(5, 2)}")
print(f"Permutations P(5,2): {math.perm(5, 2)}")
print(f"Integer sqrt of 1000: {math.isqrt(1000)} (since {math.isqrt(1000)}² ≤ 1000 < {(math.isqrt(1000)+1)}²)")

# Trigonometry
angle_deg = 45
angle_rad = math.radians(angle_deg)
print(f"\nsin({angle_deg}°): {math.sin(angle_rad):.4f}")
print(f"cos({angle_deg}°): {math.cos(angle_rad):.4f}")
print(f"tan({angle_deg}°): {math.tan(angle_rad):.4f}")

# Random module (seeded for reproducibility)
random.seed(42)
print(f"\nRandom integer 1-10: {random.randint(1, 10)}")
print(f"Random choice: {random.choice(['Heads', 'Tails'])}")
print(f"Random float 0-1: {random.random():.4f}")

# Random sample without replacement
lottery_numbers = random.sample(range(1, 50), 6)
print(f"Lottery numbers: {sorted(lottery_numbers)}")

# Gaussian (normal) distribution
gauss_samples = [random.gauss(mu=0, sigma=1) for _ in range(5)]
print(f"Gaussian samples (μ=0, σ=1): {[f'{x:.2f}' for x in gauss_samples]}")

# Shuffle a list
deck = ['A♠', 'K♠', 'Q♠', 'J♠', '10♠']
random.shuffle(deck)
print(f"Shuffled deck: {deck}")
Output

9.0 THE NUMERIC TOWER AND ABSTRACT BASE CLASSES (ABCs)

9.1 The Conceptual Hierarchy

Python defines a conceptual "Numeric Tower" inspired by Scheme and Lisp. It establishes a hierarchy of numeric types: Number > Complex > Real > Rational > Integral. These are defined in the numbers module as Abstract Base Classes (ABCs).

While you rarely use these directly in day-to-day scripting, they are crucial for library development and type checking. For instance, you can check if a variable is a number (including custom types) with isinstance(x, numbers.Number). This is far more flexible than checking type(x) is int because it respects the numeric hierarchy and custom subclasses.

9.2 Practical Applications of the Numeric Tower

If you are writing a function that accepts any numeric type (e.g., a vector math library), you can use isinstance(x, numbers.Complex) to accept int, float, complex, Fraction, and any custom complex number implementation. The ABCs also define the expected abstract methods (e.g., __complex__, __float__, __int__) that custom numeric types should implement to integrate seamlessly with Python's built-in conversion functions.

EXAMPLE 9.1 — Using the Numeric Tower for Type Checking 
import numbers
from fractions import Fraction
from decimal import Decimal

def classify_number(x):
    """Determine where a value sits in the numeric tower."""
    if isinstance(x, numbers.Integral):
        return f"{x} is Integral"
    elif isinstance(x, numbers.Rational):
        return f"{x} is Rational"
    elif isinstance(x, numbers.Real):
        return f"{x} is Real"
    elif isinstance(x, numbers.Complex):
        return f"{x} is Complex"
    elif isinstance(x, numbers.Number):
        return f"{x} is a Number"
    else:
        return f"{x} is NOT a Number"

test_values = [
    42,                           # int
    True,                          # bool (subclass of int)
    3.14,                         # float
    Fraction(1, 2),             # Fraction
    Decimal('2.5'),              # Decimal
    3 + 4j,                      # complex
    "hello",                       # string
    [1, 2, 3]                       # list
]

for val in test_values:
    print(classify_number(val))

# Demonstrate the hierarchy relationships
print(f"\n--- Hierarchy Relationships ---")
print(f"issubclass(bool, numbers.Integral): {issubclass(bool, numbers.Integral)}")
print(f"issubclass(int, numbers.Real): {issubclass(int, numbers.Real)}")
print(f"issubclass(float, numbers.Real): {issubclass(float, numbers.Real)}")
print(f"issubclass(complex, numbers.Complex): {issubclass(complex, numbers.Complex)}")

# Custom class integration
class MyComplex(numbers.Complex):
    def __init__(self, real, imag):
        self._real = real
        self._imag = imag
    def __complex__(self):
        return complex(self._real, self._imag)
    def __abs__(self):
        return (self._real**2 + self._imag**2)**0.5
    # In a real implementation, you'd implement all abstract methods

my_num = MyComplex(1, 2)
print(f"\nMyComplex instance is a Number? {isinstance(my_num, numbers.Number)}")
Output

10.0 PERFORMANCE OPTIMIZATION: __slots__ FOR NUMERIC CLASSES

10.1 The Memory Overhead of __dict__

When defining custom numeric classes (e.g., Vector2D, Matrix, Currency, ComplexNumber), memory overhead can become significant if you create millions of instances. By default, Python stores instance attributes in a dynamic dictionary (__dict__), which consumes a substantial amount of memory (approx 72-104 bytes overhead per object in CPython, depending on the version).

By defining __slots__ in your class, you tell Python to allocate a fixed-size C array of pointers for the named attributes. This eliminates the __dict__ overhead entirely, reducing memory usage by roughly 50-70% and improving attribute access speed (since it's a direct C struct offset rather than a hash table lookup). This is a university-level optimization technique crucial for high-performance numeric computing, scientific simulations, and game development.

10.2 Trade-offs and Limitations

Using __slots__ prevents adding new attributes to instances dynamically (e.g., obj.new_attr = 5 raises AttributeError). It also requires that all parent classes use __slots__ for maximum benefit, and it interacts with multiple inheritance in specific ways. You can still define __weakref__ in __slots__ if you need weak references.

EXAMPLE 10.1 — Memory Optimization with __slots__ 
import sys
import tracemalloc

# Class with __dict__ (default)
class PointWithDict:
    def __init__(self, x, y, z):
        self.x = x
        self.y = y
        self.z = z

# Class with __slots__
class PointWithSlots:
    __slots__ = ('x', 'y', 'z')
    def __init__(self, x, y, z):
        self.x = x
        self.y = y
        self.z = z

p1 = PointWithDict(1.0, 2.0, 3.0)
p2 = PointWithSlots(1.0, 2.0, 3.0)

print(f"Size with __dict__: {sys.getsizeof(p1)} bytes")
print(f"Size with __slots__: {sys.getsizeof(p2)} bytes")
print(f"Memory saved per instance: {sys.getsizeof(p1) - sys.getsizeof(p2)} bytes")

# Has __dict__ vs no __dict__
print(f"\nPointWithDict has __dict__? {hasattr(p1, '__dict__')}")
print(f"PointWithSlots has __dict__? {hasattr(p2, '__dict__')}")

# Calculate savings for 1,000,000 objects
saving_per_million = (sys.getsizeof(p1) - sys.getsizeof(p2)) * 1_000_000
print(f"\nSavings for 1,000,000 points: ~{saving_per_million / (1024**2):.1f} MB")

# Dynamic attribute addition
p1.new_attr = "dynamic"  # Works with __dict__
print(f"\nAdded 'new_attr' to PointWithDict: {p1.new_attr}")

try:
    p2.new_attr = "dynamic"  # Fails with __slots__

    print(f"Cannot add attribute to PointWithSlots: {e}")

SAMPLE 1 — Large Factorial Calculator 
import sys
import math

def factorial_iterative(n):
    """Compute n! using iteration."""
    if n < 0:
        raise ValueError("Factorial undefined for negative numbers")
    if n == 0:
        return 1
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

def factorial_recursive(n):
    """Compute n! using recursion."""
    if n < 0:
        raise ValueError("Factorial undefined for negative numbers")
    if n <= 1:
        return 1
    return n * factorial_recursive(n - 1)

# Test with various sizes
for n in [5, 10, 20, 50, 100]:
    result = factorial_iterative(n)
    digits = len(str(result))
    size_bytes = sys.getsizeof(result)
    
    print(f"{n}! has {digits} digits")
    if n <= 20:
        print(f"  Value: {result}")
    else:
        s = str(result)
        print(f"  Value: {s[:30]}...{s[-20:]}")
    print(f"  Memory: {size_bytes} bytes ({size_bytes/1024:.2f} KB)")
    print()

# Verify against math.factorial
n = 50
my_fact = factorial_iterative(n)
math_fact = math.factorial(n)
print(f"Verification: my_fact(50) == math.factorial(50): {my_fact == math_fact}")

SAMPLE PROGRAM 2 — Vector3D Class with Operator Overloading and __slots__

A production-ready 3D vector class that demonstrates operator overloading (+, -, *, /, abs(), ==), memory optimization with __slots__, and integration with Python's data model via __repr__ and __str__. This is the foundation of any game engine or scientific computing library.

Concepts demonstrated: __slots__, dunder methods (__add__, __sub__, __mul__, __truediv__, __abs__, __eq__), type hints, and math.hypot for 3D magnitude.

SAMPLE 2 — Optimized Vector3D Class 
from __future__ import annotations
import math
import sys

class Vector3D:
    """A memory-efficient 3D vector with operator overloading."""
    
    __slots__ = ('x', 'y', 'z')
    
    def __init__(self, x: float, y: float, z: float) -> None:
        self.x = x
        self.y = y
        self.z = z
    
    def __add__(self, other: Vector3D) -> Vector3D:
        if not isinstance(other, Vector3D):
            return NotImplemented
        return Vector3D(self.x + other.x, self.y + other.y, self.z + other.z)
    
    def __sub__(self, other: Vector3D) -> Vector3D:
        if not isinstance(other, Vector3D):
            return NotImplemented
        return Vector3D(self.x - other.x, self.y - other.y, self.z - other.z)
    
    def __mul__(self, scalar: float) -> Vector3D:
        return Vector3D(self.x * scalar, self.y * scalar, self.z * scalar)
    
    def __rmul__(self, scalar: float) -> Vector3D:
        return self.__mul__(scalar)
    
    def __truediv__(self, scalar: float) -> Vector3D:
        if scalar == 0:
            raise ZeroDivisionError("Cannot divide vector by zero")
        return Vector3D(self.x / scalar, self.y / scalar, self.z / scalar)
    
    def __neg__(self) -> Vector3D:
        return Vector3D(-self.x, -self.y, -self.z)
    
    def __abs__(self) -> float:
        """Return the magnitude (length) of the vector."""
        return math.hypot(self.x, self.y, self.z)
    
    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Vector3D):
            return NotImplemented
        return (math.isclose(self.x, other.x) and 
                math.isclose(self.y, other.y) and 
                math.isclose(self.z, other.z))
    
    def dot(self, other: Vector3D) -> float:
        """Compute dot product."""
        return self.x * other.x + self.y * other.y + self.z * other.z
    
    def cross(self, other: Vector3D) -> Vector3D:
        """Compute cross product."""
        return Vector3D(
            self.y * other.z - self.z * other.y,
            self.z * other.x - self.x * other.z,
            self.x * other.y - self.y * other.x
        )
    
    def normalize(self) -> Vector3D:
        """Return a unit vector in the same direction."""
        mag = abs(self)
        if mag == 0:
            raise ValueError("Cannot normalize zero vector")
        return self / mag
    
    def __repr__(self) -> str:
        return f"Vector3D({self.x}, {self.y}, {self.z})"
    
    def __str__(self) -> str:
        return f"({self.x:.2f}, {self.y:.2f}, {self.z:.2f})"


# Demonstrate the Vector3D class
v1 = Vector3D(3, 4, 0)
v2 = Vector3D(1, 2, 3)

print(f"v1 = {v1}")
print(f"v2 = {v2}")
print(f"v1 + v2 = {v1 + v2}")
print(f"v1 - v2 = {v1 - v2}")
print(f"v1 * 2.5 = {v1 * 2.5}")
print(f"|v1| = {abs(v1):.4f}")
print(f"|v2| = {abs(v2):.4f}")
print(f"Dot product: {v1.dot(v2)}")
print(f"Cross product: {v1.cross(v2)}")
print(f"Normalized v2: {v2.normalize()}")
print(f"|Normalized v2| = {abs(v2.normalize()):.8f}")

# Memory comparison
print(f"\nMemory per Vector3D: {sys.getsizeof(v1)} bytes")

SAMPLE PROGRAM 3 — Financial Portfolio Tracker (Decimal)

A robust portfolio management system demonstrating the Decimal module for exact financial calculations. It tracks stock positions, calculates total value with precise decimal arithmetic, and generates a formatted report. This illustrates how to avoid floating-point errors in money-related applications.

Concepts demonstrated: Decimal for financial math, @dataclass integration, dictionary comprehensions, and formatted currency output.

SAMPLE 3 — Financial Portfolio Tracker 
from decimal import Decimal, ROUND_HALF_UP
from dataclasses import dataclass

@dataclass
class StockPosition:
    """Represents a holding in a specific stock."""
    symbol: str
    shares: int
    purchase_price: Decimal
    current_price: Decimal
    
    @property
    def cost_basis(self) -> Decimal:
        """Total amount paid for the position."""
        return (self.purchase_price * Decimal(self.shares)).quantize(Decimal('0.01'))
    
    @property
    def market_value(self) -> Decimal:
        """Current total value of the position."""
        return (self.current_price * Decimal(self.shares)).quantize(Decimal('0.01'))
    
    @property
    def gain_loss(self) -> Decimal:
        """Unrealized profit or loss."""
        return self.market_value - self.cost_basis
    
    @property
    def gain_loss_pct(self) -> Decimal:
        """Percentage gain or loss."""
        if self.cost_basis == 0:
            return Decimal('0')
        return (self.gain_loss / self.cost_basis * Decimal('100')).quantize(Decimal('0.01'))

class Portfolio:
    """Manages a collection of stock positions."""
    
    def __init__(self, name: str):
        self.name = name
        self._positions: dict[str, StockPosition] = {}
    
    def add_position(self, position: StockPosition) -> None:
        if position.symbol in self._positions:
            raise ValueError(f"Symbol {position.symbol} already in portfolio")
        self._positions[position.symbol] = position
    
    def update_price(self, symbol: str, new_price: Decimal) -> None:
        if symbol not in self._positions:
            raise KeyError(f"Symbol {symbol} not found")
        self._positions[symbol].current_price = new_price
    
    @property
    def total_cost(self) -> Decimal:
        return sum((p.cost_basis for p in self._positions.values()), Decimal('0'))
    
    @property
    def total_value(self) -> Decimal:
        return sum((p.market_value for p in self._positions.values()), Decimal('0'))
    
    @property
    def total_gain_loss(self) -> Decimal:
        return self.total_value - self.total_cost
    
    def report(self) -> str:
        lines = [
            f"\n{'='*70}",
            f"  PORTFOLIO REPORT: {self.name}",
            f"{'='*70}",
            f"  {'Symbol':<8} {'Shares':>8} {'Cost Basis':>12} {'Market Value':>13} {'Gain/Loss':>12} {'Pct':>7}",
            f"  {'-'*66}"
        ]
        
        for pos in sorted(self._positions.values(), key=lambda p: p.symbol):
            lines.append(
                f"  {pos.symbol:<8} {pos.shares:>8,} "
                f"${pos.cost_basis:>11,.2f} ${pos.market_value:>12,.2f} "
                f"${pos.gain_loss:>11,.2f} {pos.gain_loss_pct:>6}%"
            )
        
        lines.extend([
            f"  {'-'*66}",
            f"  {'TOTAL':<8} {'':8} ${self.total_cost:>11,.2f} ${self.total_value:>12,.2f} ${self.total_gain_loss:>11,.2f}",
            f"{'='*70}"
        ])
        return "\n".join(lines)

# Create portfolio with exact decimal values
portfolio = Portfolio("Tech Growth Fund")

portfolio.add_position(StockPosition(
    "AAPL", 150, 
    Decimal('175.50'), 
    Decimal('192.35')
))

portfolio.add_position(StockPosition(
    "MSFT", 80, 
    Decimal('330.25'), 
    Decimal('415.80')
))

portfolio.add_position(StockPosition(
    "GOOGL", 45, 
    Decimal('138.75'), 
    Decimal('175.20')
))

portfolio.add_position(StockPosition(
    "NVDA", 200, 
    Decimal('45.60'), 
    Decimal('128.44')
))

# Simulate a price update

print(portfolio.report())

13.0 KNOWLEDGE CHECK: PYTHON NUMBERS QUIZ

1. Why does 0.1 + 0.2 not equal 0.3 in Python?
Click to reveal answer
Because 0.1 and 0.2 cannot be represented exactly as binary fractions in the IEEE 754 floating-point format. They are stored as the closest possible binary approximations, leading to a small rounding error when added. Use math.isclose() for comparisons or Decimal for exact decimal arithmetic.
2. What is the difference between `/` and `//` in Python 3?
Click to reveal answer
/ performs true division and always returns a float result. // performs floor division and returns an int (for int operands) representing the mathematical floor of the quotient (rounded toward negative infinity).
3. When should you use the Decimal module instead of float?
Click to reveal answer
Use Decimal for financial applications or any context where exact decimal representation is required (e.g., money, accounting). Decimal avoids the binary rounding errors inherent to floats and provides configurable precision and rounding modes that meet legal and financial standards.
4. How does Python handle integer overflow?
Click to reveal answer
Python integers have arbitrary precision — they do not overflow. When an operation would exceed the capacity of a fixed-width integer, Python automatically allocates more memory to store the larger value. The only limit is the available system memory.
5. What is the purpose of __slots__ in a numeric class?
Click to reveal answer
__slots__ tells Python to allocate a fixed-size array for instance attributes instead of a dynamic dictionary (__dict__). This reduces memory overhead by 50-70% per instance and improves attribute access speed. It's essential for classes that will have millions of instances (e.g., points in a 3D mesh, financial ticks).
