🔢 Integers — The Foundation

Python integers (int) are unbounded, meaning they can grow to any size. They support the usual binary, octal, and hexadecimal representations.

🔢 1.1 Integer Literals and Bases

Integers can be written in decimal, binary (prefix 0b), octal (0o), or hexadecimal (0x).

  • Step 1: Define demo_integers() to show different literal forms.
  • Step 2: Assign decimal 42 to dec.
  • Step 3: Assign binary 0b1010 (10) to bin.
  • Step 4: Assign octal 0o52 (42) to oct.
  • Step 5: Assign hexadecimal 0x2A (42) to hex.
  • Step 6: Print each value and its type.
  • Step 7: Call the function to execute.

def demo_integers():
    dec = 42

    print(f"Octal: {oct} (type: {type(oct).__name__})")
    print(f"Hexadecimal: {hex} (type: {type(hex).__name__})")

🔢 1.2 Bitwise Operations

Integers support bitwise operations: & (and), | (or), ^ (xor), ~ (not), << (left shift), >> (right shift).

Step-by-step explanation:

  • Step 1: Define bitwise_demo().
  • Step 2: Set a = 0b1100 (12) and b = 0b1010 (10).

💧 Floating‑Point Numbers

float represents double‑precision (64‑bit) numbers. They are fast but have limited precision.

💧 2.1 Float Literals and Precision

Floats can be written with a decimal point or in scientific notation (1.2e3).

  • Step 1: Define float_demo().
  • Step 2: Assign 3.14, 1.2e3 (1200.0), -0.5.
  • Step 3: Print each and their types.
  • Step 4: Show the classic precision issue: 0.1 + 0.2 == 0.3 is False.
  • Step 5: Print the exact representation using repr().
  • Step 6: Call the function.
python 
def float_demo():
    pi = 3.14
    sci = 1.2e3
    neg = -0.5

    print(f"pi = {pi} (type: {type(pi).__name__})")
    print(f"sci = {sci} (type: {type(sci).__name__})")
    print(f"neg = {neg} (type: {type(neg).__name__})")

    result = 0.1 + 0.2
    print(f"\n0.1 + 0.2 = {result}")
    print(f"0.1 + 0.2 == 0.3? {0.1 + 0.2 == 0.3}")
    print(f"repr(0.1) = {repr(0.1)}")
    print(f"repr(0.2) = {repr(0.2)}")
    print(f"repr(0.1 + 0.2) = {repr(0.1 + 0.2)}")

⚠️ Floating‑point arithmetic is not exact. Use Decimal for money or when exact representation is required.

💵 The decimal Module – Exact Arithmetic

Use Decimal for exact decimal arithmetic, crucial for financial and scientific applications where rounding errors are unacceptable.

💵 7.1 Decimal Basics

Create Decimal objects from strings or integers to preserve precision.

Step-by-step explanation:

  • Step 1: Import Decimal from decimal.
  • Step 2: Define decimal_demo().
  • Step 3: Create Decimal('0.1') and Decimal('0.2').
  • Step 4: Add them and compare to Decimal('0.3').
  • Step 5: Show the exact result.
  • Step 6: Perform multiplication and division.
  • Step 7: Set precision using getcontext().prec.
python 
from decimal import Decimal, getcontext

def decimal_demo():
    print("Decimal arithmetic:")
    a = Decimal("0.1")
    b = Decimal("0.2")
    c = a + b
    print(f"0.1 + 0.2 = {c}")
    print(f"0.1 + 0.2 == 0.3? {c == Decimal("0.3")}")

    print(f"0.1 * 0.2 = {a * b}")
    print(f"0.2 / 0.1 = {a / b}")

    getcontext().prec = 6
    print(f"0.1 / 0.3 (precision 6) = {Decimal("0.1") / Decimal("0.3")}")

decimal_demo()
⚠️ Always create Decimal from strings to avoid floating‑point artifacts. Decimal(0.1) is not exact.

❓ Frequently Asked Questions – Python Numbers

Common questions about Python numbers, answered.

Why does 0.1 + 0.2 not equal 0.3?

+

Floating‑point numbers are represented in binary, and 0.1 cannot be represented exactly. This is a fundamental limitation of binary floating‑point. Use Decimal for exact arithmetic.

How do I round a float to 2 decimal places?

+

Use round(3.14159, 2) or format with f"{3.14159:.2f}". For exact rounding, use Decimal with quantize().

What is the difference between / and //?

+

/ performs floating‑point division (returns a float). // performs floor division (returns an integer, discarding the remainder).

When should I use Decimal vs float?

+

Use float for performance in scientific and graphics applications. Use Decimal for financial, monetary, and any case where exact decimal representation is required.

Can Python handle very large integers?

+

Yes! Python integers are unbounded and can grow to any size limited only by available memory. This is a key advantage over many other languages.

How do I generate a random integer in a range?

+

Use random.randint(a, b) for inclusive range, or random.randrange(start, stop) for exclusive stop.
