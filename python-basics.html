1.0 INTRODUCTION — WHY PYTHON IS THE PERFECT FIRST LANGUAGE

Every journey into programming has to begin somewhere, and for millions of learners worldwide that starting point is Python. Created by Guido van Rossum and first released in 1991, Python was designed with a single guiding principle: code should be easy to read and easy to write. The language strips away the syntactic noise found in languages such as Java or C++ — no semicolons required at the end of lines, no curly braces to delimit blocks, no mandatory type declarations — allowing the programmer to focus entirely on expressing ideas rather than wrestling with punctuation.

In 2026, Python holds the top spot in virtually every major programming language index. It powers artificial intelligence and machine learning pipelines, drives data science dashboards, automates cloud infrastructure, runs the back-ends of major web applications, and is the language of choice for scientific research, financial modelling, and cybersecurity tooling. Learning Python today means joining the most active and welcoming community in software development and gaining access to thousands of battle-tested libraries that solve almost any problem imaginable.

2.0 PYTHON SYNTAX AND INDENTATION

2.1 The Golden Rule: Indentation Defines Structure

In most programming languages, code blocks are delimited by curly braces { }. Python takes a radically different approach: it uses indentation — the blank space at the start of a line — to define which statements belong to which block. This is not a style preference; it is a language rule enforced by the interpreter. A block is started by ending the header line with a colon :, and all lines that are part of that block must be indented by the same amount.

The Python community standard, defined in PEP 8, is four spaces per indentation level. You may use two spaces or even tabs, but you must be consistent within a file — mixing spaces and tabs causes a TabError. Most modern code editors insert four spaces automatically when you press the Tab key in Python files.

2.2 Statements and Line Continuation

In Python, each logical line of code is called a statement. Normally one statement occupies one physical line. If a statement is very long, you can split it across multiple physical lines by using a backslash \ at the end of each intermediate line, or by wrapping the expression inside any pair of brackets: parentheses ( ), square brackets [ ], or curly braces { }. The latter is preferred because it is cleaner.

⚠️ IndentationError: If Python reports an IndentationError, it means a line's indentation does not match the expected level. Check for accidental mixing of tabs and spaces, or look for a missing or extra level of indentation.

EXAMPLE 2.1 — Syntax, Indentation, and Line Continuation 
# Correct indentation: 4 spaces per level
temperature = 37.5

if temperature > 37.0:
    print("Above normal temperature.")
    if temperature > 39.0:
        print("High fever — seek medical advice.")
    else:
        print("Mild fever — rest and hydrate.")
else:
    print("Normal temperature.")

# Long line split using parentheses (preferred)
total = (
    100
    + 200
    + 300
    + 400
)
print(f"Total: {total}")

# Long string split using backslash (alternative)
message = "Python was designed to be readable, "  \
           "practical, and expressive."
print(message)

3.0 VARIABLES AND ASSIGNMENT

3.1 What Is a Variable?

A variable is a name that refers to a value stored in the computer's memory. Think of it as a labelled box: the label is the variable name, and the contents of the box is the value. In Python, you create a variable simply by writing a name, followed by the equals sign =, followed by the value you want to store. Python does not require you to declare the type of the variable beforehand — it figures out the type automatically based on the value you assign. This property is called dynamic typing.

3.2 Naming Rules and Conventions

Variable names in Python must follow these rules: they may contain letters (a–z, A–Z), digits (0–9), and underscores (_); they cannot begin with a digit; they cannot be a Python keyword (such as if, for, class, or return); and they are case-sensitive (age and Age are two different variables). By convention (PEP 8), variable names use snake_case — all lowercase with underscores separating words — for example student_name, total_score, is_active.

3.3 Multiple Assignment and Swapping

Python allows several elegant shorthand forms. You can assign a single value to multiple variables in one line, assign multiple variables from a sequence in one line (called unpacking), and swap two variables without a temporary variable. These idioms are idiomatic Python and appear frequently in real-world code.

💡 Dynamic Typing: Unlike Java or C, Python variables are not bound to a type. The same variable can hold an integer, then a string, then a list. However, this flexibility comes with responsibility — be intentional about what type a variable holds to avoid hard-to-find bugs.

EXAMPLE 3.1 — Variables, Naming, and Multiple Assignment 
# Basic assignment
student_name = "Maria Santos"
age          = 21
gpa          = 3.85
is_enrolled  = True

print(student_name, age, gpa, is_enrolled)

# Assign the same value to multiple variables
x = y = z = 0
print(x, y, z)

# Multiple assignment (unpacking)
first, second, third = "Python", "is", "powerful"
print(first, second, third)

# Swap two variables without a temporary
a, b = 10, 20
print(f"Before swap: a={a}, b={b}")
a, b = b, a
print(f"After swap:  a={a}, b={b}")

# Check a variable's type at runtime
score = 95
print(type(score))          # <class 'int'>
score = 95.5
print(type(score))          # <class 'float'>

4.0 CORE DATA TYPES

4.1 Integers (int)

An integer is a whole number — positive, negative, or zero — with no fractional part. Python's integers have unlimited precision: you can represent numbers as large as your system's available memory allows without overflow. Integer literals are written without quotes: 42, -7, 0. For readability, Python 3 allows underscores as digit separators: 1_000_000 is the same as 1000000.

4.2 Floating-Point Numbers (float)

A float represents a real number with a decimal point, such as 3.14, -0.001, or 2.0. Python floats are IEEE 754 double-precision numbers, which means they provide approximately 15–17 significant decimal digits. Floats can also be expressed in scientific notation: 1.5e10 means 1.5 × 10¹⁰. Be aware that floats cannot represent most decimal fractions exactly — a consequence of binary floating-point representation.

4.3 Strings (str)

A string is a sequence of Unicode characters enclosed in single quotes, double quotes, or triple quotes (for multi-line strings). Strings are immutable — once created, their contents cannot be changed. Python provides a rich set of string methods and supports f-strings (formatted string literals), introduced in Python 3.6, for embedding expressions directly inside string literals.

4.4 Booleans (bool)

Python has two boolean literals: True and False (note the capital first letter). Booleans are a subclass of integers — True has integer value 1 and False has value 0. This is why you can add booleans: True + True evaluates to 2. Every Python object has a truth value — objects that evaluate to False include 0, 0.0, "", [], {}, None, and any empty container.

4.5 None

None is Python's null value — it represents the explicit absence of a value. It is the single instance of the NoneType class. Functions that do not explicitly return a value return None implicitly. Use is None or is not None to test for None — never == None.

# Integers — unlimited precision
big_number = 10 ** 100    # googol
print(f"Googol: {big_number}")

# Floats — IEEE 754 precision quirk

5.0 OPERATORS

5.1 Arithmetic Operators

Python provides all standard arithmetic operators. Addition +, subtraction -, multiplication *, and division / behave as expected. Division always returns a float, even when dividing two integers. For integer division (floor division), use //. The modulo operator % returns the remainder after division — essential for testing divisibility. The exponentiation operator ** raises a number to a power and is right-associative.

5.2 Comparison and Logical Operators

Comparison operators compare two values and return a boolean. They include == (equal), != (not equal), <, >, <=, >=. Python uniquely supports chained comparisons such as 0 < x < 10, which is evaluated correctly without parentheses. Logical operators and, or, and not combine boolean expressions. They use short-circuit evaluation: and stops at the first False and or stops at the first True.

5.3 Augmented Assignment Operators

Python provides augmented assignment operators as shorthand for updating a variable: +=, -=, *=, /=, //=, %=, **=. For example, x += 5 is exactly equivalent to x = x + 5. These operators make code more concise and readable.

5.4 Membership and Identity Operators

The membership operators in and not in test whether a value exists within a sequence (string, list, tuple, set, or dictionary). The identity operators is and is not test whether two variables point to the exact same object in memory — as distinct from testing value equality with ==. Always use is for comparisons against None, True, and False.

EXAMPLE 5.1 — All Operator Categories 
# Arithmetic operators
a, b = 17, 5
print(f"Add:       {a} + {b}  = {a + b}")
print(f"Sub:       {a} - {b}  = {a - b}")
print(f"Mul:       {a} * {b}  = {a * b}")
print(f"Div:       {a} / {b}  = {a / b}")    # float
print(f"FloorDiv:  {a} // {b} = {a // b}")   # int
print(f"Modulo:    {a} % {b}  = {a % b}")
print(f"Exponent:  {a} ** {b} = {a ** b}")

# Comparison + chained comparison
x = 7
print(0 < x < 10)       # True — Python chain
print(x == 7, x != 5)

# Logical operators with short-circuit
print(True  and False)   # False
print(False or  True)    # True
print(not True)           # False

# Augmented assignment
score = 100
score += 10
score -= 5
score *= 2
print(f"Final score: {score}")   # (100+10-5)*2 = 210

# Membership and identity
langs = ["Python", "JavaScript", "Rust"]
print("Python" in langs)         # True
print("Java"   not in langs)     # True

val = None
print(val is None)               # True

6.0 TYPE CONVERSION (TYPE CASTING)

6.1 Implicit vs Explicit Conversion

Python automatically converts between compatible numeric types when it is safe to do so — for example, adding an integer and a float produces a float. This is called implicit type conversion (or type coercion). When Python cannot do this automatically, or when you explicitly want to change a value's type, you use explicit type conversion (type casting) by calling one of Python's built-in type constructor functions: int(), float(), str(), bool().

6.2 Common Conversion Functions

The int() function converts a float (by truncating the decimal part, not rounding) or a string of digits into an integer. The float() function converts an integer or numeric string into a float. The str() function converts any object into its string representation. The bool() function returns False for falsy values (zero, empty containers, None) and True for everything else.

⚠️ Conversion Errors: If you try to convert a non-numeric string such as "hello" to an integer with int("hello"), Python raises a ValueError. Always validate or handle potential errors when converting user-provided strings.

EXAMPLE 6.1 — Type Conversion in Practice 
# Implicit conversion (int + float → float)
result = 5 + 2.0
print(result, type(result))     # 7.0 <class 'float'>

# int() — truncates, does NOT round
print(int(9.9))               # 9
print(int(-3.7))              # -3 (truncates toward zero)
print(int("42"))              # 42

# float()
print(float(10))              # 10.0
print(float("3.14"))         # 3.14

# str()
print(str(255))               # '255'
print(str(True))              # 'True'

# bool() — falsy values
for val in [0, 0.0, "", [], {}, None, 42, "hi"]:
    print(f"bool({val!r:8}) = {bool(val)}")

# Safe conversion with error handling
user_input = "not_a_number"
try:
    num = int(user_input)
except ValueError:
    print(f"Cannot convert '{user_input}' to int.")

7.0 CONDITIONAL STATEMENTS

7.1 The if / elif / else Structure

Conditional statements allow your program to make decisions. The if keyword evaluates an expression; if it is truthy, the indented block beneath it executes. If the condition is falsy, Python checks any elif (else if) clauses in order, executing the first one whose condition is truthy. If no condition is satisfied, the optional else block runs. There is no limit on the number of elif clauses. Only one branch executes per if–elif–else chain.

7.2 The Ternary (Conditional) Expression

Python provides a concise one-line conditional called the ternary expression (or conditional expression): value_if_true if condition else value_if_false. This is useful for simple value selection inside an assignment or function call, but should not be nested deeply — use a full if–else block when the logic is complex.

7.3 match / case (Python 3.10+)

Python 3.10 introduced structural pattern matching via match and case statements — Python's equivalent of switch/case found in other languages, but far more powerful. It can match on literal values, variable bindings, sequences, mappings, and class patterns. For straightforward multi-way branching on simple values, match/case is cleaner than a chain of elifs.

💡 Truthy Conditions: You do not need to write if x == True: — simply write if x:. Similarly, if my_list: is the idiomatic way to check that a list is non-empty. This is more Pythonic and slightly faster.

EXAMPLE 7.1 — if/elif/else, Ternary, and match/case 
# 1. Standard if / elif / else
def grade(score: int) -> str:
    if score >= 90:
        return "A — Excellent"
    elif score >= 80:
        return "B — Good"
    elif score >= 70:
        return "C — Satisfactory"
    elif score >= 60:
        return "D — Needs Improvement"
    else:
        return "F — Failing"

for s in [95, 83, 72, 61, 45]:
    print(f"Score {s}: {grade(s)}")

# 2. Ternary (conditional) expression
age    = 20
status = "adult" if age >= 18 else "minor"
print(f"\nAge {age} → {status}")

# 3. match / case (Python 3.10+)
def http_status(code: int) -> str:
    match code:
        case 200: return "OK"
        case 201: return "Created"
        case 400: return "Bad Request"
        case 401: return "Unauthorized"
        case 404: return "Not Found"
        case 500: return "Internal Server Error"
        case  _  : return "Unknown Status"

print()
for code in [200, 404, 500, 418]:
    print(f"HTTP {code}: {http_status(code)}")
Output

8.0 LOOPS: for AND while

8.1 The for Loop

A for loop iterates over any iterable object — a sequence of values that can be visited one at a time. Common iterables include strings, lists, tuples, ranges, dictionaries, and files. On each iteration, the loop variable is bound to the next item in the sequence. The built-in range() function generates sequences of integers and is the standard way to repeat code a fixed number of times. enumerate() wraps an iterable and yields (index, value) pairs; zip() interleaves two or more iterables element by element.

8.2 The while Loop

A while loop repeats its body as long as a condition remains truthy. It is the right tool when you do not know in advance how many iterations are needed — for example, when reading user input until a valid entry is received, or when polling a sensor until a threshold is crossed. Always ensure the condition will eventually become false, or include a break statement, to avoid an infinite loop.

8.3 break, continue, and else

Inside any loop, break immediately terminates the loop and resumes execution after it. continue skips the rest of the current iteration and starts the next one. Both for and while loops support an optional else clause that runs only if the loop completed normally — that is, without a break. This is useful for search loops where you want to signal "not found" only after exhausting all items.

💡 Avoid range(len(x)): Writing for i in range(len(my_list)): and then using my_list[i] is un-Pythonic. Use for item in my_list: when you only need values, and for i, item in enumerate(my_list): when you need both index and value.

EXAMPLE 8.1 — for Loop, while Loop, and Loop Control 
# for loop with range
print("Multiplication table for 7:")
for i in range(1, 11):
    print(f"  7 × {i:2d} = {7 * i:3d}")

# enumerate — index and value together
fruits = ["apple", "banana", "cherry"]
print("\nFruit list:")
for i, fruit in enumerate(fruits, start=1):
    print(f"  {i}. {fruit}")

# zip — iterate two lists in parallel
students = ["Alice", "Bob", "Carol"]
scores   = [92, 87, 95]
print("\nScoreboard:")
for student, score in zip(students, scores):
    print(f"  {student}: {score}")

# while loop — count down
countdown = 5
print("\nCountdown:")
while countdown > 0:
    print(f"  {countdown}...")
    countdown -= 1
print("  Lift-off! 🚀")

# break, continue, and for-else
print("\nSearching for 7 in a range:")
for n in range(1, 15):
    if n % 2 == 0:
        continue          # skip even numbers
    print(f"  Odd: {n}", end=" ")
    if n == 7:
        print("← found it!")
        break
else:
    print("7 not found.")    # only runs if no break
Output

9.0 FUNCTIONS

9.1 Defining and Calling Functions

A function is a named, reusable block of code that performs a specific task. You define a function with the def keyword, give it a name, list its parameters inside parentheses, end the header with a colon, and indent the body. Calling a function executes its body with the arguments you provide. Functions are first-class objects in Python — they can be assigned to variables, passed as arguments, and returned from other functions.

9.2 Parameters and Arguments

Python functions support several kinds of parameters. Positional parameters are matched by position. Keyword arguments are passed by name: greet(name="Alice"). Default parameter values are specified in the function definition and used when an argument is not provided: def greet(name="World"):. Variadic parameters collect any number of positional arguments into a tuple (*args) or any number of keyword arguments into a dictionary (**kwargs).

9.3 Return Values and None

The return statement ends a function's execution and optionally passes a value back to the caller. A function can return multiple values as a tuple using comma-separated return values: return x, y. If a function has no return statement, or a bare return, it returns None implicitly. Attempting to use the return value of a function that returns None (such as list.sort()) is a common beginner mistake.

9.4 Docstrings

A docstring is a string literal that appears as the first statement in a function body, enclosed in triple quotes. It documents what the function does, its parameters, and its return value. Docstrings are accessible at runtime via the __doc__ attribute and are used by IDEs, linters, and documentation generators. Every public function should have one.

💡 Single Responsibility: A well-designed function does one thing and does it well. If you find yourself writing a function that does two unrelated things, split it into two functions. This makes each function easier to test, read, and reuse.

EXAMPLE 9.1 — Functions: Parameters, Defaults, *args, **kwargs 
# Basic function with docstring
def celsius_to_fahrenheit(celsius: float) -> float:
    """Convert Celsius temperature to Fahrenheit.
    
    Args:
        celsius: Temperature in degrees Celsius.
    Returns:
        Temperature in degrees Fahrenheit.
    """
    return celsius * 9 / 5 + 32

for c in [-40, 0, 20, 37, 100]:
    print(f"  {c}°C = {celsius_to_fahrenheit(c):.1f}°F")

# Default parameter values
def greet(name: str = "Learner", greeting: str = "Hello") -> str:
    return f"{greeting}, {name}! Welcome to Poetic Bytes."

print(greet())                          # uses both defaults
print(greet("Maria"))                  # overrides name only
print(greet("Carlos", "Good morning")) # overrides both

# Multiple return values
def min_max(numbers: list) -> tuple:
    """Return (minimum, maximum) of a list."""
    return min(numbers), max(numbers)

lo, hi = min_max([8, 3, 15, 1, 9])
print(f"\nMin: {lo}, Max: {hi}")

# *args — variable positional arguments
def total(*numbers) -> float:
    """Sum any number of numeric arguments."""
    return sum(numbers)

print(f"\nTotal: {total(1, 2, 3, 4, 5)}")

# **kwargs — variable keyword arguments
def create_profile(**fields) -> str:
    """Build a user profile from arbitrary keyword arguments."""
    lines = [f"  {k.capitalize()}: {v}" for k, v in fields.items()]
    return "\n".join(lines)

print("\nUser Profile:")
print(create_profile(name="Ana", age=28, country="Philippines", role="Developer"))
Output

10.0 INPUT AND OUTPUT

10.1 The print() Function

The built-in print() function outputs text to the standard output (usually the terminal). It accepts any number of arguments, separated by commas, and converts each to a string using str(). By default, it separates multiple arguments with a space (controlled by the sep parameter) and ends with a newline (controlled by the end parameter). For precise control over formatting, use f-strings or the format() method.

10.2 Formatted Output with f-strings

F-strings, introduced in Python 3.6, are the modern and most readable way to embed expressions inside strings. Prefix the string with f and place any valid Python expression inside { }. Format specifiers after a colon control number formatting: :.2f for two decimal places, :, for thousands separators, :>10 to right-align in a field of width 10, :^20 to centre, and :b for binary representation.

10.3 The input() Function

The input() function pauses program execution and waits for the user to type something and press Enter. It always returns the entered text as a string — if you need a number, you must convert the result explicitly with int() or float(). In browser-based Python environments like this one, input() is not interactive; the examples below demonstrate formatting without live input.

EXAMPLE 10.1 — print(), f-string Formatting, and input() Simulation 
# print() with sep and end
print("Python", "is", "awesome")                # default sep=" "
print("Python", "is", "awesome", sep="-")     # sep="-"
print("No newline here...", end=" ")
print("same line!")

# f-string formatting — numbers
pi      = 3.141592653589793
salary  = 125000
percent = 0.8765

print(f"\npi (4 dp):        {pi:.4f}")
print(f"salary with sep:  {salary:,}")
print(f"percentage:       {percent:.1%}")
print(f"scientific:       {pi:e}")

# f-string formatting — alignment
header  = f"{'Name':<15} {'Score':>6} {'Grade':^8}"
divider = "-" * len(header)
print(f"\n{header}")
print(divider)
data = [("Alice", 95, "A"), ("Bob", 82, "B"), ("Carol", 78, "C+")]
for name, score, grade in data:
    print(f"{name:<15} {score:>6} {grade:^8}")

# Simulated input() — in real scripts use: name = input("Enter name: ")
simulated_input = "28"    # pretend the user typed this
age = int(simulated_input)
print(f"\nIn 10 years you will be {age + 10} years old.")

11.0 COMMENTS AND CODE STYLE

11.1 Single-Line Comments

A comment begins with the hash character # and extends to the end of the physical line. Python ignores everything from the # to the end of the line. Comments exist solely for the human reader. Use them to explain why code does something non-obvious — not what it does (the code itself should show that). Avoid redundant comments like # increment x by 1 before x += 1.

11.2 Multi-Line Comments and Docstrings

Python has no dedicated multi-line comment syntax. Developers typically stack multiple single-line comments or use triple-quoted strings that are not assigned to any variable (the interpreter evaluates and discards them). However, the canonical use of triple-quoted strings as documentation is the docstring at the beginning of a module, class, or function — as described in Section 9.4.

11.3 PEP 8 — The Python Style Guide

PEP 8 is the official Python style guide, written by Guido van Rossum. Following it makes your code consistent with the vast majority of Python code in the world. Key rules: use 4 spaces per indentation level; limit lines to 79 characters; surround binary operators with single spaces; add two blank lines between top-level definitions; use snake_case for variables and functions, CamelCase for classes, and ALL_CAPS for module-level constants. Use a linter like flake8 or formatter like black to check compliance automatically.

🐍 snake_case

Variables and functions: student_name, calculate_total()

🏛️ CamelCase

Class names: BankAccount, HttpRequest

📢 ALL_CAPS

Module-level constants: MAX_RETRIES = 3, PI = 3.14

EXAMPLE 11.1 — Comments, Docstrings, and PEP 8 Style 
"""
Module: grade_calculator.py
Purpose: Demonstrate good commenting and PEP 8 style.
Author:  Poetic Bytes — Python Basics Guide
"""

# ── Module-level constant (ALL_CAPS) ──────────────────────────────
PASSING_SCORE = 50
HONOUR_ROLL   = 90


def classify_student(name: str, score: float) -> dict:
    """Classify a student based on their score.

    Args:
        name:  The student's full name.
        score: Numeric score, expected in range [0, 100].

    Returns:
        A dict containing the student name, score, letter grade,
        pass/fail status, and honour roll eligibility.
    """
    # Determine letter grade
    if   score >= 90: letter = "A"
    elif score >= 80: letter = "B"
    elif score >= 70: letter = "C"
    elif score >= 60: letter = "D"
    else:            letter = "F"

    return {
        "name":       name,
        "score":      score,
        "grade":      letter,
        "passed":     score >= PASSING_SCORE,   # use named constant
        "honour_roll": score >= HONOUR_ROLL,
    }

students = [
    ("Maria", 92),

    ("Ana",   45),
]

for student_name, student_score in students:
    result = classify_student(student_name, student_score)
    flag   = "🏆" if result["honour_roll"] else ("✅" if result["passed"] else "❌")

CAPSTONE — Student Grade Book 
"""Student Grade Book — capstone demonstrating Python basics."""

PASS_MARK    = 50
DISTINCTION  = 75
CREDIT       = 65

students_data = [
    ("Alice Reyes",   [88, 92, 79, 95]),
    ("Ben Cruz",      [62, 58, 71, 65]),
    ("Carla Santos",  [45, 52, 48, 50]),
    ("Dan Liwanag",   [97, 98, 96, 99]),
    ("Eva Mendoza",   [33, 41, 38, 44]),
]

def compute_average(scores: list) -> float:
    """Return the mean of a list of numeric scores."""
    return sum(scores) / len(scores)

def letter_grade(avg: float) -> str:
    """Map an average score to a letter grade."""
    if   avg >= 90: return "A"
    elif avg >= 80: return "B"
    elif avg >= 70: return "C"
    elif avg >= 60: return "D"
    else:            return "F"

def status(avg: float) -> str:
    """Return a descriptive status string."""
    if   avg >= DISTINCTION : return "Distinction 🏆"
    elif avg >= CREDIT      : return "Credit ✅"
    elif avg >= PASS_MARK   : return "Pass ✅"
    else                    : return "Fail ❌"

# Print report
title   = " SEMESTER GRADE REPORT "
print(f"\n{'=' * 58}")
print(f"{title:^58}")
print(f"{'=' * 58}")
print(f"{'Student':<16} {'Avg':>6}  {'Ltr':^5}  {'Status'}")
print(f"{'-' * 58}")

all_averages = []
for name, scores in students_data:
    avg     = compute_average(scores)
    grade   = letter_grade(avg)
    outcome = status(avg)
    all_averages.append(avg)
    print(f"{name:<16} {avg:>6.1f}  {grade:^5}  {outcome}")

class_avg = compute_average(all_averages)
print(f"{'=' * 58}")
print(f"{'Class Average':<16} {class_avg:>6.1f}  {letter_grade(class_avg):^5}  {status(class_avg)}")
print(f"\nHighest: {max(all_averages):.1f}   Lowest: {min(all_averages):.1f}")
passing = sum(1 for a in all_averages if a >= PASS_MARK)
print(f"Passing: {passing}/{len(all_averages)} students "
      f"({passing/len(all_averages):.0%})")

12.0 COMMON BEGINNER PITFALLS AND HOW TO AVOID THEM

MistakeProblem DescriptionCorrect Approach
Using = instead of ==Writing if x = 5: — a single equals sign is assignment, not comparison. Python raises a SyntaxError immediately, which actually makes this easy to catch.Use == for equality comparison: if x == 5:. Use is only when comparing identity (e.g. against None).
Forgetting to convert input()input() always returns a string. Writing age = input("Age: ") and then age + 5 raises a TypeError because you cannot add a string and an integer.Explicitly convert: age = int(input("Age: ")). Wrap in try/except ValueError to handle non-numeric input gracefully.
Off-by-one in range()range(1, 10) produces 1 through 9 — not 10. The stop value is exclusive. range(10) produces 0 through 9. Misunderstanding this leads to loops that run one too few or one too many times.Remember: range(start, stop) includes start but excludes stop. To include stop, use range(start, stop + 1).
Mutable default argumentsDefining def f(items=[]): creates a single list that is shared across every call. Appending to it in one call permanently modifies it for all future calls — a subtle and painful bug.Use None as the default and create the object inside the function: def f(items=None): items = items or [].
Comparing floats with ==Due to binary floating-point representation, 0.1 + 0.2 == 0.3 is False. Direct equality comparisons between floats are unreliable and should be avoided.Use abs(a - b) < 1e-9 or math.isclose(a, b) to compare floats within a tolerance.
Modifying a list while iteratingUsing a for loop to remove items from a list while iterating over it causes items to be skipped because the list's indices shift during removal.Iterate over a copy: for item in my_list[:]:, use a list comprehension to build a filtered copy, or use filter().

13.0 KNOWLEDGE CHECK: PYTHON BASICS QUIZ

1. What is the difference between = and == in Python?
Click to reveal answer
The single equals sign = is the assignment operator — it stores a value into a variable, for example x = 5. The double equals sign == is the equality comparison operator — it tests whether two values are equal and returns a boolean, for example x == 5 returns True. Using = inside an if condition is a SyntaxError in Python.
2. What will print(type(True)) output, and why?
Click to reveal answer
It outputs <class 'bool'>. Python's bool type has exactly two instances: True and False. However, bool is a subclass of int, which means isinstance(True, int) returns True and True + True evaluates to 2.
3. What does range(2, 10, 3) produce?
Click to reveal answer
range(start, stop, step) generates integers starting at start, incrementing by step, and stopping before stop. So range(2, 10, 3) produces the sequence 2, 5, 8. The value 11 is not included because it would exceed the stop value of 10.
4. Why should you never use a mutable object as a function's default argument?
Click to reveal answer
Default argument values are evaluated once when the function is defined, not each time the function is called. A mutable default (like a list or dictionary) is therefore a single shared object — modifications made during one call persist into all subsequent calls. The fix is to use None as the default and create the mutable object inside the function body.
5. What is the result of 17 // 5 and 17 % 5?
Click to reveal answer
17 // 5 is floor division, which returns the largest integer less than or equal to the exact quotient: 3 (since 17 ÷ 5 = 3.4, floor is 3). 17 % 5 is the modulo operator, which returns the remainder after division: 2 (since 3 × 5 = 15, and 17 − 15 = 2). These two operators are related by the identity a == (a // b) * b + (a % b).
6. What does a for–else construct do in Python?
Click to reveal answer
In Python, a for loop can have an optional else clause. The else block runs only if the loop completes normally — that is, without encountering a break statement. If the loop is exited early by break, the else block is skipped. This is useful in search loops: if the item is found, break exits; if the loop exhausts all items without finding it, else handles the "not found" case.
