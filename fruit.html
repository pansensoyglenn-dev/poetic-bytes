01 — The Foundation

Understanding the Fruit Retail Program

📘 A Real-World Python Learning Experience

Programming becomes meaningful when it solves real problems. The Philippine Fruit Retail Business simulation is a perfect beginner-to-intermediate Python project because it touches nearly every core concept you'll use in production code.

This program uses dictionaries for storing inventory, functions for organizing logic, loops for user interaction, exception handling for crash prevention, and arithmetic for business calculations.

Every code example in this guide is fully interactive — you can copy and run each snippet right in your browser using the Pyodide Python runtime. No setup required.

  • while True + break — user-driven indefinite loops

A command-line fruit market simulation. Users browse Philippine fruit prices, add items to a cart, apply discount codes, and receive a formatted receipt — all built with core Python, no external libraries.

02 — Full Source Code

Complete Implementation

Copy and save as fruit_retail.py — or Run it directly in your browser.

⚠️ Interactive program: this code calls input() several times. When you press Run, your browser will pop up prompt dialogs asking for the fruit, quantity, and discount code. Answer each prompt and the receipt will print once the program finishes.

python · fruit_retail.py 
# Philippine Fruit Retail Business Simulation
# Core Python — no external libraries required

fruits = {
    "mangoes":    180.00,
    "papayas":    75.00,
    "rambutan":   180.00,
    "coconuts":   50.00,
    "lanzones":   120.00,
    "mangosteen": 135.00,
    "bananas":    60.00,
    "pineapples": 75.00,
    "guavas":     50.00,
    "guyavanos":  130.00,
    "avocados":   120.00
}

def display_fruits():
    print("Available Fruits and Prices (per kg in PHP):")
    for fruit, price in fruits.items():
        print(f"{fruit.capitalize()}: {price:.2f}")

def get_discount():
    codes = {"SALE10": 10, "FRESH15": 15, "FRUIT20": 20}
    while True:
        ans = input("Do you have a discount code? (yes/no): ").lower()
        if ans == 'yes':
            code = input("Enter your discount code: ").upper()
            if code in codes:
                discount = codes[code]
                print(f"Discount code accepted! {discount}% off applied.")
                return discount
            else:
                print("Invalid discount code. No discount applied.")
                return 0.0
        elif ans == 'no':
            return 0.0
        else:
            print("Please enter 'yes' or 'no'.")

def calculate_total():
    total_cost = 0.0
    shopping_cart = []

    while True:
        display_fruits()
        choice = input("\nEnter the fruit you want to buy (or 'done' to finish): ").lower()

        if choice == 'done':
            break

        if choice not in fruits:
            print("Invalid fruit selection.")
            continue

        try:
            quantity = float(input(f"Enter quantity in kg for {choice}: "))
            if quantity <= 0:
                raise ValueError
        except ValueError:
            print("Invalid quantity. Please enter a positive number.")
            continue

        cost = fruits[choice] * quantity
        shopping_cart.append((choice, quantity, cost))
        total_cost += cost
        print(f"Added {quantity} kg of {choice} for {cost:.2f} PHP.")

    if shopping_cart:
        print("\nShopping Cart Summary:")
        for item in shopping_cart:
            print(f"{item[0].capitalize()}: {item[1]} kg = {item[2]:.2f} PHP")
        discount_percent = get_discount()
        discount_amount = total_cost * (discount_percent / 100)
        final_total = total_cost - discount_amount
        print(f"\nSubtotal: {total_cost:.2f} PHP")
        if discount_percent > 0:
            print(f"Discount ({discount_percent}%): -{discount_amount:.2f} PHP")
        print(f"Total Cost: {final_total:.2f} PHP")
    else:
        print(f"\nTotal Cost: {total_cost:.2f} PHP")

if __name__ == "__main__":
    print("Welcome to the Philippine Fruit Retail Business!")
    calculate_total()
    print("Thank you for shopping with us!")
03 — Step-by-Step Explanation

8 Concepts, Fully Explained

Each section covers: Why · Snippet · How it operates · Significance

🔹 STEP 01 — Dictionary: Storing Fruits & Prices

fruits = {
    "mangoes":    180.00,
    "papayas":    75.00,
    "rambutan":   180.00,
    "coconuts":   50.00,
    "lanzones":   120.00,
    "mangosteen": 135.00,
    "bananas":    60.00,
    "pineapples": 75.00,
    "guavas":     50.00,
    "guyavanos":  130.00,
    "avocados":   120.00
}
📝 Why This Matters
  1. Why a dictionary? A dictionary binds each fruit name to its price as a single key-value pair. Lookup is O(1) constant time — instant regardless of how many fruits exist.
  2. How it operates: Python stores this as a hash table internally. fruits["mangoes"] retrieves 180.00 in O(1) time — no looping required.
  3. Significance: Speed and simplicity. If we used a list instead, we'd have to loop through every item to find a match — an O(n) operation.

🔹 STEP 02 — display_fruits(): Iteration & Formatting

def display_fruits():
    print("Available Fruits and Prices (per kg in PHP):")
    for fruit, price in fruits.items():
        print(f"{fruit.capitalize()}: {price:.2f}")
📝 Why This Matters
  1. Why isolate display logic? The display function can be called at the start of every shopping loop without duplicating code.
  2. How it operates: fruits.items() returns each key-value pair as a tuple. The for fruit, price in ... line unpacks that tuple into two named variables. {price:.2f} formats the float to exactly 2 decimal places.
  3. Significance: Reusability and separation of concerns. If the display format ever changes, you change it in one place only.

🔹 STEP 03 — while True: Continuous Shopping Loop

while True:
    display_fruits()
    choice = input("\nEnter fruit or 'done' to finish: ").lower()

    if choice == 'done':
        break

    if choice not in fruits:
        print("Invalid selection.")
        continue
📝 Why This Matters
  1. Why while True? The number of items a customer wants is unknown in advance. The program should keep asking until the user explicitly decides to stop.
  2. How it operates: while True creates an infinite loop. The only exit is break when the user types "done". continue skips the rest of the loop and returns to the top.
  3. Significance: User-driven flow control. The while True + break pattern hands control to the user — the program ends only when they say so.

🔹 STEP 04 — try/except: Input Validation

try:
    quantity = float(input(f"Enter quantity in kg for {choice}: "))
    if quantity <= 0:
        raise ValueError
except ValueError:
    print("Invalid quantity. Please enter a positive number.")
    continue
📝 Why This Matters
  1. Why try/except? input() always returns a string. Converting with float() crashes the program if the user types anything non-numeric.
  2. How it operates: Python attempts float(...). If that fails, Python raises ValueError automatically. If float() succeeds but the result is ≤ 0, we manually raise ValueError. Either way, the except block catches it, prints an error, and continue restarts the loop.
  3. Significance: Crash prevention. Without this block, a single bad keystroke would terminate the entire program — losing all cart data.

🔹 STEP 05 — Shopping Cart: List of Tuples

cost = fruits[choice] * quantity
shopping_cart.append((choice, quantity, cost))
total_cost += cost

for item in shopping_cart:
    print(f"{item[0].capitalize()}: {item[1]} kg = {item[2]:.2f} PHP")
📝 Why This Matters
  1. Why a tuple? Each purchase is a fixed 3-part record. Tuples are immutable — once created, that record cannot be accidentally changed.
  2. Why a list? Lists are ordered and dynamic, allowing unlimited items to be appended. .append() adds the tuple as a new element at the end.
  3. Significance: Data integrity and clarity. Immutable tuples inside a mutable list give the best of both worlds — you can grow the cart freely, but individual purchase records stay protected.

🔹 STEP 06 — Discount System: Dictionary Lookup

codes = {"SALE10": 10, "FRESH15": 15, "FRUIT20": 20}
code = input("Enter your discount code: ").upper()

if code in codes:
    discount = codes[code]
    print(f"Discount code accepted! {discount}% off applied.")
    return discount
else:
    print("Invalid discount code. No discount applied.")
    return 0.0
📝 Why This Matters
  1. Why a dictionary for discounts? Code validation requires fast, exact key lookups. Using a dictionary instead of a chain of if/elif statements means the logic stays the same whether there are 3 codes or 300.
  2. How it operates: if code in codes is a safe O(1) check that will never raise a KeyError. .upper() makes codes case-insensitive from the user's perspective.
  3. Significance: Scalability and maintainability. Adding a new discount code only requires one new line in the dictionary — no logic changes needed.

🔹 STEP 07 — Discount Calculation: Arithmetic

discount_percent = get_discount()
discount_amount  = total_cost * (discount_percent / 100)
final_total      = total_cost - discount_amount

print(f"Subtotal:           {total_cost:.2f} PHP")
if discount_percent > 0:
    print(f"Discount ({discount_percent}%): -{discount_amount:.2f} PHP")
print(f"Total Cost:         {final_total:.2f} PHP")
📝 Why This Matters
  1. Why split the calculation? Breaking it into named variables makes the math readable and easy to debug. A customer can see exactly how much was deducted and why.
  2. How it operates: discount_percent / 100 converts e.g. 15 → 0.15. Multiplying by total_cost gives the peso amount to deduct. Subtracting yields the final price.
  3. Significance: Transparency and auditability. This mirrors how real POS (point-of-sale) receipt systems work — showing subtotal, discount, and grand total as separate line items.

🔹 STEP 08 — if __name__ == "__main__": Entry Guard

if __name__ == "__main__":
    print("Welcome to the Philippine Fruit Retail Business!")
    calculate_total()
    print("Thank you for shopping with us!")
📝 Why This Matters
  1. Why the guard? Without it, the moment another Python file does import fruit_retail, the program would immediately launch the interactive session — an unintended side effect.
  2. How it operates: Python sets __name__ to "__main__" when a file is run directly. When imported, it's set to the module name. The guard means the program only runs in the first case.
  3. Significance: Reusability and testability. This single pattern transforms the script into a proper module. It is considered mandatory in professional Python development.

06 — Common Mistakes

Avoid These Pitfalls

❌ Mistake 1 — Using a list instead of a dictionary

Storing fruit names and prices in parallel lists and using list.index() to find a price is O(n) and breaks the moment the lists fall out of sync. A dictionary is always the right choice.

❌ Mistake 2 — Catching bare except:

Writing except: without specifying an exception type will silently swallow every error — including KeyboardInterrupt and memory errors. Always catch the most specific exception type possible.

❌ Mistake 3 — Forgetting if __name__ == "__main__"

Omitting the entry guard means any test file or other module that imports your functions will immediately trigger the full interactive program. Even for small scripts, always wrap top-level execution code in this guard.

❌ Mistake 4 — Using input() without stripping whitespace

A user who types a space before their fruit name — " mangoes" instead of "mangoes" — will get an invalid selection error. Always sanitize user input with .strip().lower().

07 — Deep Dive

Python Concepts, Deeper

Explore these concepts beyond the basics — a structured learning roadmap.

🗄️

Dictionary Internals

Python's dict is a hash table. Understanding hash collisions, load factors, and why insertion order is preserved since Python 3.7 will make you a more confident user of this structure.

🛡️

Exception Hierarchy

ValueError is just one node in Python's exception tree. Learning when to raise TypeError vs a custom exception is essential for writing maintainable code.

🔤

Format Specification Mini-Language

The :.2f in f-strings is part of a powerful format specification mini-language. Mastering it eliminates most manual string manipulation in display code.

📦

Named Tuples

Python's collections.namedtuple or typing.NamedTuple lets you access fields by name — item.fruit instead of item[0] — dramatically improving readability in larger codebases.
