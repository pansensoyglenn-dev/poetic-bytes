🔤 Variables & Data Types

Dynamic typing is one of Python's defining features, enabling rapid development without type boilerplate. It reduces verbosity and makes code more flexible. This snippet serves as the foundation for all subsequent examples. It establishes how data is stored, updated, and inspected, which is essential for every other operation in the program. 🎯 Match Statement (Python 3.10+)

Structural pattern matching – like a powerful switch statement.

value = 42

    case 0:

        print("One")

        print("Other")

match value:
    case n if n > 0:
        print("Positive")
    case n if n < 0:
        print("Negative")
    case _:
        print("Zero")

point = (3, 4)

    case (0, 0):
        print("Origin")
    case (x, 0):
        print(f"On X-axis at {x}")
    case (0, y):
        print(f"On Y-axis at {y}")
    case (x, y):

📘 Comprehensive Explanation The match statement is a modern addition that brings powerful pattern matching to Python, replacing long if-elif chains with cleaner, more expressive code. match compares a value against multiple case patterns. The wildcard _ matches anything. Guards (if after a pattern) add conditions. Structural matching works with tuples, lists, and data classes, extracting values into variables. Pattern matching makes code more declarative and less error-prone. It's particularly useful for parsing, handling different data shapes, and implementing state machines. This snippet demonstrates how to use match for value-based branching, conditional guards, and destructuring complex data structures. 🔄 Closures

A closure is a function that remembers its enclosing scope even when called outside that scope.

def make_multiplier(factor):
    def multiplier(x):
        return x * factor
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))
print(triple(5))

def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

c = counter()
print(c())
print(c())
print(c())
📘 Comprehensive Explanation Closures are a powerful functional programming pattern that enables data hiding and function factories. They are used to create stateful functions without classes. A closure is created when a nested function references variables from its enclosing scope. The nonlocal keyword allows the inner function to modify variables from the outer scope. Each call to the outer function creates a new closure with its own state. Closures enable encapsulation and state management without classes. They are used in decorators, callback functions, and function factories. Closures are used to create specialized functions, maintain private state, and implement function factories throughout the page. The output shows double and triple multipliers working with their captured factors, and the counter incrementing its state across calls.
💡 Closures are powerful for data hiding and function factories. The nonlocal keyword allows modifying variables from the enclosing scope.

🔒 Encapsulation and Properties

Use @property to control access to attributes, enabling getters and setters.

class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("Balance cannot be negative")
        self._balance = value

    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("Deposit amount must be positive")
        self._balance += amount

    def withdraw(self, amount):
        if amount > self._balance:
            raise ValueError("Insufficient funds")
        self._balance -= amount

acc = BankAccount("Glenn", 1000)
print(acc.balance)
acc.deposit(500)
print(acc.balance)
📘 Comprehensive Explanation Encapsulation hides internal state and exposes controlled access. Properties provide a clean way to implement getters and setters without breaking the external interface. The @property decorator turns a method into a read-only attribute. The @balance.setter decorator allows assignment with validation. Private attributes (with a leading underscore) signal internal use. Methods like deposit() and withdraw() provide controlled operations. Encapsulation is a key principle of OOP, protecting internal consistency and enabling validation. Properties provide a Pythonic way to implement it without verbose getter/setter methods. Properties are used to control attribute access, enforce validation, and maintain invariants throughout the page. The output shows the initial balance, the deposit operation increasing the balance, and the property-based access to balance.

🌿 Inheritance

Inheritance allows a class to extend and reuse behaviour from a parent class.

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("Subclass must implement speak()")

    def __str__(self):
        return f"Animal: {self.name}"

class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

dog = Dog("Rex")
cat = Cat("Whiskers")

print(dog.speak())
print(cat.speak())

class GoldenRetriever(Dog):
    def speak(self):
        parent_speak = super().speak()
        return f"{parent_speak} And I'm a Golden Retriever!"

golden = GoldenRetriever("Buddy")
print(golden.speak())
📘 Comprehensive Explanation Inheritance promotes code reuse and establishes hierarchical relationships between classes. A subclass is defined with class Subclass(Parent). The super() function calls methods from the parent class. Abstract methods (with raise NotImplementedError) enforce that subclasses provide implementations. Inheritance is a cornerstone of OOP, enabling polymorphic behavior and code organization. It reduces duplication and makes relationships explicit. Inheritance is used to create specialized classes that extend or override parent behavior throughout the page. The output shows Dog and Cat speaking with their own implementations, and GoldenRetriever extending Dog's speak method with additional text.

🎭 Polymorphism

Polymorphism allows objects of different classes to be treated uniformly through a common interface.

def make_sound(animal):
    return animal.speak()

print(make_sound(dog))
print(make_sound(cat))

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)

rect = Rectangle(5, 3)
print(rect.area())
print(rect.perimeter())
📘 Comprehensive Explanation Polymorphism enables writing code that works with objects of different types through a common interface, making systems more flexible and extensible. Duck typing allows any object with a speak() method to be passed to make_sound(). Abstract base classes (ABC) define interfaces that concrete subclasses must implement. The @abstractmethod decorator enforces implementation. Polymorphism is what makes OOP powerful — it allows substitution and extensibility without modifying existing code. ABCs provide formal interface contracts. Polymorphism is used throughout the page to write generic code that works with various types. The output shows that both Dog and Cat can be passed to make_sound(), and that Rectangle correctly computes area and perimeter.
⚡ Advanced Python

🎨 Decorators

Decorators are functions that modify or extend the behaviour of other functions.

import time
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end - start:.4f} seconds")
        return result
    return wrapper

def retry(times=3, delay=1):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            attempts = 0
            while attempts < times:
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    attempts += 1
                    if attempts == times:
                        raise
                    time.sleep(delay)
        return wrapper
    return decorator

@timer
def slow_function():
    time.sleep(1)
    return "Done"

@retry(times=3, delay=0.5)
def unreliable_function():
    import random
    if random.random() < 0.7:
        raise ValueError("Random failure")
    return "Success!"

🔮 Metaclasses (Advanced)

class Meta(type):
    def __new__(cls, name, bases, dct):
        dct["version"] = "1.0"
        return super().__new__(cls, name, bases, dct)

    def __call__(cls, *args, **kwargs):
        instance = super().__call__(*args, **kwargs)
        instance.created_at = time.time()
        return instance

class MyClass(metaclass=Meta):

class Singleton(type):
    _instances = {}
    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

🧠 Regular Expressions (re)

Regular expressions are powerful for pattern matching and text processing.

import re

text = "Contact me at glenn@example.com or call +1-555-1234"

email_pattern = r"[\w.+-]+@[\w-]+\.\w+"
emails = re.findall(email_pattern, text)
print(emails)

phone_pattern = r"\+\d-\d{3}-\d{4}"
phones = re.findall(phone_pattern, text)
print(phones)

new_text = re.sub(email_pattern, "[REDACTED]", text)
print(new_text)

pattern = re.compile(r"\b\w+\b")
words = pattern.findall(text)
print(words)

date_pattern = re.compile(r"(?P\d{4})-(?P\d{2})-(?P\d{2})")
match = date_pattern.search("2026-06-18")
if match:
    print(match.group("year"))
    print(match.group("month"))
    print(match.group("day"))

html = "
Content 1
Content 2
"
greedy = re.findall(r"
.*
", html)
non_greedy = re.findall(r"
.*?
", html)

📘 Comprehensive Explanation Regular expressions are the most powerful tool for pattern-based text processing. Regex is a critical tool for text processing in any language. Python's re module provides a comprehensive regex implementation. Regex is used for text validation, extraction, and transformation throughout the page. SQLite is a lightweight, file-based database that comes with Python.

import sqlite3

conn = sqlite3.connect("example.db")
cursor = conn.cursor()

cursor.execute("""
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    age INTEGER,
    email TEXT UNIQUE
)
""")

cursor.execute("INSERT INTO users (name, age, email) VALUES (?, ?, ?)",
               ("Glenn", 28, "glenn@example.com"))
conn.commit()

users = [
    ("Alice", 25, "alice@example.com"),
    ("Bob", 30, "bob@example.com")
]
cursor.executemany("INSERT INTO users (name, age, email) VALUES (?, ?, ?)", users)
conn.commit()

cursor.execute("SELECT * FROM users")
rows = cursor.fetchall()
for row in rows:
    print(row)

cursor.execute("SELECT * FROM users WHERE age > ?", (25,))
rows = cursor.fetchall()
print(rows)

cursor.execute("UPDATE users SET age = ? WHERE name = ?", (29, "Glenn"))
conn.commit()

cursor.execute("DELETE FROM users WHERE name = ?", ("Bob",))
conn.commit()

conn.close()

SQLite is a zero-configuration database that's perfect for prototyping and small to medium applications. Python's built-in support makes it easy to get started. 📝 Context Managers with contextlib

The contextlib module provides utilities for creating context managers.

from contextlib import contextmanager, suppress, closing
import urllib.request

with suppress(FileNotFoundError):
    with open("nonexistent.txt") as f:
        content = f.read()

with closing(urllib.request.urlopen("https://example.com")) as response:
    data = response.read()

@contextmanager
def timed(name):
    print(f"Starting {name}")
    start = time.time()
    try:
        yield
    finally:
        end = time.time()
        print(f"{name} took {end - start:.2f} seconds")

with timed("sleep"):
    time.sleep(1)

from contextlib import ExitStack

with ExitStack() as stack:
    files = [
        stack.enter_context(open("file1.txt", "w")),
        stack.enter_context(open("file2.txt", "w"))
    ]
    for f in files:
        f.write("Hello")
📘 Comprehensive Explanation contextlib provides additional context manager utilities for common patterns like suppression, closing, and stacking. suppress() ignores specified exceptions. closing() calls close() on exit. @contextmanager turns a generator into a context manager. ExitStack manages multiple context managers programmatically. These utilities reduce boilerplate and make code more robust. They're particularly useful for resource management and cleanup. contextlib utilities are used for resource management, exception suppression, and clean code throughout the page. The output shows the suppress context manager handling a missing file, the closing context manager fetching a URL, the timed context manager measuring execution time, and ExitStack managing multiple files.

🧬 Descriptors

Descriptors control attribute access in classes. They power @property and @classmethod.

class PositiveNumber:
    def __set_name__(self, owner, name):
        self.name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return obj.__dict__.get(self.name, 0)

    def __set__(self, obj, value):
        if value < 0:
            raise ValueError(f"{self.name} must be positive")
        obj.__dict__[self.name] = value

class Product:
    price = PositiveNumber()
    quantity = PositiveNumber()

    def __init__(self, price, quantity):
        self.price = price
        self.quantity = quantity

p = Product(10, 5)
print(p.price)

class LazyAttribute:
    def __init__(self, func):
        self.func = func

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        value = self.func(obj)
        setattr(obj, self.func.__name__, value)
        return value

class Circle:
    def __init__(self, radius):
        self.radius = radius

    @LazyAttribute
    def area(self):
        print("Calculating area...")
        return 3.14159 * self.radius ** 2

c = Circle(5)
print(c.area)
print(c.area)
📘 Comprehensive Explanation Descriptors are the mechanism behind Python's property system. They enable reusable attribute logic across multiple classes. __set_name__ captures the attribute name. __get__ retrieves the value, __set__ validates and sets it. Data descriptors have both __get__ and __set__. Non-data descriptors only have __get__ and are used for lazy attributes. Descriptors are a fundamental but often overlooked feature of Python. They power properties, class methods, and static methods. Descriptors are used for validation, lazy loading, and attribute management throughout the page. The output shows the price (10) from the PositiveNumber descriptor, and the lazy area calculation that runs only once (the second access returns the cached value).
