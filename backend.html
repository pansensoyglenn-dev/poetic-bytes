What is Backend Development?

The server‑side logic that powers web applications

Backend development handles everything that happens on the server: processing requests, interacting with databases, handling authentication, and serving responses. Python is a top choice for backend thanks to frameworks like Flask (lightweight), Django (full‑stack), and FastAPI (modern async).

Backend Responsibilities

1. Handle HTTP requests → 2. Process data → 3. Access databases → 4. Return responses (JSON, HTML, etc.). This course covers all with practical examples.

🌐

Web Frameworks

Flask for simplicity, Django for batteries‑included, FastAPI for modern async APIs. Choose based on your project needs.

🗄️

Databases

SQLite for development, PostgreSQL for production. SQLAlchemy and Django ORM abstract away SQL with Python classes.

🔐

Authentication

JWT (JSON Web Tokens), sessions, OAuth2 — secure your APIs and web apps.

Deployment

Flask — Lightweight Microframework

Simple, flexible, and beginner‑friendly

🏗️ Basic Flask App

Flask is minimal — you build up exactly what you need. Route decorators map URLs to functions.

from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route("/")
def home():
    return "<h1>Welcome to Flask!</h1>"

@app.route("/api/hello", methods=["GET"])
def hello():
    return jsonify({"message": "Hello, Flask!"})

@app.route("/user/<name>")
def user(name):
    return jsonify({"user": name})

@app.route("/api/data", methods=["POST"])
def data():
    data = request.json
    return jsonify({"received": data}), 201

if __name__ == "__main__":
    app.run(debug=True)
Flask app running on http://127.0.0.1:5000
GET / → <h1>Welcome to Flask!</h1>
GET /api/hello → {"message": "Hello, Flask!"}
GET /user/Alice → {"user": "Alice"}
💡 @app.route binds a URL to a function. Flask uses decorators for routing — elegant and clear.

📦 Flask with Templates (Jinja2)

Flask uses Jinja2 for server‑side rendering. Render HTML templates with dynamic data.

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/profile/<username>")
def profile(username):
    user_data = {
        "name": username,
        "age": 30,
        "city": "Berlin"
    }
    return render_template("profile.html", user=user_data)

# templates/profile.html
# <h1>{{ user.name }}</h1>
# <p>Age: {{ user.age }}, City: {{ user.city }}</p>
Rendered HTML for profile/Glenn →

Glenn

Age: 30, City: Berlin

💡 Jinja2 syntax uses {{ }} for variables and {% %} for logic (loops, conditionals).

⚙️ Flask Blueprints — Organizing Larger Apps

Blueprints keep your code modular as your app grows. Group related routes into separate files.

# auth.py
from flask import Blueprint, jsonify

auth_bp = Blueprint("auth", __name__)

@auth_bp.route("/login", methods=["POST"])
def login():
    return jsonify({"status": "logged in"})

# app.py
from flask import Flask
from auth import auth_bp

app = Flask(__name__)
app.register_blueprint(auth_bp, url_prefix="/api/auth")
Blueprint registered at /api/auth/login → {"status": "logged in"}

Django — Full‑Stack Framework

Batteries‑included: ORM, admin panel, authentication, and more

🧩 Django Models (ORM)

Define your database schema as Python classes. Django generates SQL migrations automatically.

from django.db import models

class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    email = models.EmailField(unique=True)
    age = models.IntegerField(null=True, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.username

# In admin.py
from django.contrib import admin
admin.site.register(User)

# Run migrations
python manage.py makemigrations
python manage.py migrate
Model User created. Migrations generated and applied.
💡 Django's ORM is powerful — you can query with User.objects.filter(age__gt=18).

📊 Django REST Framework (DRF)

Build REST APIs with serializers, viewsets, and routers — the industry standard for Django APIs.

from rest_framework import serializers, viewsets, routers
from .models import User

class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ["id", "username", "email", "age"]

class UserViewSet(viewsets.ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer

# urls.py
router = routers.DefaultRouter()
router.register("users", UserViewSet)
urlpatterns += router.urls
DRF endpoints: /api/users/ (GET, POST), /api/users/{id}/ (GET, PUT, DELETE)
💡 DRF handles serialization, validation, and view logic — you write minimal code for full CRUD APIs.

FastAPI — Modern and Fast

Async, automatic OpenAPI docs, and Pydantic validation

⚡ FastAPI Basics

FastAPI uses Python type hints to validate request data and generate interactive API docs automatically.

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Optional

app = FastAPI()

class Item(BaseModel):
    name: str
    price: float
    tax: Optional[float] = None

@app.get("/")
async def root():
    return {"message": "FastAPI is running"}

@app.post("/items/")
async def create_item(item: Item):
    total = item.price + (item.tax or 0)
    return {"name": item.name, "total": total}

@app.get("/items/{item_id}")
async def get_item(item_id: int):
    if item_id not in [1, 2, 3]:
        raise HTTPException(status_code=404, detail="Item not found")
    return {"id": item_id, "name": "Example Item"}
FastAPI app running at /docs for interactive API docs.
POST /items/ → {"name": "Apple", "total": 1.5}
💡 FastAPI automatically generates OpenAPI documentation at /docs and /redoc — zero configuration.

🔧 Dependency Injection

FastAPI's dependency system makes it easy to reuse code for authentication, database sessions, and more.

from fastapi import Depends

def get_db():
    # Simulate DB connection
    db = {"connection": "active"}
    return db

@app.get("/users/")
async def get_users(db: dict = Depends(get_db)):
    return {"db_status": db["connection"], "users": ["Alice", "Bob"]}
GET /users/ → {"db_status": "active", "users": ["Alice", "Bob"]}
💡 Dependencies can be async, cached, and composed — a powerful pattern for clean, testable code.

Databases & ORM

SQLAlchemy, Django ORM, and raw SQL — interacting with databases

🗄️ SQLAlchemy — Python's Most Powerful ORM

SQLAlchemy works with Flask, FastAPI, and any Python app. Define models and query with Python.

from sqlalchemy import create_engine, Column, Integer, String, Float
from sqlalchemy.orm import declarative_base, sessionmaker

Base = declarative_base()

class Product(Base):
    __tablename__ = "products"
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    price = Column(Float, nullable=False)

# Engine and session
engine = create_engine("sqlite:///mydb.sqlite")
Base.metadata.create_all(engine)
Session = sessionmaker(bind=engine)
session = Session()

# Create
product = Product(name="Laptop", price=999.99)
session.add(product)
session.commit()

# Query
products = session.query(Product).filter(Product.price > 500).all()
for p in products:
    print(p.name, p.price)
SQLite DB created. Product Laptop added. Query returned Laptop 999.99
💡 SQLAlchemy's ORM is flexible — you can write raw SQL when needed, or use the ORM for most operations.

🐘 PostgreSQL with FastAPI (async)

FastAPI works with async databases. Use asyncpg and SQLAlchemy's async support.

from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import sessionmaker

DATABASE_URL = "postgresql+asyncpg://user:pass@localhost/db"
engine = create_async_engine(DATABASE_URL)
async_session = sessionmaker(engine, class_=AsyncSession)

async def get_db():
    async with async_session() as session:
        yield session

# FastAPI route with async DB
@app.get("/products")
async def list_products(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(Product))
    return result.scalars().all()
Async PostgreSQL connection configured. GET /products returns list.
💡 For production, use PostgreSQL — it's robust and supports advanced features like full‑text search.

📄 JSON:API and Serialization

Serialize models to JSON and validate incoming data. Pydantic (FastAPI) and Marshmallow (Flask) are popular.

from pydantic import BaseModel, Field

class UserCreate(BaseModel):
    username: str = Field(..., min_length=3, max_length=20)
    email: str
    age: int = Field(..., ge=18)

# FastAPI validation
@app.post("/users")
async def create_user(user: UserCreate):
    # user is validated automatically
    return {"status": "created", "user": user}
Pydantic validation: age >= 18, username length 3-20. Invalid data raises 422.

Authentication & Authorization

JWT, OAuth2, and session‑based authentication

🔑 JWT Authentication (FastAPI)

JSON Web Tokens are stateless and widely used for API authentication.

from jose import JWTError, jwt
from datetime import datetime, timedelta

SECRET_KEY = "your-secret-key"
ALGORITHM = "HS256"

def create_access_token(data: dict, expires_delta: timedelta = None):
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:
        expire = datetime.utcnow() + timedelta(minutes=15)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

@app.post("/token")
async def login(username: str, password: str):
    # Validate user (pseudo)
    token = create_access_token(data={"sub": username})
    return {"access_token": token, "token_type": "bearer"}
JWT token generated. Use Bearer token in Authorization header.
💡 Always use HTTPS in production. Never store secrets in code — use environment variables.

🛡️ OAuth2 with FastAPI

FastAPI provides built‑in OAuth2 flows for password and authorization code grants.

from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

@app.post("/token")
async def login(form_data: OAuth2PasswordRequestForm = Depends()):
    # Validate form_data.username, form_data.password
    token = create_access_token({"sub": form_data.username})
    return {"access_token": token, "token_type": "bearer"}

async def get_current_user(token: str = Depends(oauth2_scheme)):
    # decode token, get user
    return {"username": "demo"}

@app.get("/users/me")
async def read_users_me(current_user = Depends(get_current_user)):
    return current_user
OAuth2 flow with /token endpoint. Protected route /users/me requires token.

Deployment

Taking your backend live — Gunicorn, Docker, and cloud platforms

🚀 Gunicorn — Production WSGI Server

Flask and Django apps need a production server. Gunicorn is the standard WSGI server for Python.

# Install
pip install gunicorn

# Run Flask app
gunicorn -w 4 -b 0.0.0.0:8000 app:app

# For Django
gunicorn myproject.wsgi:application

# With environment variables
GUNICORN_CMD_ARGS="--workers=4 --timeout=60" gunicorn app:app
Gunicorn serving app on port 8000 with 4 workers.
💡 Use gunicorn -w 4 for 4 worker processes — adjust based on your CPU cores.

🐳 Docker — Containerization

Docker packages your app and dependencies into a container for consistent deployment everywhere.

# Dockerfile for FastAPI
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

# Build and run
docker build -t myapp .
docker run -p 8000:8000 myapp
Docker image built. App running on http://localhost:8000
💡 Use docker-compose to run your app with PostgreSQL, Redis, and other services.

Backend FAQ

Quick answers to common questions

Which framework should I learn first?

+

Start with Flask — it's minimal and you learn core concepts. Then move to FastAPI for modern async APIs, and Django when you need a full‑stack framework.

What's the difference between Flask and Django?

+

Flask is a microframework — you add only what you need. Django is batteries‑included — it comes with an ORM, admin, authentication, forms, and more. Flask gives more flexibility, Django gives more structure.

What is an ORM?

+

Object‑Relational Mapper — it lets you work with databases using Python classes instead of raw SQL. Examples: SQLAlchemy (Flask/FastAPI) and Django ORM.

How do I handle authentication securely?

+

Use JWT for stateless APIs (with short expiration and refresh tokens), or session‑based authentication with HTTP‑only cookies. Always use HTTPS and hash passwords with bcrypt or Argon2.

How do I deploy my backend?

+

Use Gunicorn (WSGI) or Uvicorn (ASGI) with a reverse proxy like Nginx. For simplicity, use platforms like Render, Railway, or Heroku for easy deployment.
