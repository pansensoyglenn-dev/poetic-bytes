📖 What Are Python Strings?

A string in Python is a sequence of characters enclosed in quotes. They are one of the most fundamental and widely used data types — everything from user input, file contents, web pages, and API responses comes as strings.

Why Strings Matter

Strings are the universal medium of communication between humans and computers. Every piece of text you see on a screen, every file you read, every message you send — all are strings. In Python, strings are immutable (cannot be changed in place), which means every operation creates a new string. This design makes them safe and predictable.

This tutorial covers everything you need: creating strings, using escape sequences, slicing, powerful methods, formatting with f‑strings, and advanced topics like Unicode and regular expressions. 🔤 String Basics – Creation, Quotes, and Operations

The foundation of all string handling

📝 Creating Strings

Single, Double, and Triple Quotes

Use single or double quotes interchangeably. Triple quotes allow multi-line strings and preserve formatting.

# Single and double quotes are equivalent
single = 'Hello'
double = "World"
print(single, double)

# Triple quotes for multi-line strings
multiline = """This is a
multi-line string
that preserves newlines."""
print(multiline)

# Including a quote inside a string
quote = "She said, 'Python is awesome!'"
print(quote)
🔍 Why this code is used: Strings are the primary way to represent text. This example shows the three quoting styles available in Python.

⚙️ How it operates: Single and double quotes work identically — Python treats them the same. Triple quotes (either ''' or """) allow strings to span multiple lines, preserving newline characters. This is useful for docstrings, error messages, or embedded SQL.

📌 Significance: Understanding quoting styles helps you write cleaner code — you can avoid escaping quotes by choosing the right quote style. Triple-quoted strings are essential for documentation and multi-line text blocks.
🔧 Escape Sequences & Raw Strings

Special Characters with Backslashes

Escape sequences let you include newlines (\n), tabs (\t), and quotes inside strings.

# Escape sequences
escaped = "Line1\nLine2\tTabbed"
print(escaped)

# Raw strings (r"...") ignore escape sequences
path = r"C:\Users\Glenn\Documents"
print(path)
🔍 Why this code is used: Escape sequences let you embed special characters (newlines, tabs, quotes) into strings. Raw strings are used when you want backslashes to be literal (e.g., file paths, regex).

⚙️ How it operates: A backslash (\) signals that the next character has special meaning. \n becomes a newline, \t becomes a tab. Raw strings (r"...") disable this processing — every character is taken literally.

📌 Significance: Escape sequences are essential for formatting output (newlines, tabs) and including quotes. Raw strings are a lifesaver for Windows file paths and regular expressions, where backslashes are common and escaping them would create unreadable code.
➕ Concatenation & Repetition

Joining and Repeating Strings

Use + to join and * to repeat.

first = "Py"
last = "thon"
full = first + last
print(full)

repeat = "Ha" * 3
print(repeat)
🔍 Why this code is used: Concatenation and repetition are fundamental string operations used to build strings dynamically.

⚙️ How it operates: + creates a new string by joining two strings. * repeats the string a specified number of times. Both operations return new strings — they don't modify the originals (strings are immutable).

📌 Significance: These operations are the building blocks of string construction. However, for joining many strings (e.g., in a loop), ''.join(list) is far more efficient than repeated + because it allocates memory only once.
🔍 Indexing & Slicing

Accessing Characters and Substrings

Strings are sequences – you can index with [i] and slice with [start:stop:step].

s = "Python"
print(s[0])    # P
print(s[-1])   # n (last character)
print(s[1:4])  # yth (characters 1,2,3)
print(s[::-1]) # nohtyP (reverse)
🔍 Why this code is used: Indexing and slicing let you extract specific characters or substrings from a string — essential for text parsing and data extraction.

⚙️ How it operates: Indexes start at 0. Negative indexes count from the end (-1 is the last character). Slices use [start:stop:step] — start is inclusive, stop is exclusive. Omitting values uses defaults: [:5] = first 5, [3:] = from index 3 to end.

📌 Significance: Slicing is one of Python's most elegant features. It's used everywhere — from extracting file extensions (filename[-4:]) to reversing strings (s[::-1]) to parsing data. Mastering slicing makes your code concise and expressive.
⚖️ Comparison & Membership

Comparing Strings and Testing "Contains"

Strings compare lexicographically (character by character, by Unicode code point), and you can test membership with in.

print("apple" == "apple")     # True
print("apple" < "banana")    # True ('a' < 'b')
print("Apple" == "apple")    # False (case-sensitive)
print("cat" in "concatenate") # True
print("dog" not in "concatenate") # True

# Case-insensitive comparison
a, b = "Python", "PYTHON"
print(a.casefold() == b.casefold())  # True
🔍 Why this code is used: Comparisons and membership tests are used constantly — filtering data, validating input, and building conditionals.

⚙️ How it operates: Relational operators (<, >, ==) compare strings character by character based on Unicode code points, similar to dictionary order. in/not in check for substring presence, not just exact matches.

📌 Significance: Because comparisons are case-sensitive, always normalise with .lower() or .casefold() before comparing user-facing text like usernames or search terms.

📋 Essential String Methods (45+)

Python's string methods are powerful and extensive – here are the most frequently used ones

🔡 Case Conversion
text = "  hello WORLD  "
print(text.upper())     # "  HELLO WORLD  "
print(text.lower())     # "  hello world  "
print(text.title())     # "  Hello World  "
print(text.capitalize())# "  Hello world  "
print(text.swapcase())  # "  HELLO world  "
print(text.casefold())  # aggressive lower for case-insensitive comparison
🔍 Why this code is used: Case conversion methods normalize text for comparison, display, or storage. They're used in search, user input validation, and formatting.

⚙️ How it operates: Each method returns a new string with the case transformed. title() capitalises each word, capitalize() only the first character. casefold() is more aggressive than lower() — it handles special cases like German "ß" → "ss", making it ideal for case-insensitive comparisons.

📌 Significance: Case conversion is critical for user input normalization (e.g., email addresses), search functionality, and consistent data storage. casefold() is the recommended method for case-insensitive equality checks.
🧹 Stripping Whitespace
messy = "  \t\n  Python  \n\t  "
clean = messy.strip()
print(clean)           # "Python"
print(messy.lstrip())  # left strip
print(messy.rstrip())  # right strip
# You can also strip specific characters
s = "---hello---"
print(s.strip("-"))    # "hello"
🔍 Why this code is used: User input, file data, and API responses often contain extra whitespace. Stripping removes it, making data clean and consistent.

⚙️ How it operates: strip() removes leading and trailing whitespace (spaces, tabs, newlines). lstrip() and rstrip() target one side. Passing an argument strips those specific characters instead of whitespace.

📌 Significance: Stripping is one of the most commonly used string operations. It prevents errors from accidental spaces in user input and ensures data integrity when reading from files or databases.
🔪 Split & Join
csv = "apple,banana,cherry"
items = csv.split(",")
print(items)              # ['apple', 'banana', 'cherry']

# Split with max splits
parts = "a,b,c,d".split(",", 2)
print(parts)              # ['a', 'b', 'c,d']

# rsplit (splits from the right)
rs = "a,b,c,d".rsplit(",", 2)
print(rs)                # ['a,b', 'c', 'd']

# Join
fruits = ["apple", "banana", "cherry"]
joined = " | ".join(fruits)
print(joined)            # "apple | banana | cherry"
🔍 Why this code is used: split() and join() are inverse operations used to parse and construct strings. They're essential for CSV parsing, URL parsing, and building output.

⚙️ How it operates: split(delim, maxsplit) breaks a string into a list. rsplit() starts from the right — useful for extracting the last part (e.g., file extension). join(iterable) concatenates strings with the separator. join is called on the separator, not the list.

📌 Significance: join() is the most efficient way to concatenate many strings. It allocates memory once, unlike repeated + which creates multiple intermediate strings. split() is the go-to for parsing delimited data like CSVs, log files, and command-line arguments.
🔍 Search & Replace
s = "hello world, hello python"
print(s.find("hello"))      # 0
print(s.rfind("hello"))     # 13 (last occurrence)
print(s.index("world"))     # 6
print(s.count("l"))          # 5
print(s.replace("hello", "hi")) # "hi world, hi python"
# Replace only first 2 occurrences
print(s.replace("hello", "hi", 2))
🔍 Why this code is used: Searching and replacing are fundamental for text processing — from finding keywords to cleaning data.

⚙️ How it operates: find() returns the first index or -1 if not found. index() is similar but raises a ValueError on failure. count() counts non-overlapping occurrences. replace() substitutes all occurrences (or a limited number if the third argument is provided).

📌 Significance: find() is used for conditional checks (e.g., "does this string contain X?"). replace() is used for data cleaning, template filling, and sanitization. Choose find() over index() when you want to handle "not found" gracefully.
✔️ Checks & Predicates
s = "Python123"
print(s.isalpha())   # False (contains digits)
print(s.isalnum())   # True (letters and digits)
print(s.isdigit())   # False
print(s.islower())   # False (has uppercase)
print(s.isspace())   # False
print(s.startswith("Py")) # True
print(s.endswith("23"))  # True
print(s.isascii())   # True (all characters are ASCII)
print("123".isdecimal()) # True
🔍 Why this code is used: Predicate methods validate the content of a string — used in form validation, data cleaning, and input checks.

⚙️ How it operates: Each method returns a boolean based on the string's content. isalnum() checks if all characters are alphanumeric. startswith() and endswith() can also accept tuples for multiple prefixes/suffixes.

📌 Significance: These methods are your first line of defence against invalid input. For example, checking isnumeric() before converting to int(), or using startswith() to filter log lines by level.
📊 Alignment & Padding
s = "42"
print(s.zfill(5))             # "00042"
print(s.rjust(5, "*"))    # "***42"
print(s.ljust(5, "*"))    # "42***"
print(s.center(7, "="))   # "===42==="
🔍 Why this code is used: Padding and alignment are used for formatting output in reports, logs, and terminal applications.

⚙️ How it operates: zfill(width) pads with zeros on the left (useful for numbers). rjust, ljust, and center pad with a specified character (default space) to reach the desired width.

📌 Significance: These methods make it easy to create aligned columns in terminal output, format invoice numbers, or pad IDs. zfill() is particularly useful for zero-padded serial numbers or date components.
🔧 More Useful Methods
# partition – splits at first occurrence
s = "name: John"
left, sep, right = s.partition(":")
print(left, sep, right)  # "name" ":" " John"

# rpartition – from the right
s = "a:b:c"
left, sep, right = s.rpartition(":")
print(left, sep, right)  # "a:b" ":" "c"

# expandtabs – convert tabs to spaces
tabbed = "1\t2\t3"
print(tabbed.expandtabs(4)) # "1   2   3"

# translate – character mapping
trans = str.maketrans("aeiou", "12345")
print("hello".translate(trans)) # "h2ll4"

# format_map – like .format() but with a dict
data = {"name": "Glenn", "age": 30}
print("{name} is {age}".format_map(data))
🔍 Why this code is used: These specialized methods cover specific use cases: parsing key-value pairs, cleaning text, and efficient character replacement.

⚙️ How it operates: partition() returns a tuple of (head, sep, tail) — perfect for splitting on a delimiter while keeping the delimiter. expandtabs() converts tabs to spaces with a specified width. translate() with maketrans() is the fastest way to replace characters one-to-one. format_map() works like .format(**dict).

📌 Significance: partition() is cleaner than split() when you need the delimiter. translate() is the go-to for fast character substitutions (e.g., cleaning punctuation). format_map() is useful when you already have a dictionary and don't want to unpack it.
📏 Line Splitting & Prefix/Suffix Removal

splitlines(), removeprefix(), removesuffix()

Modern additions (Python 3.9+) that make common cleanup one-liners instead of slicing tricks.

log = "line1\nline2\r\nline3"
print(log.splitlines())        # ['line1', 'line2', 'line3']
print(log.splitlines(keepends=True))

# removeprefix / removesuffix (3.9+)
url = "https://example.com"
print(url.removeprefix("https://"))  # "example.com"

filename = "report.csv"
print(filename.removesuffix(".csv"))  # "report"

# Safe: does nothing if the prefix/suffix isn't there
print("data.json".removesuffix(".csv"))  # "data.json" (unchanged)
🔍 Why this code is used: splitlines() correctly handles mixed line endings (\n, \r\n) across platforms, which naive split("\n") does not. removeprefix()/removesuffix() replace error-prone slicing like s[len(prefix):].

⚙️ How it operates: splitlines() splits on any recognized line boundary and, unlike split("\n"), does not produce a trailing empty string. removeprefix()/removesuffix() only remove the target text if it's actually present — otherwise the original string is returned untouched, avoiding accidental data loss.

📌 Significance: These two additions quietly fixed a long list of common bugs. Before 3.9, s[len(prefix):] would silently corrupt strings that didn't start with the expected prefix — removeprefix() makes that mistake impossible.

🎨 String Formatting – f‑strings, .format(), and %

How to create dynamic, readable strings

f‑strings (Python 3.6+) – The Modern Way

Prefix a string with f and embed expressions inside {}.

name = "Glenn"
age = 30
score = 95.678

print(f"Name: {name}, Age: {age}")
print(f"Score: {score:.2f}")        # two decimals
print(f"Hello, {name.upper()}!")    # call methods
print(f"Result: {5 + 3 * 2}")       # expressions

# Format specifiers: alignment, padding, sign, grouping
num = 12345.6789
print(f"Currency: ${num:,.2f}")      # $12,345.68
print(f"Right-aligned: {name:>10}")  # "     Glenn"
print(f"Centered: {name:^10}")       # "  Glenn   "
print(f"Hex: {255:#x}")              # 0xff
print(f"Percent: {0.875:.1%}")        # 87.5%

# Debugging with = (Python 3.8+)
x = 42
print(f"{x=}")  # x=42
🔍 Why this code is used: f‑strings are the preferred way to format strings in modern Python. They're used in logging, error messages, report generation, and UI text.

⚙️ How it operates: f‑strings are evaluated at runtime. Expressions inside {} are executed and their results are converted to strings. Format specifiers after : control alignment (>, <, ^), padding, sign, number grouping (,), and more. The = specifier (Python 3.8+) prints the variable name and its value.

📌 Significance: f‑strings are the fastest and most readable formatting method in Python. They are now the standard in the community. Use them for all new code. The = debug specifier is a game-changer for quick debugging — no more print("x =", x).

The .format() Method

Older but still widely used. Placeholders {} are replaced by arguments.

# Positional arguments
print("{} + {} = {}".format(2, 3, 5))

# Named arguments
print("{name} is {age} years old".format(name="Alice", age=25))

# Formatting numbers
print("{:.2f}".format(3.14159))
print("{:,.2f}".format(12345.6789))
🔍 Why this code is used: .format() is the predecessor to f‑strings. It's still used in many codebases and is more flexible than f‑strings in some cases (e.g., when the format string is dynamic).

⚙️ How it operates: Placeholders {} are replaced by arguments in order or by name. Format specifiers work similarly to f‑strings ({:.2f} for two decimals).

📌 Significance: While f‑strings are preferred, .format() remains useful when the format template is stored in a variable or loaded from a configuration file. It's also used in older codebases, so being able to read it is essential.

%-formatting (Legacy)

Similar to C's printf. Still seen in old codebases.

print("Hello, %s" % "world")
print("Pi = %.2f" % 3.14159)
🔍 Why this code is used: %-formatting is the oldest method, borrowed from C's printf. It's still present in legacy code and some logging libraries.

⚙️ How it operates: The % operator takes a format string on the left and a tuple (or single value) on the right. %s is a string placeholder, %d for integers, %f for floats.

📌 Significance: You'll encounter this in older code and in some libraries that use it for logging (e.g., logging.info("Value: %s", val)). It's less readable and more error-prone than f‑strings, but it's good to recognize it.

📌 Performance Tip

f"-strings" are the fastest formatting method in Python 3.6+. They are also the most readable. Prefer them over .format() and % in modern code.

⚙️ Advanced – Unicode, Encoding, and Regular Expressions

Working with characters beyond ASCII and powerful pattern matching

Unicode & Encoding

Python strings are Unicode by default. You can encode to bytes and decode back.

text = "Café 🐍"
encoded = text.encode("utf-8")
print(encoded)        # b'Caf\xc3\xa9 \xf0\x9f\x90\x8d'

decoded = encoded.decode("utf-8")
print(decoded)        # Café 🐍

# Handling encoding errors
text = "Hello"
print(text.encode("ascii"))                 # works
# text.encode("ascii") would raise for non-ASCII; use errors='ignore'
print("Café".encode("ascii", errors="ignore")) # b'Caf'
🔍 Why this code is used: When reading from/writing to files, sending data over networks, or interfacing with databases, you need to convert between strings and bytes.

⚙️ How it operates: encode() converts a string to bytes using the specified encoding. decode() reverses the process. UTF-8 is the standard encoding on the web. The errors parameter controls how to handle unsupported characters ('ignore', 'replace', 'strict').

📌 Significance: Understanding encoding is critical for internationalization and working with external data. Mismatched encodings are a common source of bugs (e.g., UnicodeDecodeError). UTF-8 is the universal standard — use it unless you have a specific reason not to.

Regular Expressions with re

Pattern matching for advanced string searches and substitutions.

import re

text = "Contact: john@example.com or jane@company.co.uk"
pattern = r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b"
matches = re.findall(pattern, text)
print(matches)      # ['john@example.com', 'jane@company.co.uk']

# Substitute
cleaned = re.sub(r"\d+", "X", "My number is 123-456")
print(cleaned)      # My number is X-X

# Split with regex
parts = re.split(r"[;,\s]+", "apple, banana; cherry")
print(parts)        # ['apple', 'banana', 'cherry']
🔍 Why this code is used: Regular expressions are the most powerful tool for text pattern matching — used in validation, parsing, search, and data extraction.

⚙️ How it operates: The re module provides functions like findall (return all matches), search (find first match), sub (substitute), and split (split by pattern). Patterns use a special syntax: \d = digit, \w = word character, + = one or more, * = zero or more, {2,} = at least 2. Always use raw strings (r"...") for regex to avoid escaping backslashes.

📌 Significance: Regex is the Swiss Army knife of string processing. It can replace dozens of lines of manual parsing with a single line. However, it has a steep learning curve and can be hard to read — use it when the pattern is complex, and comment your regex for readability.

String Immutability & Interning

Understanding what happens in memory when you manipulate strings — and why some strings are shared, and some aren't.

a = "hello"
b = "hello"
print(a is b)          # True — short literals are often interned

c = "".join(["hel", "lo"])
print(a == c, a is c)  # True False — same value, different object

# Immutability in practice: methods never modify in place
name = "glenn"
name.upper()
print(name)          # still "glenn" — upper() returned a NEW string we discarded
name = name.upper()  # reassign to capture the result
print(name)          # "GLENN"
🔍 Why this code is used: This is one of the most common beginner bugs — calling a string method and expecting the original variable to change.

⚙️ How it operates: CPython interns some short, identifier-like string literals so identical values can share memory (is returns True), but this is an implementation detail, not a guarantee — never rely on is for string equality, always use ==. Every string method returns a brand-new string object; the original is untouched.

📌 Significance: Forgetting to reassign the result of a string method (name.upper() instead of name = name.upper()) is a top-5 beginner mistake. Always capture the return value.

🧾 Project 1 – CSV Row Formatter

Turn raw CSV text into an aligned report

Combines split(), list comprehensions, and f‑string padding to build a readable console table from delimited text — a common task when working with exported spreadsheet data.

raw = """name,score,grade
Ana,88,B
Marco,95,A
Liza,72,C"""

rows = [line.split(",") for line in raw.splitlines()]
header, *data = rows

# Compute column widths dynamically
widths = [max(len(row[i]) for row in rows) for i in range(len(header))]

def format_row(row):
    return "  ".join(cell.ljust(w) for cell, w in zip(row, widths))

print(format_row(header))
print("-" * (sum(widths) + 4))
for row in data:
    print(format_row(row))
🔍 Why this code is used: Real CSV data rarely displays nicely by default — this pattern is the basis for any command-line reporting tool.

⚙️ How it operates: Each line is split on commas into a list of cells. Column widths are computed by taking the max cell length per column across all rows. ljust() pads each cell to that width so columns line up regardless of content length.

📌 Significance: This exact pattern — split, measure, pad, join — is how libraries like tabulate work under the hood. Understanding it means you never need external dependencies for simple table output.
🔐 Project 2 – Simple Slug Generator

Convert titles into URL-safe slugs

A blog engine or CMS needs to turn "Python Strings: The Complete Guide!" into "python-strings-the-complete-guide". This mirrors what platforms like WordPress or Ghost do automatically.

import re

def slugify(title: str) -> str:
    slug = title.lower().strip()
    slug = re.sub(r"[^a-z0-9\s-]", "", slug)   # drop punctuation
    slug = re.sub(r"[\s]+", "-", slug)          # spaces to hyphens
    slug = re.sub(r"-+", "-", slug)             # collapse repeats
    return slug.strip("-")

print(slugify("Python Strings: The Complete Guide!"))
# python-strings-the-complete-guide
print(slugify("  Café & Croissants — 2026  "))
# caf-croissants-2026
🔍 Why this code is used: Slugs are used everywhere URLs need to be human-readable and safe — blog posts, product pages, documentation anchors.

⚙️ How it operates: Three regex substitutions run in sequence: strip anything that isn't a lowercase letter, digit, space, or hyphen; turn runs of whitespace into a single hyphen; then collapse multiple hyphens into one. A final .strip("-") removes any leading/trailing hyphen.

📌 Significance: This is a good example of chaining small, single-purpose regex operations rather than one giant unreadable pattern — each line is easy to test and reason about independently.
📨 Project 3 – Mail-Merge Style Templating

Fill a message template from a dictionary

Useful for bulk emails, notifications, or generated reports where the same message shape is reused with different data each time.

template = """Hi {name},

Your order #{order_id} totaling ${total:,.2f} has shipped.
Estimated delivery: {eta}.

— Poetic Bytes"""

orders = [
    {"name": "Ana", "order_id": 1042, "total": 129.5, "eta": "July 4"},
    {"name": "Marco", "order_id": 1043, "total": 2499.99, "eta": "July 6"},
]

for order in orders:
    message = template.format(**order)
    print(message)
    print("=" * 40)
🔍 Why this code is used: Mail-merge templating is the string-processing pattern behind newsletter tools, invoice generators, and notification systems.

⚙️ How it operates: template.format(**order) unpacks each dictionary into keyword arguments matching the named placeholders ({name}, {order_id}, etc). The {total:,.2f} specifier formats the float as currency with thousands separators, right inside the template.

📌 Significance: Unlike f‑strings (which need the variables to exist at write time), .format() works beautifully with data loaded from a database or JSON file, making it the right formatting tool for this specific job.

❓ Frequently Asked Questions

Common questions about Python strings, answered

Are Python strings mutable?

+

No. Strings are immutable – once created, they cannot be changed in place. Every operation that modifies a string (like .upper() or .replace()) returns a new string. This design makes strings safe to use as dictionary keys and ensures predictable behaviour.

What is the difference between find() and index()?

+

find() returns -1 if the substring is not found, while index() raises a ValueError. Use find() when you want to check existence without error handling, and index() when you're certain the substring exists.

How do I check if a string starts with a certain prefix?

+

Use .startswith(prefix). It returns True if the string begins with the given prefix. You can also pass a tuple of prefixes to check multiple possibilities.

What is the fastest way to concatenate many strings?

+

Use ''.join(list_of_strings). This is significantly faster than repeated + because it allocates the final string only once. Avoid + inside loops.

What are raw strings (r"...") used for?

+

Raw strings treat backslashes as literal characters – they do not interpret escape sequences. They are ideal for file paths (Windows) and regular expressions, where backslashes are common and you don't want them to be escaped.

How do I reverse a string?

+

Use slicing with a step of -1: s[::-1]. This creates a reversed copy. For large strings, consider ''.join(reversed(s)).

What is the difference between is and == for strings?

+

== compares the value (the characters). is checks whether two variables refer to the same object in memory. For strings, you almost always want == because Python may intern some strings (small ones) but you shouldn't rely on that.

How can I remove a prefix or suffix from a string?

+

Python 3.9+ provides .removeprefix(prefix) and .removesuffix(suffix). They are more readable than slicing and handle cases where the prefix/suffix doesn't exist gracefully.

What is the difference between str and bytes?

+

str is a sequence of Unicode characters (text). bytes is a sequence of integers between 0 and 255 (binary data). Use .encode() to go from str to bytes, and .decode() to go from bytes to str. This is important when reading/writing files or sending data over networks.

Why do I get UnicodeDecodeError when reading a file?

+

This happens when you try to read a file with the wrong encoding. Specify the correct encoding when opening the file: open('file.txt', 'r', encoding='utf-8'). If the file has a different encoding (e.g., cp1252), use that instead. Alternatively, read as bytes and decode.

Why doesn't my_string.upper() change my variable?

+

Because strings are immutable, .upper() returns a new string instead of modifying the original in place. You must reassign it: my_string = my_string.upper(). Calling the method alone and discarding the result is one of the most common beginner mistakes.

Should I use f‑strings or .format() in 2026?

+

Use f‑strings for almost everything — they're faster and more readable. Reach for .format() only when the template string itself is dynamic (e.g., loaded from a config file or database), since f‑strings must be written directly in source code to work.

import string

def word_freq(text: str) -> dict:
    cleaned = text.lower().translate(str.maketrans("", "", string.punctuation))
    counts = {}
    for word in cleaned.split():
        counts[word] = counts.get(word, 0) + 1
    return counts
