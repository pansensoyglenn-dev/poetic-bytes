⚡ Short‑Circuit Evaluation — and and or Stop Early

Python's logical operators short‑circuit: they stop evaluating as soon as the result is known.

⚡ 5.1 and Short‑Circuit

and stops at the first False value.

Step-by-step explanation:

  • Step 1: Define a function that prints when called.
  • Step 2: Use False and expensive_function() — the function is never called.
  • Step 3: Use True and expensive_function() — the function is called.
  • Step 4: Show that the second operand is only evaluated if needed.
python 
def expensive_operation():
    print("Expensive operation executed!")
    return True

def short_circuit_and():
    print("False and expensive_operation():")
    result = False and expensive_operation()
    print(f"Result: {result}")

    print("\nTrue and expensive_operation():")
    result = True and expensive_operation()
    print(f"Result: {result}")

short_circuit_and()

⚡ 5.2 or Short‑Circuit

or stops at the first True value.

Step-by-step explanation:

  • Step 1: Use True or expensive_function() — the function is never called.
  • Step 2: Use False or expensive_function() — the function is called.
  • Step 3: Show that the second operand is only evaluated if needed.
  • Step 4: This is useful for defaults: name = user_input or "Guest".
python 
def expensive_operation():
    print("Expensive operation executed!")
    return True

def short_circuit_or():
    print("True or expensive_operation():")
    result = True or expensive_operation()
    print(f"Result: {result}")

    print("\nFalse or expensive_operation():")
    result = False or expensive_operation()
    print(f"Result: {result}")

    print("\nUsing or for defaults:")
    user_input = ""
    name = user_input or "Guest"
    print(f"Name: {name}")

short_circuit_or()
⚠️ Short‑circuit evaluation is a powerful tool, but be careful when the second operand has side effects. It won't run if the first operand determines the result.

❓ Frequently Asked Questions — Booleans

Common questions about Python booleans, answered.

What is the difference between == and is?

+

== compares values (equality). is compares identity (memory address). For booleans, True is True is True, but 1 == True is also True while 1 is True is False. Use == for value comparison unless you specifically need identity.

Why does if []: evaluate to False?

+

Empty collections are falsy in Python. This is intentional — it allows you to write if items: to check if a list or dict is non‑empty, which is cleaner than if len(items) > 0:.

What is short‑circuit evaluation?

+

Short‑circuit evaluation means Python stops evaluating a logical expression as soon as the result is determined. For and, it stops at the first False. For or, it stops at the first True. This can improve performance and prevent errors.

Can I use booleans in arithmetic?

+

Yes — True is 1 and False is 0. However, this is generally considered bad practice because it makes code harder to read. Use explicit conditions instead of relying on boolean arithmetic.

How do I check if a variable is None?

+

Use if variable is None: or if variable is not None:. Since None is a singleton, identity comparison (is) is the correct way to check for None.

What does any() and all() do?

+

any(iterable) returns True if at least one element is truthy. all(iterable) returns True only if all elements are truthy. They are useful for validating collections.
