Accessibility Statement

Poetic Bytes is committed to making Python education accessible to everyone, regardless of ability.

Last updated: June 18, 2026

WCAG 2.1 AA compliant · Continuous improvement

My Commitment

I believe that learning to code should be accessible to everyone. Poetic Bytes is designed with accessibility as a core principle, not an afterthought. I strive to ensure that people with disabilities can access, understand, and benefit from all Python tutorials and resources on this site.

Accessibility Features

This site includes the following accessibility features:

⌨️

Keyboard Navigation

All interactive elements can be accessed using a keyboard alone. You can navigate using the Tab key, activate buttons with Enter or Space, and close overlays with Escape.

📖

Screen Reader Compatible

I use semantic HTML elements, proper heading hierarchies, ARIA landmarks, and descriptive link text to ensure screen readers can interpret the content correctly. Images include meaningful alt text.

🎨

High Contrast & Readable Typography

I use high-contrast colour combinations (WCAG AA compliant) and a clear, legible font (Syne). Text size is responsive and can be enlarged using browser zoom without breaking the layout.

📱

Responsive & Mobile-Friendly

The site is fully responsive and works across devices — from desktop monitors to small phone screens. Content reflows without horizontal scrolling.

🔊

Reduced Motion

Animations are subtle and can be disabled via the prefers-reduced-motion media query. The typewriter effect on the homepage respects this user preference.

📝

Clear Language & Structure

I write in plain English, use clear headings, and break content into scannable sections. Code examples are clearly separated from explanatory text.

Known Limitations

I am continuously working to improve accessibility. Some known areas I am actively addressing:

  • Older code examples may not include full ARIA labelling — I am gradually updating them.
  • Some third-party embedded content (e.g., ads, YouTube videos) may not be fully accessible. I am working with providers to improve this.
  • The Pyodide Python runner, while functional, may have accessibility limitations in the console output. I am exploring improvements.

Standards & Compliance

I aim to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. I regularly audit the site and test it with automated tools and manual checks to ensure compliance.

Testing Tools

I use the following tools to test and improve accessibility:

  • axe DevTools – automated accessibility testing in the browser.
  • WAVE – web accessibility evaluation tool.
  • Lighthouse – accessibility audits built into Chrome DevTools.
  • NVDA – free screen reader testing on Windows.
  • VoiceOver – screen reader testing on macOS and iOS.

Feedback & Reporting Issues

I welcome your feedback. If you encounter any accessibility barriers while using this site, please let me know. I take accessibility seriously and will address issues as quickly as possible.

To report an accessibility issue or request an accommodation, please:

  • Email me directly at pansensoyglenn150@gmail.com
  • Use the contact page to send a message

Please include a description of the issue and the URL where you encountered it. I will respond within 2 business days.

Continuous Improvement

Accessibility is an ongoing journey, not a one-time task. I am committed to:

  • Regularly reviewing and updating the site's accessibility.
  • Testing with real users who have disabilities.
  • Staying up-to-date with evolving accessibility standards and best practices.
  • Building accessibility into every new feature I add.

Thank you for learning Python with Poetic Bytes. Everyone is welcome here.
