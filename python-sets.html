🔵 What Are Python Sets?

An unordered collection of unique, hashable objects — built for speed

A set is an unordered collection of unique elements. Sets are mutable (you can add and remove items), but the items themselves must be hashable (immutable). The defining characteristics:

  • Unordered — no index, no ordering guarantee
  • Unique — duplicates are automatically removed
  • Fast membership — O(1) average time for in checks
  • Mathematical operations — union, intersection, difference, symmetric difference
# A set of unique languages
languages = {"Python", "JavaScript", "Rust", "Python"}
print(languages)   # {'Python', 'JavaScript', 'Rust'} — duplicates removed
print(len(languages)) # 3
print("Python" in languages) # True — O(1) membership
💡 Sets are implemented as hash tables — the same underlying structure as dictionaries, but without the values. This gives them O(1) average time for add, remove, and membership tests.

🔧 Creating Sets

Different ways to build a set in Python

📦 Set Literals & Constructors

Use curly braces { } for a set literal — but be careful: {} creates an empty dictionary, not a set. Use set() for an empty set.

# Set literal
colors = {"red", "green", "blue"}
print(colors)   # {'red', 'green', 'blue'}

# Empty set — must use set()
empty_set = set()
empty_dict = {}
print(type(empty_set))   # <class 'set'>
print(type(empty_dict))  # <class 'dict'>

# From a list (deduplicates automatically)
numbers = [1, 2, 2, 3, 3, 3, 4]
unique_nums = set(numbers)
print(unique_nums)   # {1, 2, 3, 4}

# From a string (unique characters)
chars = set("hello")
print(chars)   # {'h', 'e', 'l', 'o'} — 'l' appears once

# From a range
evens = set(range(0, 10, 2))
print(evens)   # {0, 2, 4, 6, 8}
💡 set() accepts any iterable — lists, tuples, strings, ranges, and even other sets. Duplicates are automatically discarded.

⚠️ What Can Go Inside a Set?

Set elements must be hashable — that means they are immutable. Lists, dicts, and other sets cannot be elements of a set. Tuples are fine because they are immutable.

# Valid set elements — all immutable types
valid = {42, 3.14, "hello", (1, 2, 3), True, None}
print(valid)

# Invalid — TypeError: unhashable type: 'list'
# invalid = {1, [2, 3], 4}

# Invalid — TypeError: unhashable type: 'dict'
# invalid = {1, {"a": 2}, 3}

# Invalid — TypeError: unhashable type: 'set'
# invalid = {1, {2, 3}, 4}

# But frozenset can be nested inside a set
valid = {1, frozenset({2, 3}), 4}
print(valid)   # {1, 4, frozenset({2, 3})}
⚠️ Lists, dictionaries, and sets are not hashable — they cannot be stored in a set. Use tuples or frozensets instead.

🔗 Set Operations — Union, Intersection, Difference

Python supports mathematical set operations with both operators and methods

🔀 Union — | or .union()

Returns a new set with all elements from both sets. Duplicates are removed.

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Using the | operator
result = a | b
print(result)   # {1, 2, 3, 4, 5, 6}

# Using the union() method
result = a.union(b)
print(result)   # {1, 2, 3, 4, 5, 6}

# Union with multiple sets
c = {7, 8}
result = a | b | c
print(result)   # {1, 2, 3, 4, 5, 6, 7, 8}

🔀 Intersection — & or .intersection()

Returns a new set with elements that appear in both sets.

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Using the & operator
result = a & b
print(result)   # {3, 4}

# Using the intersection() method
result = a.intersection(b)
print(result)   # {3, 4}

🔀 Difference — - or .difference()

Returns a new set with elements in the first set that are not in the second.

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Using the - operator
result = a - b
print(result)   # {1, 2}

# Using the difference() method
result = a.difference(b)
print(result)   # {1, 2}

# Order matters — b - a gives {5, 6}
result = b - a
print(result)   # {5, 6}

🔀 Symmetric Difference — ^ or .symmetric_difference()

Returns a new set with elements in either set, but not both.

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# Using the ^ operator
result = a ^ b
print(result)   # {1, 2, 5, 6}

# Using the symmetric_difference() method
result = a.symmetric_difference(b)
print(result)   # {1, 2, 5, 6}

🔍 Subset & Superset Checks

Test whether one set is contained in another.

a = {1, 2, 3}
b = {1, 2, 3, 4, 5}

print(a.issubset(b))   # True — all elements of a are in b
print(a <= b)          # True — subset (same operator)
print(a < b)           # True — proper subset (a != b)

print(b.issuperset(a)) # True — all elements of a are in b
print(b >= a)          # True — superset
print(b > a)           # True — proper superset (b != a)

# Disjoint — no common elements
c = {7, 8, 9}
print(a.isdisjoint(c)) # True
💡 The operators <=, <, >=, > are intuitive for subset and superset checks. Use isdisjoint() to test if two sets have no overlap — it's faster than len(a & b) == 0.

🛠️ Set Methods — Add, Remove, Discard, Pop

Mutating sets with essential methods

➕ Adding Elements — .add() & .update()

.add() inserts a single element. .update() merges multiple elements from another iterable.

s = {1, 2, 3}

# Add a single element
s.add(4)
print(s)   # {1, 2, 3, 4}

# Add multiple elements from an iterable
s.update([5, 6, 7])
print(s)   # {1, 2, 3, 4, 5, 6, 7}

# Update with another set
s.update({8, 9})
print(s)   # {1, 2, 3, 4, 5, 6, 7, 8, 9}

# Duplicates are ignored
s.add(3)   # No change — 3 already exists
print(s)   # unchanged

➖ Removing Elements — .remove(), .discard(), .pop(), .clear()

.remove() raises KeyError if the element is missing. .discard() does nothing if the element is missing. .pop() removes and returns an arbitrary element. .clear() empties the set.

s = {1, 2, 3, 4, 5}

# remove() — raises KeyError if missing
s.remove(3)
print(s)   # {1, 2, 4, 5}
# s.remove(10)  # KeyError: 10

# discard() — safe, no error if missing
s.discard(4)
print(s)   # {1, 2, 5}
s.discard(99)   # No error, does nothing

# pop() — removes and returns an arbitrary element
item = s.pop()
print(item)   # 1 or 2 or 5 (arbitrary)
print(s)      # remaining elements

# clear() — removes all elements
s.clear()
print(s)   # set()
⚠️ .pop() on an empty set raises KeyError. Always check if s: before calling .pop().

📋 Copying — .copy()

Sets are mutable — use .copy() to create a shallow copy. Since sets cannot contain mutable elements, shallow copy is sufficient.

original = {1, 2, 3}
copy = original.copy()
copy.add(4)

print(original)   # {1, 2, 3} — unchanged
print(copy)       # {1, 2, 3, 4}

# Assignment does NOT copy — both names point to the same set
alias = original
alias.add(5)
print(original)   # {1, 2, 3, 5} — original changed!
💡 Use .copy() when you need to modify a set without affecting the original. Simple assignment just creates another reference to the same object.

📋 Set Comprehensions

Build sets concisely with comprehension syntax

Set comprehensions use curly braces { } with a for clause — similar to list comprehensions but producing a set (unique, unordered).

# Squares of even numbers from 0 to 9
squares = {x**2 for x in range(10) if x % 2 == 0}
print(squares)   # {0, 4, 16, 36, 64}

# Unique lengths of words in a sentence
words = ["python", "is", "fun", "and", "powerful"]
lengths = {len(word) for word in words}
print(lengths)   # {2, 3, 4, 6, 8}

# Characters in a string (unique)
unique_chars = {char for char in "mississippi"}
print(unique_chars)   # {'p', 's', 'i', 'm'}

# Filtering with conditions
nums = {x for x in range(-5, 6) if x > 0}
print(nums)   # {1, 2, 3, 4, 5}
💡 Set comprehensions are a concise way to transform data while automatically removing duplicates. They are often faster than building a list and then calling set().

❄️ Frozenset — Immutable Sets

An immutable, hashable version of a set

A frozenset is an immutable set. You cannot add or remove elements after creation. Because it is immutable, it is hashable and can be used as a dictionary key or as an element of another set.

# Creating a frozenset
frozen = frozenset({1, 2, 3, 4})
print(frozen)   # frozenset({1, 2, 3, 4})

# All set operations work on frozensets
a = frozenset({1, 2, 3})
b = frozenset({2, 3, 4})
print(a | b)   # frozenset({1, 2, 3, 4})
print(a & b)   # frozenset({2, 3})

# frozenset as a dict key
key = frozenset({"Python", "Rust"})
lookup = {key: "backend stack"}
print(lookup[key])   # "backend stack"

# frozenset inside a set
s = {1, 2, frozenset({3, 4})}
print(s)   # {1, 2, frozenset({3, 4})}

# Cannot modify a frozenset
# frozen.add(5)  # AttributeError: 'frozenset' object has no attribute 'add'
💡 Use frozenset when you need an immutable, hashable set — for dictionary keys, caching, or when you want to guarantee that the data doesn't change.

🔍 Membership Testing — The Superpower of Sets

Sets offer the fastest possible membership tests in Python

Membership testing with in on a set is O(1) — constant time, regardless of the set size. The same check on a list is O(n) — it scans the entire list. This difference becomes dramatic with large collections.

import time

# Create a large collection
size = 100_000
list_data = list(range(size))
set_data = set(range(size))

# Membership test on a list — O(n)
start = time.time()
result = 99_999 in list_data
list_time = time.time() - start

# Membership test on a set — O(1)
start = time.time()
result = 99_999 in set_data
set_time = time.time() - start

print(f"List: {list_time:.6f}s")
print(f"Set:  {set_time:.6f}s")

# For membership checks, always prefer a set over a list
# Especially when the collection is large or checked frequently
💡 If you only need to check whether an item exists (and don't need ordering or duplicates), always use a set for membership testing. The performance difference is enormous.

⚡ Performance — Sets vs Lists vs Tuples

When to use a set and when to choose another data structure

🔍

Membership (in)

Set: O(1) — hash table lookup. List: O(n) — scans every element. Tuple: O(n). Use a set when you frequently check for existence.

📋

Ordering

Set: Unordered — no guarantee. List: ordered by insertion. Tuple: ordered. Use lists/tuples when order matters.

🔁

Duplicates

Set: No duplicates — automatically deduplicated. List: can contain duplicates. Use a set when you need unique elements.

🔧

Mutability

Set: mutable (add/remove). Tuple: immutable. List: mutable. Use tuples for fixed data, sets for dynamic unique collections.

🔑

Hashable

Set: Not hashable — cannot be used as dict key. Frozenset: hashable. Tuple: hashable (if elements are hashable).

📊

Memory

Set: More memory — hash table overhead. List: lower memory. Tuple: lowest memory (compact).

The Rule of Thumb

Use a set when you need uniqueness, fast membership, and order doesn't matter. Use a list when you need order, allow duplicates, or need index access. Use a tuple for ordered, immutable, hashable collections.

💼 Real-World Use Cases for Sets

Where sets shine in production code

🔹 Removing Duplicates from a List

items = ["apple", "banana", "apple", "cherry", "banana"]
unique = list(set(items))
print(unique)   # ['apple', 'banana', 'cherry'] (order may vary)

# Preserve order (Python 3.7+ preserves dict insertion order)
unique_ordered = list(dict.fromkeys(items))
print(unique_ordered)   # ['apple', 'banana', 'cherry']

🔹 Finding Common Elements Between Collections

team_a = {"Alice", "Bob", "Charlie", "Diana"}
team_b = {"Charlie", "Eve", "Frank", "Alice"}

# Who is on both teams?
common = team_a & team_b
print(common)   # {'Alice', 'Charlie'}

# Who is only on team A?
only_a = team_a - team_b
print(only_a)   # {'Bob', 'Diana'}

🔹 Tracking Visited Items

# Track which URLs have been processed
visited = set()
urls = ["page1", "page2", "page1", "page3", "page2"]

for url in urls:
    if url not in visited:
        visited.add(url)
        print(f"Processing: {url}")

# Output: page1, page2, page3 (each only once)

🔹 Filtering Invalid Items

valid_statuses = {200, 201, 202, 204}
responses = [200, 404, 201, 500, 200, 202]

# Keep only valid responses
valid = [r for r in responses if r in valid_statuses]
print(valid)   # [200, 201, 200, 202]
